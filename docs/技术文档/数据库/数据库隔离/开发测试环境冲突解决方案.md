# 开发测试环境冲突解决方案

## 问题场景

```
本地环境 (localhost:8080)
    ↓
端口映射/内网穿透 (ngrok/frp)
    ↓
公网访问 (https://xxx.ngrok.io)
    ↓
测试人员正在测试... ✅

你：想改代码、重启服务 💥 冲突！
测试人员：服务断了，测试中断 😰
```

**核心矛盾：**
- 开发需要频繁修改和重启
- 测试需要稳定的环境
- 共用同一个服务实例

## 方案一：时间协调（最简单）

### 适用场景

- ✅ 测试不频繁（一天 1-2 次）
- ✅ 团队人数少（2-3 人）
- ✅ 开发和测试时间可协调

### 实施方案

#### 1. 建立测试时间表

```
每日测试时间表：
─────────────────────────────────
时间段          状态        负责人
─────────────────────────────────
09:00-11:00    开发调试    开发人员
11:00-12:00    集中测试    测试人员
14:00-16:00    开发调试    开发人员
16:00-17:30    集中测试    测试人员
─────────────────────────────────
```

#### 2. 沟通机制

**群通知模板：**

```
开发人员：
"我要重启服务了，预计 5 分钟，大家稍等 ⏰"

测试人员回复：
"OK，我先暂停测试 👌"
或
"等一下，我还有 2 个用例要跑完，3 分钟 ⏱️"
```

#### 3. 状态标识

**在共享文档/看板上标识服务状态：**

```markdown
## 服务状态
- 🟢 运行中 - 可以测试
- 🟡 开发中 - 即将重启
- 🔴 停机中 - 正在重启
- 🟣 维护中 - 暂时不可用

当前状态：🟢 运行中
最后更新：2024-12-04 10:30
```

### 优缺点

| 优点 | 缺点 |
|-----|-----|
| ✅ 零成本，立即实施 | ❌ 频繁打断测试流程 |
| ✅ 不需要技术改造 | ❌ 沟通成本高 |
| ✅ 适合小团队 | ❌ 效率低 |
|  | ❌ 测试不够独立 |

### 适用判断

**适合：** 偶尔测试，测试人员和开发人员在同一个办公室

**不适合：** 频繁测试，异地协作

## 方案二：双端口运行（推荐）

### 核心思路

**运行两个服务实例：**

```
本地开发实例 (localhost:8080)  ← 开发人员使用
    - 可以随时修改、重启
    - 不对外暴露

测试稳定实例 (localhost:8081)  ← 测试人员使用
    - 通过端口映射到公网
    - 保持稳定运行
    - 代码稳定后才更新
```

### 实施步骤

#### 步骤一：修改配置支持多端口

**application-dev.yml（开发实例）：**

```yaml
server:
  port: 8080

spring:
  application:
    name: metersphere-dev
  profiles:
    active: dev
  datasource:
    url: jdbc:mysql://localhost:3306/metersphere_dev?...
    username: root
    password: dev123
```

**application-test-stable.yml（测试实例）：**

```yaml
server:
  port: 8081

spring:
  application:
    name: metersphere-test-stable
  profiles:
    active: test-stable
  datasource:
    url: jdbc:mysql://localhost:3306/metersphere_test_stable?...
    username: root
    password: test123
```

#### 步骤二：创建启动脚本

**start-dev.sh（开发实例）：**

```bash
#!/bin/bash

echo "========== 启动开发实例 (8080) =========="

# 检查端口是否被占用
if lsof -Pi :8080 -sTCP:LISTEN -t >/dev/null ; then
    echo "端口 8080 已被占用"
    read -p "是否强制停止？(y/n) " -n 1 -r
    echo
    if [[ $REPLY =~ ^[Yy]$ ]]; then
        kill -9 $(lsof -t -i:8080)
        sleep 2
    else
        exit 1
    fi
fi

# 启动开发实例
export SPRING_PROFILES_ACTIVE=dev
cd backend

# 方式一：Maven 启动（调试方便）
mvn spring-boot:run -Dspring-boot.run.profiles=dev

# 方式二：Jar 包启动（性能更好）
# java -jar target/metersphere.jar --spring.profiles.active=dev
```

**start-test-stable.sh（测试实例）：**

```bash
#!/bin/bash

echo "========== 启动测试稳定实例 (8081) =========="

# 检查端口是否被占用
if lsof -Pi :8081 -sTCP:LISTEN -t >/dev/null ; then
    echo "⚠️  警告：端口 8081 已被占用（测试实例可能正在运行）"
    echo "测试人员可能正在使用，建议不要强制停止！"
    read -p "确定要停止吗？(yes/no) " -r
    if [[ $REPLY == "yes" ]]; then
        kill -9 $(lsof -t -i:8081)
        sleep 2
    else
        exit 1
    fi
fi

# 后台启动测试实例
export SPRING_PROFILES_ACTIVE=test-stable
cd backend

nohup java -jar target/metersphere.jar \
    --spring.profiles.active=test-stable \
    > logs/test-stable.log 2>&1 &

echo $! > test-stable.pid
echo "✅ 测试实例已启动，PID: $(cat test-stable.pid)"
echo "日志文件: logs/test-stable.log"
echo "访问地址: http://localhost:8081"
```

**stop-test-stable.sh（停止测试实例）：**

```bash
#!/bin/bash

echo "========== 停止测试稳定实例 =========="

if [ -f test-stable.pid ]; then
    PID=$(cat test-stable.pid)
    
    # 二次确认
    echo "⚠️  警告：即将停止测试实例（测试人员可能正在使用）"
    read -p "确定要停止吗？(yes/no) " -r
    
    if [[ $REPLY == "yes" ]]; then
        kill $PID
        rm test-stable.pid
        echo "✅ 测试实例已停止"
    else
        echo "已取消"
    fi
else
    echo "测试实例未运行"
fi
```

**update-test-stable.sh（更新测试实例）：**

```bash
#!/bin/bash

echo "========== 更新测试稳定实例 =========="

# 1. 二次确认
echo "⚠️  即将更新测试实例，测试人员正在使用的服务会中断 1-2 分钟"
read -p "确定要更新吗？(yes/no) " -r

if [[ $REPLY != "yes" ]]; then
    echo "已取消"
    exit 0
fi

# 2. 通知测试人员（可选）
echo "💬 建议先通知测试人员：我要更新测试环境了，预计 2 分钟"
read -p "已通知？按回车继续..." 

# 3. 备份当前版本
if [ -f test-stable.pid ]; then
    echo "📦 备份当前版本..."
    cp target/metersphere.jar target/metersphere.jar.backup
fi

# 4. 停止测试实例
echo "⏸️  停止测试实例..."
./stop-test-stable.sh

# 5. 构建新版本
echo "🔨 构建新版本..."
cd backend
mvn clean package -DskipTests

# 6. 启动新版本
echo "🚀 启动新版本..."
./start-test-stable.sh

echo ""
echo "✅ 测试实例更新完成！"
echo "📝 通知测试人员：测试环境已更新，可以继续测试了"
```

#### 步骤三：配置端口映射

**映射两个端口到不同的公网地址：**

**使用 ngrok：**

```bash
# 开发环境（你自己用）
ngrok http 8080 --subdomain=myapp-dev

# 测试环境（测试人员用）
ngrok http 8081 --subdomain=myapp-test-stable
```

**使用 frp：**

```ini
# frpc.ini

[common]
server_addr = your-frp-server.com
server_port = 7000

# 开发环境
[metersphere-dev]
type = http
local_port = 8080
custom_domains = dev.yourdomain.com

# 测试稳定环境
[metersphere-test-stable]
type = http
local_port = 8081
custom_domains = test.yourdomain.com
```

#### 步骤四：使用流程

**日常开发：**

```bash
# 1. 启动两个实例
./start-dev.sh              # 开发实例 (8080)
./start-test-stable.sh      # 测试实例 (8081)

# 2. 开发过程中
# 随时修改代码，重启开发实例 (8080)
# 测试实例 (8081) 继续稳定运行

# 3. 开发完成后，更新测试实例
./update-test-stable.sh     # 更新到测试实例 (8081)
```

**给测试人员的说明：**

```markdown
## 测试环境访问说明

**测试稳定环境：** https://test.yourdomain.com
- 这是给你测试用的稳定环境
- 一般不会中断
- 如果需要更新，我会提前通知你

**开发环境：** https://dev.yourdomain.com
- 这是我的开发环境，会频繁重启
- 请不要用这个环境测试
```

### 数据库隔离

**创建两个数据库：**

```sql
-- 开发数据库（开发人员用，可以随便改）
CREATE DATABASE metersphere_dev CHARACTER SET utf8mb4;

-- 测试稳定数据库（测试人员用，定期同步）
CREATE DATABASE metersphere_test_stable CHARACTER SET utf8mb4;
```

**定期同步数据：**

```bash
#!/bin/bash
# sync-db.sh

echo "同步数据：dev → test_stable"

# 导出开发数据（不包含数据，只要结构）
mysqldump -u root -p \
  --no-data \
  metersphere_dev > schema.sql

# 导入到测试稳定库
mysql -u root -p metersphere_test_stable < schema.sql

# 导入测试数据
mysql -u root -p metersphere_test_stable < testdata.sql

echo "✅ 数据同步完成"
```

### 优缺点

| 优点 | 缺点 |
|-----|-----|
| ✅ 开发和测试互不干扰 | ❌ 需要运行两个实例 |
| ✅ 测试环境稳定 | ❌ 占用更多资源（内存、CPU） |
| ✅ 更新可控 | ❌ 需要管理两套配置 |
| ✅ 适合频繁开发 | ❌ 数据库需要同步 |
| ✅ 成本低（本地运行） |  |

- 16GB 内存 + 4 核 CPU 及以上

## 方案三：蓝绿部署（本地版）

### 核心思路

```
蓝色实例 (8081) 🟦 ← 当前对外服务（测试人员在用）
绿色实例 (8082) 🟩 ← 更新新版本（准备切换）

更新流程：
1. 蓝色实例 (8081) 对外服务
2. 在绿色实例 (8082) 部署新版本
3. 测试绿色实例 (8082)
4. 切换：绿色变为对外服务
5. 蓝色实例成为备用/下次更新用
```

### 实施步骤

#### 步骤一：Nginx 反向代理配置

**安装 Nginx：**

```bash
# macOS
brew install nginx

# Ubuntu
sudo apt install nginx

# CentOS
sudo yum install nginx
```

**nginx.conf：**

```nginx
# /usr/local/etc/nginx/nginx.conf (macOS)
# /etc/nginx/nginx.conf (Linux)

http {
    # 上游服务器组
    upstream metersphere_backend {
        server localhost:8081;  # 默认指向蓝色实例
    }

    server {
        listen 8000;  # Nginx 监听 8000 端口
        server_name localhost;

        location / {
            proxy_pass http://metersphere_backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            
            # 健康检查
            proxy_connect_timeout 5s;
            proxy_send_timeout 30s;
            proxy_read_timeout 30s;
        }
        
        # 健康检查端点
        location /health {
            access_log off;
            return 200 "healthy\n";
        }
    }
}
```

#### 步骤二：蓝绿切换脚本

**blue-green-deploy.sh：**

```bash
#!/bin/bash

set -e

# 颜色定义
GREEN='\033[0;32m'
BLUE='\033[0;34m'
YELLOW='\033[1;33m'
RED='\033[0;31m'
NC='\033[0m'

# 端口配置
BLUE_PORT=8081
GREEN_PORT=8082
NGINX_PORT=8000

# 获取当前活跃的服务
get_active_service() {
    CURRENT_UPSTREAM=$(grep "server localhost:" /usr/local/etc/nginx/nginx.conf | grep -o "[0-9]*")
    if [ "$CURRENT_UPSTREAM" == "$BLUE_PORT" ]; then
        echo "blue"
    else
        echo "green"
    fi
}

# 启动服务
start_service() {
    local PORT=$1
    local COLOR=$2
    
    echo -e "${BLUE}启动 ${COLOR} 实例 (端口 $PORT)...${NC}"
    
    nohup java -jar target/metersphere.jar \
        --server.port=$PORT \
        --spring.profiles.active=test \
        > logs/${COLOR}.log 2>&1 &
    
    echo $! > ${COLOR}.pid
    
    # 等待服务启动
    echo "等待服务启动..."
    for i in {1..30}; do
        if curl -s http://localhost:$PORT/actuator/health > /dev/null; then
            echo -e "${GREEN}✅ ${COLOR} 实例启动成功${NC}"
            return 0
        fi
        sleep 1
    done
    
    echo -e "${RED}❌ ${COLOR} 实例启动失败${NC}"
    return 1
}

# 停止服务
stop_service() {
    local COLOR=$1
    
    if [ -f ${COLOR}.pid ]; then
        PID=$(cat ${COLOR}.pid)
        echo -e "${YELLOW}停止 ${COLOR} 实例 (PID: $PID)...${NC}"
        kill $PID
        rm ${COLOR}.pid
        echo -e "${GREEN}✅ ${COLOR} 实例已停止${NC}"
    else
        echo -e "${YELLOW}${COLOR} 实例未运行${NC}"
    fi
}

# 切换 Nginx 上游
switch_nginx() {
    local NEW_PORT=$1
    local NEW_COLOR=$2
    
    echo -e "${BLUE}切换 Nginx 到 ${NEW_COLOR} 实例 (端口 $NEW_PORT)...${NC}"
    
    # 备份配置
    cp /usr/local/etc/nginx/nginx.conf /usr/local/etc/nginx/nginx.conf.backup
    
    # 修改上游服务器
    sed -i '' "s/server localhost:[0-9]*/server localhost:$NEW_PORT/" /usr/local/etc/nginx/nginx.conf
    
    # 重载 Nginx
    nginx -t && nginx -s reload
    
    echo -e "${GREEN}✅ Nginx 已切换到 ${NEW_COLOR} 实例${NC}"
}

# 健康检查
health_check() {
    local PORT=$1
    local COLOR=$2
    
    echo -e "${BLUE}健康检查 ${COLOR} 实例...${NC}"
    
    # 检查健康端点
    if ! curl -s http://localhost:$PORT/actuator/health | grep -q "UP"; then
        echo -e "${RED}❌ ${COLOR} 实例健康检查失败${NC}"
        return 1
    fi
    
    # 简单功能测试
    if ! curl -s http://localhost:$PORT/api/test > /dev/null; then
        echo -e "${YELLOW}⚠️  警告：${COLOR} 实例功能测试异常${NC}"
        read -p "是否继续？(y/n) " -n 1 -r
        echo
        if [[ ! $REPLY =~ ^[Yy]$ ]]; then
            return 1
        fi
    fi
    
    echo -e "${GREEN}✅ ${COLOR} 实例健康检查通过${NC}"
    return 0
}

# 主流程
main() {
    echo "=========================================="
    echo "       蓝绿部署脚本 - MeterSphere"
    echo "=========================================="
    echo ""
    
    # 1. 确定当前活跃服务
    CURRENT_SERVICE=$(get_active_service)
    echo -e "当前活跃服务: ${BLUE}$CURRENT_SERVICE${NC} 实例"
    
    # 2. 确定目标服务
    if [ "$CURRENT_SERVICE" == "blue" ]; then
        TARGET_SERVICE="green"
        TARGET_PORT=$GREEN_PORT
        OLD_SERVICE="blue"
        OLD_PORT=$BLUE_PORT
    else
        TARGET_SERVICE="blue"
        TARGET_PORT=$BLUE_PORT
        OLD_SERVICE="green"
        OLD_PORT=$GREEN_PORT
    fi
    
    echo -e "目标服务: ${GREEN}$TARGET_SERVICE${NC} 实例 (端口 $TARGET_PORT)"
    echo ""
    
    # 3. 构建新版本
    echo -e "${BLUE}构建新版本...${NC}"
    cd backend
    mvn clean package -DskipTests
    cd ..
    echo -e "${GREEN}✅ 构建完成${NC}"
    echo ""
    
    # 4. 启动目标服务
    start_service $TARGET_PORT $TARGET_SERVICE
    echo ""
    
    # 5. 健康检查
    if ! health_check $TARGET_PORT $TARGET_SERVICE; then
        echo -e "${RED}❌ 部署失败，回滚...${NC}"
        stop_service $TARGET_SERVICE
        exit 1
    fi
    echo ""
    
    # 6. 切换流量
    echo -e "${YELLOW}⚠️  即将切换流量到 ${TARGET_SERVICE} 实例${NC}"
    echo "测试人员的请求会切换到新版本"
    read -p "确认切换？(y/n) " -n 1 -r
    echo
    if [[ ! $REPLY =~ ^[Yy]$ ]]; then
        echo -e "${RED}已取消切换${NC}"
        stop_service $TARGET_SERVICE
        exit 1
    fi
    
    switch_nginx $TARGET_PORT $TARGET_SERVICE
    echo ""
    
    # 7. 验证切换
    echo -e "${BLUE}验证切换...${NC}"
    sleep 2
    if curl -s http://localhost:$NGINX_PORT/actuator/health | grep -q "UP"; then
        echo -e "${GREEN}✅ 切换成功！${NC}"
    else
        echo -e "${RED}❌ 切换失败，回滚...${NC}"
        switch_nginx $OLD_PORT $OLD_SERVICE
        stop_service $TARGET_SERVICE
        exit 1
    fi
    echo ""
    
    # 8. 停止旧服务
    echo -e "${YELLOW}旧服务 ($OLD_SERVICE) 仍在运行，可以作为回滚备份${NC}"
    read -p "是否停止旧服务？(y/n) " -n 1 -r
    echo
    if [[ $REPLY =~ ^[Yy]$ ]]; then
        stop_service $OLD_SERVICE
    fi
    
    echo ""
    echo "=========================================="
    echo -e "${GREEN}✅ 蓝绿部署完成！${NC}"
    echo "=========================================="
    echo ""
    echo "当前状态："
    echo -e "  活跃服务: ${GREEN}$TARGET_SERVICE${NC} (端口 $TARGET_PORT)"
    echo "  访问地址: http://localhost:$NGINX_PORT"
    echo ""
    echo "如需回滚："
    echo "  ./blue-green-rollback.sh"
}

main
```

**blue-green-rollback.sh（快速回滚）：**

```bash
#!/bin/bash

echo "========== 快速回滚 =========="

# 获取当前服务
CURRENT_UPSTREAM=$(grep "server localhost:" /usr/local/etc/nginx/nginx.conf | grep -o "[0-9]*")

# 切换到备份服务
if [ "$CURRENT_UPSTREAM" == "8081" ]; then
    NEW_PORT=8082
    NEW_COLOR="green"
else
    NEW_PORT=8081
    NEW_COLOR="blue"
fi

echo "回滚到 $NEW_COLOR 实例 (端口 $NEW_PORT)"

# 检查备份服务是否在运行
if ! curl -s http://localhost:$NEW_PORT/actuator/health > /dev/null; then
    echo "❌ 备份服务未运行，无法回滚"
    exit 1
fi

# 切换 Nginx
sed -i '' "s/server localhost:[0-9]*/server localhost:$NEW_PORT/" /usr/local/etc/nginx/nginx.conf
nginx -s reload

echo "✅ 已回滚到 $NEW_COLOR 实例"
```

**自动化部署脚本：**

```bash
#!/bin/bash
# deploy-to-test-server.sh

SERVER_IP="your-server-ip"
SERVER_USER="ubuntu"

echo "========== 部署到测试服务器 =========="

# 1. 本地构建
echo "📦 本地构建..."
cd backend
mvn clean package -DskipTests
cd ..

# 2. 上传到服务器
echo "📤 上传 JAR 包..."
scp backend/target/metersphere.jar $SERVER_USER@$SERVER_IP:/tmp/

# 3. 远程部署
echo "🚀 远程部署..."
ssh $SERVER_USER@$SERVER_IP << 'EOF'
    # 停止旧服务
    if [ -f /opt/metersphere/app.pid ]; then
        kill $(cat /opt/metersphere/app.pid)
        rm /opt/metersphere/app.pid
    fi
    
    # 备份旧版本
    if [ -f /opt/metersphere/metersphere.jar ]; then
        mv /opt/metersphere/metersphere.jar /opt/metersphere/metersphere.jar.backup
    fi
    
    # 部署新版本
    mv /tmp/metersphere.jar /opt/metersphere/
    
    # 启动新服务
    cd /opt/metersphere
    nohup java -jar metersphere.jar > logs/app.log 2>&1 &
    echo $! > app.pid
    
    # 健康检查
    sleep 10
    if curl -s http://localhost:8080/actuator/health | grep -q "UP"; then
        echo "✅ 部署成功"
    else
        echo "❌ 部署失败，回滚..."
        kill $(cat app.pid)
        mv metersphere.jar.backup metersphere.jar
        nohup java -jar metersphere.jar > logs/app.log 2>&1 &
    fi
EOF

echo "✅ 部署完成"
echo "测试环境地址: http://$SERVER_IP:8080"
```

```

    - 对外暴露
```

## 方案六：Feature Toggle（动态开关）

### 核心思路

**不重启应用，通过配置开关动态启用/禁用功能：**

```
测试人员访问 → 使用稳定功能（旧代码）
开发人员测试 → 启用新功能（新代码）

同一个服务实例，不同的代码路径
```

### 实施步骤

#### 步骤一：引入 Feature Toggle 库

**pom.xml：**

```xml
<dependency>
    <groupId>org.ff4j</groupId>
    <artifactId>ff4j-spring-boot-starter</artifactId>
    <version>1.8.12</version>
</dependency>
```

#### 步骤二：配置开关

**application.yml：**

```yaml
ff4j:
  autocreate: true
  audit: true
```

**FF4jConfiguration.java：**

```java
@Configuration
public class FF4jConfiguration {
    
    @Bean
    public FF4j ff4j() {
        FF4j ff4j = new FF4j();
        
        // 定义功能开关
        ff4j.createFeature("new-ui", false);  // 新 UI，默认关闭
        ff4j.createFeature("new-api", false);  // 新 API，默认关闭
        
        return ff4j;
    }
}
```

#### 步骤三：在代码中使用开关

**示例 1：控制器层：**

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @Autowired
    private FF4j ff4j;
    
    @GetMapping
    public List<User> getUsers() {
        // 根据开关选择不同的代码路径
        if (ff4j.check("new-api")) {
            // 新版本 API（正在开发）
            return userService.getUsersV2();
        } else {
            // 旧版本 API（稳定）
            return userService.getUsersV1();
        }
    }
}
```

**示例 2：服务层：**

```java
@Service
public class UserService {
    
    @Autowired
    private FF4j ff4j;
    
    public List<User> getUsers() {
        if (ff4j.check("use-cache")) {
            // 使用缓存（新功能）
            return getUsersFromCache();
        } else {
            // 直接查数据库（稳定）
            return getUsersFromDB();
        }
    }
}
```

#### 步骤四：动态控制开关

**通过 HTTP API 控制：**

```bash
# 启用新功能（开发人员测试用）
curl -X POST http://localhost:8080/ff4j/ff4j-web-console/features/new-api/enable

# 禁用新功能（测试人员用稳定版本）
curl -X POST http://localhost:8080/ff4j/ff4j-web-console/features/new-api/disable

# 查看所有开关状态
curl http://localhost:8080/ff4j/ff4j-web-console/features
```

**通过管理界面：**

```
访问: http://localhost:8080/ff4j-web-console

可以：
- 查看所有功能开关
- 启用/禁用功能
- 查看使用统计
- 设置按用户/角色启用
```

#### 步骤五：按用户启用功能

**只对开发人员启用新功能：**

```java
@RestController
public class UserController {
    
    @Autowired
    private FF4j ff4j;
    
    @GetMapping("/api/users")
    public List<User> getUsers(Authentication auth) {
        String username = auth.getName();
        
        // 根据用户角色决定是否启用新功能
        if (ff4j.check("new-api") && isDeveloper(username)) {
            return userService.getUsersV2();  // 开发人员看到新功能
        } else {
            return userService.getUsersV1();  // 其他人看到稳定版本
        }
    }
    
    private boolean isDeveloper(String username) {
        return username.startsWith("dev_");
    }
}
```

### 使用流程

```bash
# 1. 开发新功能（代码中用开关控制）
# 代码合并到主分支，但功能默认关闭

# 2. 测试人员测试（功能关闭，使用稳定代码）
curl http://localhost:8080/api/users
# → 返回旧版本结果

# 3. 开发人员测试新功能（临时启用）
curl -X POST http://localhost:8080/ff4j/features/new-api/enable
curl http://localhost:8080/api/users
# → 返回新版本结果

# 4. 测试完成后，再次关闭
curl -X POST http://localhost:8080/ff4j/features/new-api/disable

# 5. 新功能稳定后，永久启用
# 或删除开关，直接使用新代码
```

| ✅ 按用户/角色启用 | ❌ 不适合大改动 |
| ✅ 真正的零停机 |  |
