·# MeterSphere 项目结构分析

## 一、项目基本信息

### 项目名称
**MeterSphere** - 一站式开源持续测试平台

### 项目类型
- **项目性质**：开源企业级测试平台
- **架构模式**：微服务架构（Spring Cloud）
- **开发模式**：前后端分离
- **部署方式**：容器化部署（Docker/Kubernetes）
- **构建工具**：Maven（后端）+ npm（前端）
- **当前版本**：v2.10-lts

### 项目定位
一站式开源持续测试平台，涵盖测试跟踪、接口测试、UI测试和性能测试等功能，全面兼容 JMeter、Selenium 等主流开源标准。

## 四、核心模块说明

### 4.1 业务模块（7个）

#### 1. api-test（接口测试模块）
- **端口**：8004
- **功能**：API管理、Mock服务、场景编排、多协议支持
- **特点**：
  - 支持HTTP、HTTPS、WebSocket、Dubbo、TCP等多种协议
  - 支持Postman、Swagger等格式导入
  - 场景化测试编排
  - Mock服务支持
- **主要包结构**：
  - `controller/` - API控制器
  - `service/` - 业务服务层
  - `dto/` - 数据传输对象
  - `parse/` - 解析器（Postman/Swagger/HAR等）
  - `jmeter/` - JMeter执行相关
  - `base/mapper/` - MyBatis数据访问层

#### 2. performance-test（性能测试模块）
- **端口**：8005
- **功能**：性能压测、分布式测试、监控与报告
- **特点**：
  - 兼容JMeter脚本
  - 分布式压测支持
  - 实时监控与报告
  - 压测任务统一调度与管理
- **主要包结构**：
  - `controller/` - 性能测试控制器
  - `service/` - 性能测试服务
  - `dto/` - 性能测试DTO

#### 3. test-track（测试跟踪模块）
- **端口**：8003
- **功能**：测试用例管理、测试计划、测试执行、缺陷管理
- **特点**：
  - 对接主流项目管理平台
  - 测试过程全链路跟踪管理
  - 列表脑图模式自由切换
  - 用例编写简单、测试报告清晰
- **主要包结构**：
  - `controller/` - 测试跟踪控制器
  - `service/` - 测试跟踪服务
  - `dto/` - 测试跟踪DTO

#### 4. project-management（项目管理模块）
- **端口**：8002
- **功能**：项目创建与管理、项目成员管理、权限管理
- **特点**：
  - 多项目支持
  - 细粒度权限控制
  - 项目模板管理
- **主要包结构**：
  - `controller/` - 项目管理控制器
  - `service/` - 项目管理服务
  - `dto/` - 项目管理DTO

#### 5. system-setting（系统设置模块）
- **端口**：8001
- **功能**：系统参数配置、用户管理、工作空间管理、组织管理
- **特点**：
  - 系统级配置管理
  - 用户与权限管理
  - 工作空间隔离
- **主要包结构**：
  - `controller/` - 系统设置控制器

  - `dto/` - 系统设置DTO

#### 6. report-stat（报告统计模块）

- **功能**：测试报告生成、统计分析、企业级报告

### 4.2 框架层模块（3个）

#### 1. gateway（API网关）
- **技术**：Spring Cloud Gateway + WebFlux
- **功能**：
  - 统一入口，所有外部请求通过网关
  - 路由转发到各个微服务
  - 负载均衡

- **特点**：
  - 基于WebFlux响应式编程

  - 集成前端页面

#### 2. eureka（服务注册中心）
- **技术**：Spring Cloud Netflix Eureka
- **功能**：
  - 服务注册与发现

  - 负载均衡支持
- **特点**：
  - 服务间通信的基础设施
  - 支持服务高可用

  - 自动配置（`autoconfigure/`）
  - 安全框架集成（`security/`）

  - 通知服务（`notice/`）

##### 3.2 domain（领域模型）
- **功能**：
  - 数据模型定义
  - 数据库实体类
  - MyBatis映射文件
- **特点**：
  - 所有业务模块共享的数据模型
  - 统一的数据库表结构定义

##### 3.3 jmeter（JMeter核心）
- **功能**：
  - JMeter引擎封装
  - 测试脚本生成
  - 测试结果解析

- **特点**：

## 五、项目架构特点

### 5.1 微服务架构
- **服务拆分**：按业务领域拆分（DDD原则）
- **独立部署**：每个服务可独立部署和扩展
- **服务通信**：通过REST API和消息队列（Kafka）
- **服务发现**：通过Eureka实现服务注册与发现

### 5.2 前后端分离
- **独立开发**：前后端可并行开发
- **独立部署**：前后端可独立部署
- **技术栈**：前端Vue.js，后端Spring Boot

### 5.3 微前端架构
- **框架**：使用qiankun实现微前端
- **模块化**：各业务模块前端独立
- **按需加载**：提高页面加载速度

### 5.4 统一技术栈
- **后端**：统一使用Spring Boot 3.2.12 + Java 17
- **前端**：统一使用Vue.js 2.7.3
- **数据库**：统一使用MySQL 8.0.x
