# 页面顶部公告栏配置说明

## 功能说明

系统支持在页面顶部显示公告信息，用于向用户展示系统维护、升级等重要通知。

## 配置方式

### 方式一：直接操作数据库（推荐）

直接执行以下 SQL 语句在数据库中插入或更新系统参数：

```sql
-- 配置公告内容
INSERT INTO system_parameter (param_key, param_value, type, sort) 
    VALUES ('announcement.content', '系统将于今晚22:00进行维护升级', 'text', NULL)
ON DUPLICATE KEY UPDATE param_value = '系统将于今晚22:00进行维护升级';
```

**说明**：
- `announcement.content`：公告内容，可以包含任意文本
- 如果参数已存在，使用 `ON DUPLICATE KEY UPDATE` 会更新现有值
- 如果参数不存在，会创建新记录

**更新示例**：
```sql
-- 更新公告内容
UPDATE system_parameter 
SET param_value = '系统维护已完成，所有功能已恢复正常，统维护已完成，所有功能已恢复正常统维护已完成，所有功能已恢复正常统维护已完成，所有功能已恢复正常统维护已完成，所有功能已恢复正常统维护已完成，所有功能已恢复正常统维护已完成，所有功能已恢复正常统维护已完成，所有功能已恢复正常统维护已完成，所有功能已恢复正常统维护已完成，所有功能已恢复正常' 
WHERE param_key = 'announcement.content';
```

**删除配置**（隐藏公告栏）：
```sql
-- 删除公告配置
DELETE FROM system_parameter WHERE param_key = 'announcement.content';
```

### 方式二：通过 API 接口配置

#### 2.1 获取系统参数

**接口**：`GET /system/get/info/{key}`

**示例**：
```bash
# 获取上线时间
curl -X GET "http://your-domain/system/get/info/deployment.time" \
  -H "Cookie: your-session-cookie"

# 获取上线内容
curl -X GET "http://your-domain/system/get/info/deployment.content" \
  -H "Cookie: your-session-cookie"
```

#### 2.2 创建/更新系统参数

**接口**：`POST /system/edit/info`

**请求体**：
```json
{
  "paramKey": "announcement.content",
  "paramValue": "系统将于今晚22:00进行维护升级",
  "type": "text",
  "sort": null
}
```

**示例**：
```bash
# 配置公告内容
curl -X POST "http://your-domain/system/edit/info" \
  -H "Content-Type: application/json" \
  -H "Cookie: your-session-cookie" \
  -d '{
    "paramKey": "announcement.content",
    "paramValue": "系统将于今晚22:00进行维护升级",
    "type": "text"
  }'
```

**注意**：使用 API 接口需要：
1. 已登录系统（需要有效的 session cookie）
2. 具有 `SYSTEM_SETTING:READ+EDIT` 权限

## 显示效果

配置完成后，页面顶部会显示橙色横幅，从左到右显示公告文本：

```
系统将于今晚22:00进行维护升级
```

**显示规则**：
- 如果配置了公告内容，显示公告栏
- 如果未配置或内容为空，不显示公告栏
- 文本从左到右显示，超出部分会自动换行

## 参数说明

| 参数名 | 说明 | 示例值 | 是否必填 |
|--------|------|--------|----------|
| `announcement.content` | 公告内容 | `系统将于今晚22:00进行维护升级` | 否 |

## 注意事项

1. **权限要求**：使用 API 接口需要系统管理员权限
2. **缓存问题**：配置后可能需要刷新页面才能看到效果
3. **字符编码**：确保数据库和系统使用 UTF-8 编码，以支持中文内容
4. **内容长度**：建议上线内容不要过长，避免影响页面布局

## 故障排查

### 问题：配置后不显示

**可能原因**：
1. 参数名称拼写错误（应该是 `announcement.content`）
2. 数据库编码问题导致中文显示异常
3. 前端缓存问题

**解决方案**：
1. 检查数据库中的参数是否正确插入
   ```sql
   SELECT * FROM system_parameter WHERE param_key = 'announcement.content';
   ```
2. 清除浏览器缓存并刷新页面
3. 检查浏览器控制台是否有错误信息

### 问题：显示乱码

**解决方案**：
1. 确保数据库字符集为 `utf8mb4`
2. 确保连接数据库时使用 UTF-8 编码
3. 检查系统参数表的字符集设置
