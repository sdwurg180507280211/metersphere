# 全局模板和字段异常数据修复报告

## 重要说明

**经过数据库验证，发现以下事实**：

1. ✅ **项目模板是有用的**：测试项目和默认项目正在使用项目模板，共有8+202=210个缺陷
2. ✅ **项目模板绑定的都是全局字段**：所有项目模板都只绑定全局字段，符合最佳实践
3. ✅ **项目自定义字段是有用的**：默认项目有2个项目自定义字段（解决时间、关闭时间），共404条数据

**修复策略调整**：

- ❌ 删除项目系统字段（异常数据，与全局系统字段重复）
- ✅ 修复模板不存在的项目（改为使用全局默认模板）

## 注意事项

### 1. 数据迁移逻辑

- 使用 `JOIN` 方式迁移数据，确保只更新存在的记录
- 如果全局字段已有数据，会被项目字段的数据覆盖
- 同步 `issues` 表的 `status` 字段，确保数据一致性

### 2. 删除顺序

- 先删除数据（`custom_field_issues`）
- 再删除关联（`custom_field_template`）
- 最后删除定义（`custom_field`、`issue_template`）

### 3. 影响范围

- **测试项目**：3条缺陷数据受影响
- **默认项目**：202条缺陷数据受影响
- **其他项目**：无数据影响，只是删除了未使用的字段定义

### 4. 执行时间

- 预计执行时间：1-2分钟
- 主要耗时在默认项目的202条数据迁移

### 5. 业务影响

- **需要停止服务**：修复期间建议停止 MeterSphere 服务
- **用户影响**：修复期间用户无法访问系统
- **数据完整性**：修复后数据完整性不受影响
