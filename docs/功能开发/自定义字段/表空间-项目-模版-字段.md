# MeterSphere 核心数据模型 - 工作空间、项目、模板、字段关联关系

## 🏗️ 一、核心表结构概览

### 1.1 四大核心表

```
workspace (工作空间)
    ↓ 1:N
project (项目)
    ↓ 1:N
issue_template (缺陷模板) + case_template (用例模板) + api_template (接口模板)
    ↓ N:N (通过 custom_field_template)
custom_field (自定义字段)
```

### 1.2 表结构详解

#### 📋 workspace (工作空间表)
```java
public class Workspace {
    private String id;                  // 主键
    private String name;                // 工作空间名称
    private String description;         // 描述
    private Long createTime;
    private Long updateTime;
    private String createUser;
}
```

**用途**：组织级别的顶层容器，用于隔离不同组织/团队的数据

---

#### 📦 project (项目表)
```java
public class Project {
    private String id;                  // 主键
    private String workspaceId;         // ⭐ 外键：关联 workspace.id
    private String name;                // 项目名称
    private String platform;            // 缺陷平台：metersphere/Jira/Zentao/TAPD等
    
    // ⭐ 模板关联字段
    private String issueTemplateId;     // 缺陷模板ID → issue_template.id
    private String caseTemplateId;      // 用例模板ID → case_template.id
    private String apiTemplateId;       // 接口模板ID → api_template.id
    
    // 第三方平台字段
    private String jiraKey;
    private String tapdId;
    private String zentaoId;
    private String azureDevopsId;
    private Boolean thirdPartTemplate;  // 是否使用第三方平台模板
}
```

**核心关联**：
- `workspace_id` → `workspace.id`（多对一）
- `issue_template_id` → `issue_template.id`（多对一）

---

#### 📝 issue_template (缺陷模板表)
```java
public class IssueTemplate {
    private String id;                  // 主键
    private String name;                // 模板名称
    private String platform;            // 适用平台：metersphere/Jira/Zentao/TAPD
    private Boolean system;             // 是否为系统模板（不可删除）
    private Boolean global;             // ⭐ 是否为全局模板
    private String projectId;           // ⭐ 所属项目ID（global=0时有值）
    private String title;               // 模板标题
    private String content;             // 模板内容（富文本）
}
```

**层级关系**：
```
全局模板 (global=1, system=1, project_id=NULL)
├── Local 默认模板
├── Jira 默认模板
├── 禅道 默认模板
└── TAPD 默认模板

项目模板 (global=0, project_id=具体项目ID)
├── 项目A - Local模板
├── 项目B - Jira模板
└── 项目C - 自定义模板
```

---

#### 🔧 custom_field (自定义字段表)
```java
public class CustomField {
    private String id;                  // 主键
    private String name;                // 字段名称（如"需求号"）
    private String scene;               // 应用场景：ISSUE/TEST_CASE/API
    private String type;                // 字段类型：input/select/date等
    private Boolean system;             // ⭐ 是否为系统字段
    private Boolean global;             // ⭐ 是否为全局字段
    private String projectId;           // 所属项目ID（global=0时有值）
    private Boolean thirdPart;          // 是否为第三方平台字段
    private String options;             // 字段选项（JSON格式）

```

**层级关系**：
```
全局系统字段 (global=1, system=1, project_id=NULL)
├── 状态
├── 严重程度
└── 需求号（新增）

全局自定义字段 (global=1, system=0, project_id=NULL)
└── 工作空间级自定义字段

项目自定义字段 (global=0, project_id=具体项目ID)
└── 项目级自定义字段
```

---

#### 🔗 custom_field_template (字段-模板关联表)
```java
public class CustomFieldTemplate {
```

**作用**：多对多关系的中间表，定义**哪些字段在哪些模板上显示**

## 🔍 三、关键查询逻辑

### 3.1 获取项目的缺陷模板和字段

```sql
-- Step 1: 根据项目ID获取模板ID
SELECT issue_template_id 
FROM project 
WHERE id = 'proj-001';
-- 结果: 'tpl-001'

-- Step 2: 根据模板ID获取字段列表
SELECT cf.id, cf.name, cf.type, cft.order, cft.required
FROM custom_field_template cft
JOIN custom_field cf ON cft.field_id = cf.id
WHERE cft.template_id = 'tpl-001'  -- ⭐ 关键：基于模板ID查询
ORDER BY cft.order;
-- 结果: 
-- field-001 | 需求号     | input  | 2 | 0
-- field-002 | 严重程度   | select | 3 | 1
-- ...
```

### 3.2 Java 代码实现

```java
// IssueTemplateService.getTemplate(projectId)
public IssueTemplateDao getTemplate(String projectId) {
    // 1. 获取项目信息
    Project project = projectService.getProjectById(projectId);
    String issueTemplateId = project.getIssueTemplateId();
    
    // 2. 获取模板信息
    if (StringUtils.isNotBlank(issueTemplateId)) {
        issueTemplate = issueTemplateMapper.selectByPrimaryKey(issueTemplateId);
    } else {
        issueTemplate = getDefaultTemplate(projectId);  // 使用默认模板
    }
    
    // 3. ⭐ 核心：根据模板ID获取字段列表
    List<CustomFieldDao> fields = customFieldService
        .getCustomFieldByTemplateId(issueTemplate.getId());
    
    issueTemplateDao.setCustomFields(fields);
    return issueTemplateDao;
}
```

## 💡 五、核心设计理念

### 5.1 分层隔离

| 层级 | global | project_id | 作用域 | 说明 |
|------|--------|------------|--------|------|
| **全局** | 1 | NULL | 整个系统 | 所有工作空间和项目共享 |
| **项目** | 0 | 具体ID | 单个项目 | 项目内独立配置 |

### 5.2 为什么需要 custom_field_template？

#### ❌ 如果没有中间表

```java
// 问题：无法控制字段的显示顺序、是否必填
custom_field → issue_template  // 直接关联，缺少配置信息
```

#### ✅ 有中间表的优势

```java
custom_field_template {
    field_id: '需求号',
    template_id: 'Local模板',
    order: 2,           // ⭐ 可以控制显示顺序
    required: 0,        // ⭐ 可以设置是否必填
    default_value: ''   // ⭐ 可以设置默认值
}
```

**好处**：
1. 同一个字段在不同模板中可以有不同配置
2. 灵活控制字段的显示顺序
3. 不同模板可以选择性显示字段

### 核心关联链路

```
workspace.id 
    ← project.workspace_id
        ← project.issue_template_id 
            → issue_template.id
                ← custom_field_template.template_id
```

### 关键点

4. **V143 脚本的重要性**：批量关联所有项目模板，确保100%覆盖
