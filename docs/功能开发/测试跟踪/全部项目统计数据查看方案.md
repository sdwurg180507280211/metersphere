# 全部项目统计数据查看方案

## 一、需求背景

### 1.1 业务需求
领导需要查看全部项目的测试跟踪数据统计，而不是单个项目的数据。当前系统只支持查看单个项目的统计数据，需要在项目选择器中添加"全部项目"选项。

### 1.2 使用场景
- **场景1**：领导查看整个工作空间的测试覆盖率
- **场景2**：质量经理对比多个项目的测试进度
- **场景3**：生成跨项目的综合测试报告
- **场景4**：分析整体测试资源分配情况

### 1.3 当前限制
- 项目选择器只能选择单个项目
- 统计数据API只接受单个projectId参数
- 报告统计模块按项目维度隔离

## 二、方案设计

### 2.1 方案概述

在左上角的项目选择器中添加"全部项目"选项，选择后：
- 前端传递特殊标识（如 `projectId = "all"` 或 `projectId = null`）
- 后端识别该标识，查询当前工作空间下所有有权限的项目数据
- 统计数据按项目分组展示，同时提供汇总数据

**文件1：Controller层**

```java
// test-track/backend/src/main/java/io/metersphere/controller/TestCaseController.java

@RestController
@RequestMapping("/test/case")
public class TestCaseController {
    
    @Autowired
    private TestCaseService testCaseService;
    
    /**
     * 查询全部项目的测试用例分析数据
     * 
     * @param request 包含workspaceId和筛选条件
     * @return 汇总数据 + 各项目数据
     */
    @PostMapping("/analysis/all")
    @RequiresPermissions("PROJECT_TRACK_CASE:READ")
    public AllProjectsAnalysisDTO getAllProjectsAnalysis(@RequestBody AnalysisRequest request) {
        // 1. 获取当前用户在该工作空间下有权限的所有项目
        String workspaceId = request.getWorkspaceId();
        String userId = SessionUtils.getUserId();
        List<Project> projects = testCaseService.getUserProjectsByWorkspace(userId, workspaceId);
        
        // 2. 查询每个项目的统计数据
        AllProjectsAnalysisDTO result = new AllProjectsAnalysisDTO();
        List<ProjectAnalysisDTO> projectDataList = new ArrayList<>();
        
        // 汇总数据初始化
        AnalysisSummaryDTO summary = new AnalysisSummaryDTO();
        int totalCount = 0;
        int passCount = 0;
        
        for (Project project : projects) {
            // 查询单个项目数据
            request.setProjectId(project.getId());
            ProjectAnalysisDTO projectData = testCaseService.getProjectAnalysis(request);
            projectData.setProjectId(project.getId());
            projectData.setProjectName(project.getName());
            projectDataList.add(projectData);
            
            // 累加到汇总数据
            totalCount += projectData.getTotalCount();
            passCount += projectData.getPassCount();
        }
        
        // 3. 计算汇总数据
        summary.setTotalCount(totalCount);
        summary.setPassCount(passCount);
        summary.setPassRate(totalCount > 0 ? (double) passCount / totalCount * 100 : 0);
        
        result.setSummary(summary);
        result.setProjects(projectDataList);
        
        return result;
    }
}
```

```java
    public List<Project> getUserProjectsByWorkspace(String userId, String workspaceId) {

        UserGroupExample example = new UserGroupExample();
        example.createCriteria()
```

#### 2.3.3 权限控制

**原则**：
- 只能查看有权限的项目数据
- 工作空间管理员可以查看该工作空间下所有项目
- 普通用户只能查看自己参与的项目

**实现**：
```java
// 在Service层过滤项目列表
public List<Project> getUserProjectsByWorkspace(String userId, String workspaceId) {
    // 1. 判断是否为工作空间管理员
    if (isWorkspaceAdmin(userId, workspaceId)) {
        // 返回该工作空间下所有项目
        return getAllProjectsByWorkspace(workspaceId);
    }
    
    // 2. 普通用户：只返回有权限的项目
    return getAuthorizedProjects(userId, workspaceId);
}
```

## 四、注意事项

### 4.1 兼容性

**问题**：现有代码中很多地方使用 `getCurrentProjectID()` 获取项目ID

**解决方案**：
1. 在关键位置添加判断：
```javascript
const projectId = getCurrentProjectID();
if (projectId === 'all') {
  // 全部项目逻辑
} else {
  // 单个项目逻辑（原有逻辑）
}
```

2. 创建工具方法：
```javascript
// utils/project.js
export function isAllProjects() {
  return getCurrentProjectID() === 'all';
}

export function isSingleProject() {
  return getCurrentProjectID() !== 'all';
}
```

### 4.2 路由处理

**问题**：某些页面路由包含 `projectId` 参数

**解决方案**：
- 统计页面路由不包含 `projectId`，使用 `getCurrentProjectID()` 获取
- 如果路由包含 `projectId`，需要特殊处理 `projectId = 'all'` 的情况

### 4.3 权限控制

**问题**：不同用户看到的项目列表不同

**解决方案**：
- 后端严格按权限过滤项目列表
- 前端不做权限判断，完全依赖后端返回的数据

### 4.4 性能问题

**问题**：项目数量多时，查询性能可能下降

**解决方案**：
1. 并行查询多个项目数据
2. 使用Redis缓存统计结果
3. 前端分页加载
4. 设置项目数量上限（如最多显示50个项目）

### 4.5 数据一致性

**问题**：不同项目的统计维度可能不一致

**解决方案**：
- 统一统计维度和计算方法
- 对于缺失的数据，使用默认值或标记为"无数据"

## 五、替代方案

### 方案B：多项目选择器

**描述**：支持同时选择多个项目（而不是"全部项目"）

**优点**：
- 更灵活，可以选择部分项目对比
- 避免一次性加载过多数据

**缺点**：
- UI交互更复杂
- 实现成本更高

```vue
    :value="item.id">
  </el-option>

```

### 方案C：独立的跨项目统计页面

**描述**：不修改项目选择器，而是创建一个独立的"跨项目统计"菜单

**优点**：
- 不影响现有功能
- 可以设计专门的跨项目统计UI

**缺点**：
- 增加菜单层级
- 用户需要切换页面

**实现**：
