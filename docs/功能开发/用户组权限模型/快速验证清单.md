## 数据库验证

### 1. 检查用户组数据

```sql
-- 查看所有用户组
SELECT * FROM `group`;

-- 查看用户组成员
SELECT ug.*, g.name as group_name, u.name as user_name
FROM user_group ug
LEFT JOIN `group` g ON ug.group_id = g.id
LEFT JOIN `user` u ON ug.user_id = u.id
WHERE ug.source_id = 'your_project_id';
```

### 2. 模拟 getUserGroupInProject 查询

```sql
-- 查询用户在项目中的用户组
SELECT g.id
FROM user_group ug
         INNER JOIN `group` g ON ug.group_id = g.id
WHERE ug.user_id = 'your_user_id'
  AND ug.source_id = 'your_project_id'
LIMIT 1;
```

### 3. 检查缺陷数据

```sql
-- 查看缺陷列表
SELECT id, num, title, creator, platform_status
FROM issues
WHERE project_id = 'your_project_id'
ORDER BY create_time DESC
LIMIT 10;

-- 查看缺陷自定义字段（处理人）
SELECT ic.issues_id, ic.field_id, ic.value, cf.name
FROM issue_custom_field ic
LEFT JOIN custom_field cf ON ic.field_id = cf.id
WHERE cf.name = '处理人'
  AND ic.issues_id IN (SELECT id FROM issues WHERE project_id = 'your_project_id');
```

## 常见问题排查

### 问题1：404 Not Found

**症状**：`GET /issues/user/group/{projectId}/{userId}` 返回 404

**排查步骤**：
1. 检查后端服务是否启动
2. 检查 `IssuesController.java` 中的接口路径是否正确
3. 检查前端API调用的URL是否正确

### 问题2：用户组查询返回 null

**症状**：接口返回成功，但数据为 null

**排查步骤**：
1. 检查数据库中是否有用户组数据
2. 检查用户是否已分配到项目
3. 检查 SQL 查询是否正确

### 问题3：权限过滤不生效

**症状**：开发人员或测试人员仍能看到所有缺陷

**排查步骤**：
1. 检查 `applyUserGroupFilter()` 方法是否被调用
2. 检查 `page.condition.filters` 是否正确设置
3. 检查浏览器控制台是否有错误
4. 检查 Network 请求中的 filters 参数

### 问题4：搜索后仍有权限过滤

**症状**：点击搜索后仍只能看到自己的缺陷

**排查步骤**：
1. 检查 `clearUserGroupFilter()` 方法是否被调用
2. 检查 `search()` 方法中是否调用了 `clearUserGroupFilter()`
3. 检查 `userGroupFilterKeys` 是否被正确清空

### 问题5：翻页后权限过滤失效

**症状**：翻页后能看到所有缺陷

**排查步骤**：
1. 检查 `handlePageChange()` 方法是否设置了 `pageRefresh = true`
2. 检查 MsTable 组件是否正确处理 `pageRefresh` 属性
3. 检查 `page.condition.filters` 是否在翻页时被清空
