# 页面操作手册 - 帮助提示方案设计

## 需求背景

为了帮助测试人员和开发人员快速上手MeterSphere系统，需要在每个页面添加操作手册提示，让用户能够快速了解当前页面的功能和操作方法。

## 方案对比

### 方案1：固定位置帮助图标（推荐）⭐

**位置**：页面标题栏右侧，搜索框旁边

**优点**：
- ✅ 位置固定，用户容易找到
- ✅ 不遮挡页面内容
- ✅ 实现简单，维护成本低
- ✅ 系统已有现成组件（`MsInstructionsIcon`）
- ✅ 符合用户习惯（类似"?"帮助按钮）

**缺点**：
- ❌ 不够醒目，新用户可能忽略

**适用场景**：
- 所有页面的常规帮助
- 简短的操作说明
- 快速参考文档

---

### 方案2：悬浮帮助按钮

**位置**：页面右下角或右侧悬浮

**优点**：
- ✅ 醒目，用户容易注意到
- ✅ 不占用页面布局空间
- ✅ 可以展开更丰富的内容

**缺点**：
- ❌ 可能遮挡页面内容
- ❌ 实现复杂度较高
- ❌ 移动端适配困难

**适用场景**：
- 需要强调的重要功能
- 新功能上线引导
- 复杂页面的详细说明

---

### 方案3：新手引导（Shepherd.js）

**位置**：覆盖在页面元素上，分步引导

**优点**：
- ✅ 交互性强，引导效果好
- ✅ 适合复杂流程的分步讲解
- ✅ 系统已集成（`VueShepherd`）

**缺点**：
- ❌ 只适合首次使用
- ❌ 配置复杂，维护成本高
- ❌ 用户可能跳过

**适用场景**：
- 新用户首次登录
- 复杂功能的分步教学
- 新功能上线推广

---

### 方案4：页面顶部提示条

**位置**：页面顶部，标题下方

**优点**：
- ✅ 醒目，用户一定能看到
- ✅ 可以展示较多内容

**缺点**：
- ❌ 占用页面空间
- ❌ 影响页面美观
- ❌ 用户可能觉得烦躁

**适用场景**：
- 重要通知
- 临时性提示
- 不推荐作为常规帮助方式

---

## 推荐方案：组合使用

### 核心方案：固定位置帮助图标（方案1）

**实现方式**：使用现有的 `MsInstructionsIcon` 组件

**位置建议**：
1. **页面标题栏右侧**（最推荐）
   ```
   [页面标题]                    [搜索框] [高级搜索] [帮助?]
   ```

2. **搜索框右侧**
   ```
   [页面标题]    [搜索框] [帮助?] [高级搜索] [其他按钮]
   ```

3. **页面右上角**
   ```
   [页面标题]                                    [帮助?]
   ```

### 辅助方案：悬浮帮助中心（可选）

对于复杂页面或需要详细说明的场景，可以添加悬浮帮助按钮。

### 2. 悬浮帮助按钮（可选）

#### 全局组件

创建 `MsPageHelpButton.vue`：

```vue
<template>
  <div class="page-help-button" v-if="visible">
    <!-- 帮助按钮 -->
    <el-tooltip content="页面帮助" placement="left">
      <div class="help-btn" @click="showHelp">
        <i class="el-icon-question"></i>
      </div>
    </el-tooltip>
    
    <!-- 帮助弹窗 -->
    <el-dialog
      title="页面操作手册"
      :visible.sync="dialogVisible"
      width="600px"
      :append-to-body="true"
    >
      <div class="help-content" v-html="content"></div>
      <span slot="footer">
        <el-button @click="dialogVisible = false">关闭</el-button>
        <el-button type="primary" @click="openFullDoc">查看完整文档</el-button>
      </span>
    </el-dialog>
  </div>
</template>

<script>
export default {
  name: 'MsPageHelpButton',
  props: {
    content: {
      type: String,
      required: true
    },
    docUrl: {
      type: String,
      default: ''
    },
    visible: {
      type: Boolean,
      default: true
    }
  },
  data() {
    return {
      dialogVisible: false
    };
  },
  methods: {
    showHelp() {
      this.dialogVisible = true;
    },
    openFullDoc() {
      if (this.docUrl) {
        window.open(this.docUrl, '_blank');
      }
    }
  }
};
</script>
```

### 3. 新手引导（Shepherd.js）

#### 使用场景

- 用户首次登录
- 新功能上线
- 复杂流程引导

#### 实现示例

```vue
<script>
export default {
  mounted() {
    // 检查是否首次访问
    if (this.isFirstVisit()) {
      this.startTour();
    }
  },
  methods: {
    isFirstVisit() {
      const key = 'test-case-tour-completed';
      return !localStorage.getItem(key);
    },
    startTour() {
      const tour = this.$shepherd({
        useModalOverlay: true,
        defaultStepOptions: {
          cancelIcon: {
            enabled: true
          },
          classes: 'shepherd-theme-custom',
          scrollTo: { behavior: 'smooth', block: 'center' }
        }
      });

      tour.addStep({
        id: 'welcome',
        text: '欢迎使用测试用例管理！让我们快速了解一下主要功能。',
        buttons: [
          {
            text: '跳过',
            action: tour.cancel
          },
          {
            text: '开始',
            action: tour.next
          }
        ]
      });

      tour.addStep({
        id: 'create-case',
        text: '点击这里可以创建新的测试用例',
        attachTo: {
          element: '.create-case-btn',
          on: 'bottom'
        },
        buttons: [
          {
            text: '上一步',
            action: tour.back
          },
          {
            text: '下一步',
            action: tour.next
          }
        ]
      });

      tour.addStep({
        id: 'search',
        text: '使用搜索框快速查找用例，按Tab键可以打开高级搜索',
        attachTo: {
          element: '.search-box',
          on: 'bottom'
        },
        buttons: [
          {
            text: '上一步',
            action: tour.back
          },
          {
            text: '完成',
            action: () => {
              localStorage.setItem('test-case-tour-completed', 'true');
              tour.complete();
            }
          }
        ]
      });

      tour.start();
    }
  }
};
</script>
```

### 方案C：后端接口获取

**优点**：
- 动态更新，无需发版
- 支持权限控制
- 可以统计查看次数

**实现**：

```javascript
export default {
  data() {
    return {
      pageHelp: ''
    };
  },
  mounted() {
    this.loadPageHelp();
  },
  methods: {
    async loadPageHelp() {
      try {
        const res = await this.$get(`/help/page/${this.$route.name}`);
        this.pageHelp = res.data.content;
      } catch (e) {
        console.error('加载帮助内容失败', e);
      }
    }
  }
};
```

## 最终建议

### 推荐组合

1. **所有页面**：使用固定位置帮助图标（方案1）
   - 位置：页面标题栏右侧
   - 组件：`MsInstructionsIcon`
   - 内容：简短的操作说明

2. **复杂页面**：额外添加悬浮帮助按钮（方案2）
   - 位置：页面右下角
   - 组件：`MsPageHelpButton`
   - 内容：详细的操作手册

3. **新用户**：首次登录时启动新手引导（方案3）
   - 工具：Shepherd.js
   - 场景：首次访问核心页面
   - 内容：分步引导

### 实施优先级

1. **高优先级**：固定位置帮助图标（立即实施）
2. **中优先级**：悬浮帮助按钮（按需实施）
3. **低优先级**：新手引导（后期优化）
