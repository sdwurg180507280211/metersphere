# 复测次数强制初始化为0实现方案

## 需求

无论通过什么手段新增缺陷（UI、API、导入），复测次数初始值都必须强制为0，即使用户指定了非0值也要重置。

## 实现位置

需要在以下3个入口强制初始化：

1. **UI/API新增** - `IssuesService.addIssues()`
2. **Excel导入** - `IssueExcelListener.buildFields()`
3. **第三方平台同步** - `AbstractIssuePlatform.addIssue()`（可选）

---

## 代码实现

### 1. IssuesService - 新增缺陷时强制初始化

**文件**：`test-track/backend/src/main/java/io/metersphere/service/IssuesService.java`

**位置**：在 `addIssues()` 方法中，保存自定义字段之前

```java
public IssuesWithBLOBs addIssues(IssuesUpdateRequest issuesRequest, List<MultipartFile> files) {
    Project project = baseProjectService.getProjectById(issuesRequest.getProjectId());
    IssuesWithBLOBs issues = null;
    
    // ... 现有的平台处理逻辑（插件平台或本地平台）
    
    String issuesId = issues.getId();
    saveFollows(issuesId, issuesRequest.getFollows());
    
    // ✅ 新增：在保存自定义字段前，强制确保复测次数为0
    ensureRetestCountIsZero(issuesRequest);
    
    customFieldIssuesService.addFields(issuesId, issuesRequest.getAddFields());
    customFieldIssuesService.editFields(issuesId, issuesRequest.getEditFields());
    
    // ... 后续的附件处理逻辑
    
    return getIssue(issues.getId());
}

/**
 * 确保复测次数字段存在且强制为0
 * 
 * 目的：
 * 1. 防止通过API传入非0值
 * 2. 确保所有新增缺陷的复测次数都是0
 * 3. 只有通过状态流转 closed -> reopened 才能增加
 * 
 * @param request 缺陷请求对象
 */
private void ensureRetestCountIsZero(IssuesUpdateRequest request) {
    try {
        // 获取复测次数字段定义
        CustomField retestCountField = baseCustomFieldService.getCustomFieldByName(
            request.getProjectId(), SystemCustomField.ISSUE_RETEST_COUNT);
        
        if (retestCountField == null) {
            // 字段不存在，跳过（可能是旧项目或未配置）
            return;
        }
        
        String fieldId = retestCountField.getId();
        
        // 检查 addFields 中是否已有复测次数
        Optional<CustomFieldResourceDTO> existing = request.getAddFields().stream()
            .filter(f -> f.getFieldId().equals(fieldId))
            .findFirst();
        
        if (existing.isPresent()) {
            // 存在则强制重置为0（防止API传入非0值）
            existing.get().setValue("0");
            LogUtil.info("强制重置复测次数为0: issueId={}", request.getId());
        } else {
            // 不存在则添加，初始值为0
            CustomFieldResourceDTO resource = new CustomFieldResourceDTO();
            resource.setFieldId(fieldId);
            resource.setValue("0");
            request.getAddFields().add(resource);
            LogUtil.info("添加复测次数字段，初始值为0: issueId={}", request.getId());
        }
        
        // 同步更新 requestFields（用于第三方平台）
        Optional<CustomFieldItemDTO> existingItem = request.getRequestFields().stream()
            .filter(f -> f.getId().equals(fieldId))
            .findFirst();
        
        if (existingItem.isPresent()) {
            existingItem.get().setValue(0);
        } else {
            CustomFieldItemDTO itemDTO = new CustomFieldItemDTO();
            itemDTO.setId(fieldId);
            itemDTO.setName(SystemCustomField.ISSUE_RETEST_COUNT);
            itemDTO.setType(CustomFieldType.INT.getValue());
            itemDTO.setValue(0);
            request.getRequestFields().add(itemDTO);
        }
        
    } catch (Exception e) {
        // 不阻断缺陷创建流程，只记录日志
        LogUtil.error("强制初始化复测次数失败: " + e.getMessage(), e);
    }
}
```

### 2. IssueExcelListener - 导入时强制忽略用户输入

**文件**：`test-track/backend/src/main/java/io/metersphere/excel/listener/IssueExcelListener.java`

**位置**：在 `buildFields()` 方法中，遍历模版字段时特殊处理

```java
/**
 * 构建自定义字段
 * 
 * 修改点：
 * 1. 遍历模版的所有字段（而不是只遍历Excel中的字段）
 * 2. 对复测次数字段特殊处理：强制为0，完全忽略Excel输入
 */
private void buildFields(IssueExcelData issueExcelData, IssuesUpdateRequest issuesUpdateRequest) {
    Boolean addFlag = issueExcelData.getAddFlag();
    List<CustomFieldResourceDTO> addFields = new ArrayList<>();
    List<CustomFieldResourceDTO> editFields = new ArrayList<>();
    List<CustomFieldItemDTO> requestFields = new ArrayList<>();
    
    // 将customFields转为Map，key=字段名称
    Map<String, List<CustomFieldDao>> customFieldMap = customFields.stream()
        .collect(Collectors.groupingBy(CustomFieldDao::getName));
    
    // ✅ 修改1：遍历模版的所有字段（确保复测次数字段被处理）
    for (CustomFieldDao customFieldDao : customFields) {
        try {
            String fieldName = customFieldDao.getName();
            String type = customFieldDao.getType();
            String fieldId = customFieldDao.getId();
            
            // ✅ 修改2：复测次数特殊处理 - 强制为0，完全忽略Excel输入
            if (StringUtils.equals(fieldName, SystemCustomField.ISSUE_RETEST_COUNT)) {
                CustomFieldResourceDTO resource = new CustomFieldResourceDTO();
                resource.setFieldId(fieldId);
                resource.setResourceId(issuesUpdateRequest.getId());
                resource.setValue("0");  // 强制为0
                
                CustomFieldItemDTO itemDTO = new CustomFieldItemDTO();
                BeanUtils.copyBean(itemDTO, customFieldDao);
                itemDTO.setValue(0);
                
                if (addFlag) {
                    addFields.add(resource);
                } else {
                    // 更新时也强制为0（防止覆盖模式修改）
                    editFields.add(resource);
                }
                requestFields.add(itemDTO);
                
                LogUtil.info("导入缺陷，复测次数强制为0: {}", issuesUpdateRequest.getTitle());
                continue;  // ✅ 跳过，不从Excel读取该字段
            }
            
            // 获取Excel中该字段的值
            Object excelValue = null;
            if (MapUtils.isNotEmpty(issueExcelData.getCustomData())) {
                excelValue = issueExcelData.getCustomData().get(fieldName);
            }
            
            // 如果Excel中没有该字段，检查是否有默认值
            if (excelValue == null || StringUtils.isEmpty(excelValue.toString())) {
                String defaultValue = customFieldDao.getDefaultValue();
                if (StringUtils.isNotEmpty(defaultValue)) {
                    // 使用默认值
                    excelValue = defaultValue;
                } else if (BooleanUtils.isTrue(customFieldDao.getRequired())) {
                    // 必填但没有默认值，设置为空（后续校验会报错）
                    excelValue = StringUtils.EMPTY;
                } else {
                    // 非必填且无默认值，跳过
                    continue;
                }
            }
            
            // 构建字段DTO（现有逻辑）
            CustomFieldResourceDTO customFieldResourceDTO = new CustomFieldResourceDTO();
            customFieldResourceDTO.setFieldId(fieldId);
            
            CustomFieldItemDTO customFieldItemDTO = new CustomFieldItemDTO();
            BeanUtils.copyBean(customFieldItemDTO, customFieldDao);
            
            // 根据字段类型处理值（现有逻辑，省略...）
            // ... 
            
            if (addFlag) {
                addFields.add(customFieldResourceDTO);
            } else {
                editFields.add(customFieldResourceDTO);
            }
            requestFields.add(customFieldItemDTO);
            
        } catch (Exception e) {
            LogUtil.error("处理字段失败: " + customFieldDao.getName(), e);
            MSException.throwException(e.getMessage());
        }
    }
    
    // 设置到请求对象
    if (addFlag) {
        issuesUpdateRequest.setAddFields(addFields);
    } else {
        issuesUpdateRequest.setEditFields(editFields);
    }
    issuesUpdateRequest.setRequestFields(requestFields);
}
```

### 3. 更新时禁止修改复测次数（可选，增强安全性）

**文件**：`test-track/backend/src/main/java/io/metersphere/service/IssuesService.java`

**位置**：在 `updateIssues()` 方法中

```java
public IssuesWithBLOBs updateIssues(IssuesUpdateRequest issuesRequest) {
    IssuesWithBLOBs before = issuesService.getIssue(issuesRequest.getId());
    
    // ... 现有的平台更新逻辑
    
    // ✅ 新增：更新时移除复测次数字段（防止用户通过API修改）
    removeRetestCountFromEditFields(issuesRequest);
    
    customFieldIssuesService.editFields(issuesRequest.getId(), issuesRequest.getEditFields());
    customFieldIssuesService.addFields(issuesRequest.getId(), issuesRequest.getAddFields());
    
    // ... 后续逻辑
    
    return after;
}

/**
 * 从编辑字段中移除复测次数
 * 
 * 目的：
 * 复测次数只能通过状态流转 closed -> reopened 自动增加
 * 不允许用户通过UI或API手动修改
 * 
 * @param request 缺陷更新请求
 */
private void removeRetestCountFromEditFields(IssuesUpdateRequest request) {
    try {
        CustomField retestCountField = baseCustomFieldService.getCustomFieldByName(
            request.getProjectId(), SystemCustomField.ISSUE_RETEST_COUNT);
        
        if (retestCountField != null) {
            String fieldId = retestCountField.getId();
            
            // 从 editFields 中移除复测次数
            int removedCount = 0;
            removedCount += request.getEditFields().removeIf(f -> f.getFieldId().equals(fieldId));
            removedCount += request.getAddFields().removeIf(f -> f.getFieldId().equals(fieldId));
            removedCount += request.getRequestFields().removeIf(f -> f.getId().equals(fieldId));
            
            if (removedCount > 0) {
                LogUtil.warn("更新缺陷时移除复测次数字段（不允许手动修改）: issueId={}", request.getId());
            }
        }
    } catch (Exception e) {
        LogUtil.error("移除复测次数字段失败: " + e.getMessage(), e);
    }
}
```

## 关键点总结

### 1. 为什么要在3个地方都处理？

| 入口 | 风险 | 处理方式 |
|------|------|----------|
| UI新增 | 用户可能通过浏览器调试工具修改请求 | `ensureRetestCountIsZero()` |
| Excel导入 | 用户可能在Excel中填写非0值 | `buildFields()` 中特殊处理 |
| API调用 | 恶意用户可能直接调用API传入非0值 | `ensureRetestCountIsZero()` |

### 2. 为什么不在数据库层面约束？

因为选择了"系统自定义字段"方案，数据存储在 `custom_field_issues` 表的 `value` 字段（TEXT类型），无法在数据库层面设置 `DEFAULT 0`。

### 3. 为什么要在更新时也移除？

防止用户通过"覆盖模式"导入或API调用修改复测次数。

### 4. 异常处理原则

所有强制初始化逻辑都使用 `try-catch`，确保即使处理失败也不会阻断缺陷的创建/更新流程。
