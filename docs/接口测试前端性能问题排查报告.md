# 接口测试前端性能问题排查报告

排查范围：`api-test/frontend/src/business/automation/` 目录
排查时间：2026-04-22

## 一、高严重度问题

### 1. scenarioDefinition 的 JSON.parse(JSON.stringify) 深拷贝

场景步骤树数据量可达数百步骤 + 嵌套子请求体，深拷贝开销极高。其中"运行环境"按钮的深拷贝已在本次优化中修复（用递归遍历替代），其余场景因业务需要确实需要拷贝，优化空间有限。

| 文件路径 | 行号 | 场景 | 状态 |
|----------|------|------|------|
| EditApiScenario.vue | 2540 | 点击"运行环境"按钮：深拷贝 + 删禁用步骤 + 序列化 + 正则提取 projectId | **已修复**：用 `_collectProjectIds` 递归遍历替代 |
| EditApiScenario.vue | 2090 | 保存场景时深拷贝 scenarioDefinition | 业务需要，暂无法替代 |
| EditApiScenario.vue | 2222 | 保存场景快照，同上 | 业务需要，暂无法替代 |
| ApiAutomation.vue | 504 | 版本对比时对 v2 深拷贝 | 业务需要，暂无法替代 |
| ApiAutomation.vue | 513 | 版本对比时连续两次深拷贝 v1 和 v3 | 可合并为一次，待优化 |
| MaximizeScenario.vue | 1260 | 点击"运行环境"按钮，同 EditApiScenario | **已修复** |
| MaximizeScenario.vue | 1027 | initProjectIds 用 jsonPath.query 提取 projectId | **已修复**：用 `_collectProjectIds` 替代 |

### 2. 搜索/筛选无防抖

整个 automation 目录没有使用 debounce/throttle，每次按键都触发完整操作。

| 文件路径 | 行号 | 问题描述 |
|----------|------|----------|
| ApiScenarioModule.vue | 157 | 树搜索框 `condition.filterText` 无防抖，每次按键触发 `this.$refs.nodeTree.filter()`，大树场景卡顿明显 |

### 3. 场景步骤树无虚拟滚动

| 文件路径 | 问题描述 |
|----------|----------|
| EditApiScenario.vue（template 区域） | `el-tree` 渲染 scenarioDefinition，100+ 步骤全部展开时 DOM 节点极多，无虚拟滚动 |
| MaximizeScenario.vue（template 区域） | 最大化场景编辑器的步骤树，同上 |

这是大场景卡顿的根本原因，但改造成本最高（需要换虚拟树组件）。

## 二、中严重度问题

### 1. ScenarioDiff setTimeout 延迟与子组件数成正比

| 文件路径 | 行号 | 问题描述 |
|----------|------|----------|
| ScenarioDiff.vue | 828 | `setTimeout(this.getDiff, (this.$refs.old.$children.length + 1) * 5000)` — 子组件多时等待可达数十秒 |

### 2. recursionDelete 中 for...in + splice

遍历中修改数组，可能导致跳过元素（逻辑 bug + 性能问题）。

| 文件路径 | 行号 |
|----------|------|
| EditApiScenario.vue | 2687 |
| MaximizeScenario.vue | 1105 |

### 3. ComponentConfig 大组件静态 import

7 个步骤组件为静态 import，未懒加载，影响首屏加载速度。

| 文件路径 | 行号 | 未懒加载组件 |
|----------|------|-------------|
| ComponentConfig.vue | 45-63 | MsIfController、MsTransactionController、MsApiComponent、MsLoopController、MsApiScenarioComponent、JmeterElementComponent、PluginComponent |

### 4. watch 配置不当

| 文件路径 | 行号 | 问题描述 |
|----------|------|----------|
| Jsr233ProcessorContent.vue | 166 | watch `jsr223Processor` 无 `deep: false`，复杂对象任何属性变化都触发 reload |
| MaximizeScenario.vue | 519 | watch `reloadDebug` 和 `debug` 无优化配置，每次变化触发 reload 强制重渲染 |

### 5. ScenarioChildDiff setInterval 无 beforeDestroy 兜底

| 文件路径 | 行号 | 问题描述 |
|----------|------|----------|
| ScenarioChildDiff.vue | 67 | `window.setInterval(this.getVnode, 1000)` 在 mounted 中启动，组件没有 `beforeDestroy` 钩子做兜底清理，极端情况下 interval 永远不清理，导致内存泄漏 |

### 6. PluginContentAssertions 直接操作 DOM

| 文件路径 | 行号 | 问题描述 |
|----------|------|----------|
| PluginContentAssertions.vue | 281 | `$nextTick` 内嵌 `setTimeout`，并用 `document.getElementsByName` 直接操作 DOM 计算高度 |

### 7. v-for 使用 index 做 key（有增删场景）

| 文件路径 | 行号 | 问题描述 |
|----------|------|----------|
| EnvSelectPopover.vue | 41, 59 | 环境列表和环境组列表用 index 做 key，应使用 id |
| ApiScenarioVariables.vue | 6 | 变量行可增删，index key 会导致复用错误 |
| PluginContentAssertions.vue | 44 | 断言列表可增删，index key 不安全 |
| ApiReportExport.vue | 10, 16 | 场景/请求结果列表 |

## 四、优化建议优先级

| 优先级 | 优化项 | 改动量 | 预期收益 |
|--------|--------|--------|----------|
| P0 | ~~showPopover 消除 JSON 序列化~~ | 已完成 | CPU 阻塞从秒级降至毫秒级 |
| P1 | 搜索框加防抖 | 小 | 大树搜索不再卡顿 |
| P1 | 核心递归函数 `for...in` → `for...of` | 中 | 消除叠加性能损失 |
| P2 | recursionDelete 中 splice 逻辑修复 | 小 | 修复潜在逻辑 bug |
| P2 | ComponentConfig 组件懒加载 | 小 | 首屏加载提速 |
| P2 | v-for key 改用 id | 小 | 避免增删场景下复用错误 |
| P3 | ScenarioChildDiff setInterval 兜底清理 | 小 | 防止内存泄漏 |
| P3 | hack 式 DOM 刷新优化 | 中 | 减少不必要的重渲染 |
| P4 | 场景步骤树虚拟滚动 | 大 | 大场景根本性提速 |
