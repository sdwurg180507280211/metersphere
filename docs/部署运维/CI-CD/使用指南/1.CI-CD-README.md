## 🚀 快速开始（30分钟）

### 步骤1：准备部署服务器（5分钟）

```bash
# 1. 创建目录
sudo mkdir -p /opt/metersphere
cd /opt/metersphere

# 2. 复制docker-compose文件
# （从WSL /opt/metersphere复制所有docker-compose-*.yml文件）

# 3. 复制脚本
mkdir -p scripts
# 复制本项目的scripts/*到这里

# 4. 添加执行权限
chmod +x scripts/*.sh

# 5. 创建配置文件
cp .env.example install.conf
vim install.conf  # 修改为实际配置
```

### 步骤2：配置Jenkins（15分钟）

```
详见：docs/Jenkins配置指南.md

关键步骤：
1. 安装必要插件
2. 配置SSH密钥（metersphere-deploy-key）
3. 配置企业微信Webhook
4. 配置服务器IP（test-server-ip、prod-server-ip）
5. 配置Maven settings.xml
6. 创建Pipeline任务
```

### 步骤3：测试部署（10分钟）

```bash
# 在部署服务器测试手动部署
cd /opt/metersphere
./scripts/start-all.sh

# 验证
./scripts/health-check.sh

# 访问
curl http://localhost:8081
```

### 步骤4：测试Jenkins自动部署

```bash
# 1. 提交代码到develop分支
git checkout develop
git commit -m "test: 测试CI/CD" --allow-empty
git push origin develop

# 2. 观察Jenkins
# Jenkins会自动：
# - 检测代码变化
# - 构建Maven项目
# - 构建Docker镜像
# - 推送到阿里云
# - 部署到测试环境
# - 发送企业微信通知

# 3. 验证部署
ssh deploy@test-server
cd /opt/metersphere
./scripts/health-check.sh
```

## 🔄 CI/CD完整流程

### 自动化流程（develop分支）

```
1. 开发提交代码到develop分支
   └── git push origin develop

2. Jenkins自动触发（pollSCM或Webhook）
   └── 检测到代码变化

3. Preparation阶段
   └── 版本号判断：develop

4. Build/Test阶段
   ├── Maven多模块构建
   ├── JAR解压为分层目录
   └── xpack包注入

5. Docker Build & Push阶段
   ├── buildx多架构构建（amd64+arm64）
   ├── 推送到阿里云镜像仓库
   └── 标签：develop

6. Deploy to Test阶段
   ├── SSH连接到测试服务器
   ├── 更新install.conf（MS_IMAGE_TAG=develop）
   ├── 执行deploy.sh test develop
   └── 健康检查验证

7. Notification
   └── 企业微信发送部署成功通知
```

### 手动发布流程（master分支）

```
1. 合并develop到master
   ├── git checkout master
   ├── git merge develop
   └── git push origin master

2. 打标签
   ├── git tag -a v2.10.24 -m "Release v2.10.24"
   └── git push origin --tags

3. Jenkins自动构建
   ├── 镜像标签：v2.10.24
   └── 停在Deploy to Production阶段

4. 人工审批
   ├── 进入Jenkins任务
   ├── 点击"继续"按钮
   └── 确认部署到生产

5. 自动部署到生产
   └── 执行deploy.sh prod v2.10.24

6. 通知
   └── 企业微信发送生产部署通知
```

```bash
./scripts/update-service.sh api-test v2.10.24
```

```bash
./scripts/rollback.sh api-test backup
```

```bash
./scripts/view-logs.sh gateway
```

### 重启服务

```bash
# 重启单个
docker-compose -f docker-compose-gateway.yml restart

# 重启所有
./scripts/stop-all.sh
```

## 🆘 故障排查

### 常见问题

**Q：Jenkins构建失败**
```
查看Console Output找到错误
常见原因：Maven依赖、Docker连接、网络问题
解决：检查settings.xml、Docker daemon状态
```

**Q：部署失败**
```
查看deploy.sh的输出
常见原因：镜像拉取失败、配置错误、依赖服务未启动
解决：./scripts/health-check.sh检查环境
```

**Q：服务启动超时**
```
查看服务日志：./scripts/view-logs.sh <服务名>
常见原因：依赖服务未就绪、配置错误、资源不足
解决：检查依赖顺序、增加内存限制
```
