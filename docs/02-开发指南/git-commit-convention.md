# Git 提交规范

## 格式

```
<type>(<scope>): <subject>
```

- **type**（必填）：提交类型
- **scope**（可选）：影响范围/模块
- **subject**（必填）：简短描述，中文

## Type 类型

| Type | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat(系统设置): 成员下拉展示成员ID` |
| `fix` | 修复缺陷 | `fix(接口测试): 修复场景自定义请求 URL 参数修改不生效问题` |
| `refactor` | 重构（不新增功能也不修复缺陷） | `refactor(接口测试): 场景编号增加索引` |
| `perf` | 性能优化 | `perf(接口测试): 引用关系图查询 SQL 优化` |
| `style` | 样式调整（不影响逻辑） | `style(接口测试): 编辑场景模块选项样式调整` |
| `test` | 测试相关 | `test: 添加断言日志` |
| `ci` | CI/CD 变更 | `ci: 关闭定时触发` |
| `build` | 构建相关 | `build: jsPDF 调整` |
| `chore` | 杂项（构建辅助工具等） | `chore(测试跟踪): 功能用例模块树优化-补充注释` |
| `docs` | 文档 | `docs: 删除过时文档文件` |

> 按本项目历史提交频率排序：fix(206) > refactor(42) > feat(17) > ci(10) > chore(9) > perf(5) > test/style/build(2) > docs(1)

不涉及特定模块时，scope 可省略：

```
fix: 修复微信通知阻塞问题
refactor: 使用 try-with-resources 确保流关闭
```

## Subject 描述规范

1. **中文描述**，简短精炼，不超过 50 字
2. **fix 类型**以动词开头：`修复...问题`、`修复...报错`
3. **feat 类型**描述新增能力：`成员下拉展示成员ID`、`接口新建更新删除增加责任人通知`
4. **refactor 类型**描述优化内容：`场景编号增加索引`、`文件异常信息处理`
5. **perf 类型**描述优化点：`引用关系图查询 SQL 优化`、`场景保存依赖关系处理优化`
6. 结尾不加句号

## 完整示例

```
feat(系统设置): 成员下拉展示成员ID
fix(接口测试): 修复场景自定义请求 URL 参数修改不生效问题
fix(测试跟踪): 功能用例列表查询时部分过滤字段未命中索引
refactor(接口测试): 场景编号增加索引
refactor: 使用 try-with-resources 确保流关闭
perf(接口测试): 引用关系图查询 SQL 优化
perf(系统设置): 资源池验证优化
style(接口测试): 编辑场景模块选项样式调整
chore(测试跟踪): 功能用例模块树优化-补充注释
ci: 关闭定时触发
build: jsPDF 调整
docs: 删除过时文档文件
test: 添加断言日志
```
