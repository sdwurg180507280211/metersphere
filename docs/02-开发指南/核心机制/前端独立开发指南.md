# MeterSphere 前端独立开发指南

## 📐 前端架构说明

MeterSphere 采用**微前端架构**（基于 qiankun 框架），但**每个前端模块都可以独立启动**进行开发。

---

## 🏗️ 前端模块结构

### 主应用（网关/框架）
```
framework/sdk-parent/frontend/
├── 作用：主框架、网关、公共组件库
├── 端口：4000
├── 命令：npm run gateway
└── 包含：登录、布局、国际化、公共组件等
```

### 子应用（业务模块）

| 模块名称 | 目录 | 端口 | 启动命令 | 功能 |
|---------|------|------|---------|------|
| **测试跟踪** | `test-track/frontend/` | 4005 | `npm run track` | 缺陷管理、用例管理 |
| **接口测试** | `api-test/frontend/` | 4009 | `npm run api` | API测试 |
| **性能测试** | `performance-test/frontend/` | 4010 | `npm run performance` | 性能测试 |
| **项目管理** | `project-management/frontend/` | 4002 | `npm run project` | 项目、模板管理 |
| **系统设置** | `system-setting/frontend/` | 4004 | `npm run setting` | 系统配置 |
| **工作台** | `workstation/frontend/` | 4007 | `npm run workstation` | 首页工作台 |
| **报告统计** | `report-stat/frontend/` | 4008 | `npm run report` | 统计报告 |

## 💻 独立启动方式（推荐开发调试）

### 🌟 适用场景
- ✅ 只修改某个模块的前端代码
- ✅ 快速调试和热更新
- ✅ 不需要启动其他模块
- ✅ 减少资源占用

### 方式1：只启动你需要的前端模块（最常用）

#### 示例：只开发"测试跟踪"模块

```bash
# 1. 确保后端已启动（在IDEA中运行）
#    test-track 后端服务运行在 http://localhost:8005

# 2. 安装依赖（首次或package.json有变更时）
cd /Users/edy/ideaProjects/metersphere/test-track/frontend
npm install --registry=https://registry.npmmirror.com

# 3. 启动前端开发服务器
npm run track

# 4. 浏览器访问
#    http://localhost:4005
```

**工作原理**：
- 前端运行在 `http://localhost:4005`
- 通过 `vue.config.js` 中的 proxy 配置，自动代理到后端 `http://localhost:8005`
- **无需启动主应用**，直接访问子应用

**优点**：
- ⚡ 启动速度快（只启动一个模块）
- 🔥 热更新（修改代码实时刷新）
- 💻 资源占用少
- 🎯 专注于单一模块开发

---

### 方式2：完整的微前端开发（需要多模块联调）

```bash
# 1. 启动后端服务（在IDEA中运行所有需要的后端模块）
#    - gateway: 8000
#    - test-track: 8005
#    - api-test: 8009
#    - 其他模块...

# 2. 启动主应用（网关）
cd /Users/edy/ideaProjects/metersphere/framework/sdk-parent/frontend
npm install --registry=https://registry.npmmirror.com
npm run gateway
# 运行在 http://localhost:4000

# 3. 启动你需要的子应用
cd /Users/edy/ideaProjects/metersphere/test-track/frontend
npm run track  # http://localhost:4005

cd /Users/edy/ideaProjects/metersphere/api-test/frontend
npm run api  # http://localhost:4009

# 4. 浏览器访问主应用
#    http://localhost:4000
#    主应用会自动加载子应用
```

**工作原理**：
- 主应用（4000）作为入口
- 通过 qiankun 框架动态加载子应用
- 各子应用运行在各自的端口
- 子应用以微前端方式嵌入到主应用中

**优点**：
- ✅ 完整的微前端体验
- ✅ 多模块联调
- ✅ 更接近生产环境

**缺点**：
- ⚠️ 需要启动多个进程
- ⚠️ 资源占用较大
- ⚠️ 启动时间较长

## 🎯 推荐开发方式

### 场景1：只修改"需求号"或"手机"字段的国际化

**最佳方案**：**独立启动 test-track 前端** ✅

```bash
# 步骤1: 确保后端已启动
# 在IDEA中运行 test-track 后端服务

# 步骤2: 启动前端
cd /Users/edy/ideaProjects/metersphere/test-track/frontend
npm install --registry=https://registry.npmmirror.com  # 首次或依赖变更时
npm run track

# 步骤3: 修改代码
# 编辑 ../../framework/sdk-parent/frontend/src/i18n/lang/track/*.js
# 保存后自动热更新，刷新浏览器即可看到效果

# 步骤4: 浏览器访问
# http://localhost:4005
# 直接测试缺陷管理功能
```

**时间对比**：
| 步骤 | 微前端方式 | 独立方式 |
|------|-----------|---------|
| 安装依赖 | 多个模块，10-20分钟 | 单个模块，3-5分钟 ⚡ |
| 启动服务 | 多个进程，2-5分钟 | 单个进程，30秒 ⚡ |
| 热更新 | 可能较慢 | 极快 ⚡ |
| **总计** | **15-25分钟** | **3-6分钟** ⚡ |

## 🔧 配置说明

### 端口配置

每个前端模块的 `vue.config.js` 中配置：

```javascript
// test-track/frontend/vue.config.js
module.exports = {
  devServer: {
    port: 4005,  // 前端开发服务器端口
    proxy: {
      ["^((?!/login)(?!/document))"]: {
        target: 'http://localhost:8005',  // 后端服务地址
        ws: false,
      },
    },
  },
}
```

**端口映射**：
```
前端端口 → 后端端口
4005 → 8005 (test-track)
4009 → 8009 (api-test)
4002 → 8002 (project-management)
4004 → 8004 (system-setting)
...
```

### 依赖关系

所有子应用都依赖主框架：

```json
// test-track/frontend/package.json
{
  "dependencies": {
    "metersphere-frontend": "file:../../framework/sdk-parent/frontend",
    // 其他依赖...
  }
}
```

**重要**：修改 framework/sdk-parent/frontend 的代码后，需要重新安装依赖：

```bash
# 如果修改了框架代码（如国际化文件）
cd test-track/frontend
rm -rf node_modules/metersphere-frontend
npm install --registry=https://registry.npmmirror.com
```

## ⚠️ 常见问题

### Q1: 独立启动后看不到登录页面？

**A:** 独立启动的子应用没有登录页面，需要：

**方案1**（推荐）：在IDEA中运行后端时，后端会返回登录页面
```bash
# 直接访问 http://localhost:4005
# 如果未登录，后端会重定向到登录页面
```

**方案2**：先在主应用登录
```bash
# 1. 访问 http://localhost:4000 登录
# 2. 登录成功后，Cookie会保存
# 3. 再访问 http://localhost:4005 就能正常使用
```

### Q2: 修改了国际化文件，前端没更新？

**A:** 国际化文件在 framework/sdk-parent/frontend，需要：

```bash
# 方式1：重新安装依赖（推荐）
cd test-track/frontend
rm -rf node_modules/metersphere-frontend
npm install --registry=https://registry.npmmirror.com

# 方式2：重启前端服务
# Ctrl+C 停止，然后重新 npm run track

# 方式3：如果是开发调试，直接修改 node_modules 中的文件（临时）
# node_modules/metersphere-frontend/src/i18n/lang/track/zh-CN.js
```

### Q3: npm install 很慢怎么办？

**A:** 已配置国内镜像源，如果还是慢：

```bash
# 方法1：手动指定镜像源
npm install --registry=https://registry.npmmirror.com

# 方法2：配置全局镜像源
npm config set registry https://registry.npmmirror.com

# 方法3：使用 cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```

### Q4: 端口被占用怎么办？

**A:** 修改 vue.config.js 中的端口：

```javascript
// test-track/frontend/vue.config.js
module.exports = {
  devServer: {
    port: 4015,  // 改成其他端口
    // ...
  },
}
```

## 🎯 总结

### 推荐开发流程

```bash
# 1. 后端：在IDEA中运行需要的后端模块
#    - test-track backend (端口8005)

# 2. 前端：独立启动对应的前端模块
cd /Users/edy/ideaProjects/metersphere/test-track/frontend
npm install --registry=https://registry.npmmirror.com  # 首次
npm run track

# 3. 开发：修改代码，自动热更新

# 4. 测试：浏览器访问 http://localhost:4005

# 5. 打包：完成开发后，执行完整打包流程
#    参考《需求号字段-打包指南.md》
```

### 关键点
- ✅ **99%的情况**：独立启动单个前端模块即可
- ✅ **修改国际化**：需要重新安装依赖
- ✅ **端口映射**：前端 4005 → 后端 8005
- ✅ **热更新**：保存代码自动刷新

**不需要启动微前端！直接独立启动你要开发的模块就行了！** 🚀
