# 故障排查

本文档收录实际遇到的问题及修复方案。

## 1. Jenkins 启动问题

### 问题：一直 activating / 返回 503

**现象**：
- `systemctl status jenkins` 显示 `activating`
- 浏览器访问返回 503 或超时
- journalctl 出现 `start operation timed out`

**原因**：systemd 等待 notify 超时

**修复**：

```bash
sudo mkdir -p /etc/systemd/system/jenkins.service.d

cat <<'EOF' | sudo tee /etc/systemd/system/jenkins.service.d/override.conf
[Service]
Type=simple
NotifyAccess=none
TimeoutStartSec=10min
TimeoutStopSec=5min
EOF

sudo systemctl daemon-reload
sudo systemctl restart jenkins
```

## 2. SSH 部署问题

### 问题：ssh-add error in libcrypto

**现象**：
```
Running ssh-add ...
Error loading key ... error in libcrypto
Finished: FAILURE
```

**原因**：
- 私钥格式与 OpenSSL 不兼容（常见于 ED25519）
- 私钥内容损坏（复制粘贴问题）
- 私钥带口令（流水线无法交互输入）

**修复**：

生成兼容性最佳的 RSA 密钥：

```bash
# 生成 RSA 4096 PEM 格式（无口令）
ssh-keygen -t rsa -b 4096 -m PEM -f ~/.ssh/jenkins_deploy -N ""

# 测试连接
ssh -i ~/.ssh/jenkins_deploy deploy@目标服务器

# 更新 Jenkins 凭证
# Manage Jenkins → Credentials → metersphere-deploy-key
# 替换私钥内容，确保 username 为 deploy
```

### 问题：Permission denied (publickey)

**检查清单**：
1. 目标机 `~deploy/.ssh/authorized_keys` 是否包含公钥
2. 权限是否正确：`.ssh` 目录 700，`authorized_keys` 文件 600
3. Jenkins 凭证 username 是否为 `deploy`

## 3. Docker 权限问题

### 问题：permission denied while trying to connect to Docker daemon

**现象**：
```
Got permission denied while trying to connect to the Docker daemon socket
```

**原因**：执行 docker 命令的用户不在 docker 组

**修复**：

```bash
# 确保 docker 组存在
getent group docker || sudo groupadd docker

# 加入 docker 组
sudo usermod -aG docker jenkins
sudo usermod -aG docker deploy

# 修复 socket 权限
sudo chgrp docker /var/run/docker.sock
sudo chmod 660 /var/run/docker.sock

# 重启服务
sudo systemctl restart docker
sudo systemctl restart jenkins

# 验证
sudo -u jenkins docker version
```

## 4. 构建依赖问题

### 问题：Could not find artifact metersphere-platform-plugin-sdk

**现象**：
```
ERROR: Could not find artifact io.metersphere:metersphere-platform-plugin-sdk:jar:1.6.0
```

**原因**：Jenkins Maven 仓库为空，缺少本地 SDK

**修复方案**（推荐上传本地 jar）：

```bash
# 1. 本地查找 jar
find ~/.m2/repository -name "*platform-plugin-sdk*1.6.0*.jar"

# 2. 上传到服务器
scp ~/.m2/repository/io/metersphere/metersphere-platform-plugin-sdk/1.6.0/*.jar 服务器:/tmp/

# 3. 移动到 Jenkins Maven 仓库
ssh 服务器 '
mkdir -p /var/lib/jenkins/.m2/repository/io/metersphere/metersphere-platform-plugin-sdk/1.6.0
mv /tmp/metersphere-platform-plugin-sdk-*.jar /var/lib/jenkins/.m2/repository/io/metersphere/metersphere-platform-plugin-sdk/1.6.0/
chown -R jenkins:jenkins /var/lib/jenkins/.m2
'
```

### 问题：maven-clean-plugin Failed to delete

**现象**：
```
maven-clean-plugin ... Failed to delete ... createdFiles.lst
```

**原因**：workspace 中有 root 创建的文件，jenkins 用户无权删除

**修复**：

```bash
# 修复归属
sudo chown -R jenkins:jenkins /var/lib/jenkins/workspace/<job>

# 预防措施：手工排查时用 jenkins 用户执行
sudo -u jenkins -H bash -lc 'cd /var/lib/jenkins/workspace/<job> && mvn clean'
```

## 5. Multibranch 分支索引问题

### 问题：git ls-remote 返回 128

**现象**：
```
Command "git ls-remote ..." returned status code 128
Incorrect username or password (access token)
```

**原因**：使用了错误的凭证（如用 ACR 凭证拉 Git）

**修复**：

1. 创建 Gitee 专用凭证（ID: `gitee-token`）
2. 在 Multibranch Job 的 Branch Sources 选择正确凭证

### 问题：扫描到分支但 Jenkinsfile 不生效

**检查**：
1. Script Path 是否为 `Jenkinsfile.enhanced`
2. 文件是否已 commit + push
3. 重新执行 `Scan Multibranch Pipeline Now`

## 6. Docker build 问题

### 问题：COPY not found

**现象**：
```
failed to calculate checksum ... "/test-track/backend/target/dependency": not found
```

**原因**：build context 与 Dockerfile COPY 路径不匹配

**修复**：

| 模块类型 | 正确命令 |
|----------|----------|
| 业务模块 | `docker build -f test-track/Dockerfile .`（从仓库根目录） |
| 框架模块 | `docker build ./framework/gateway`（从模块目录） |

**前置条件**：必须先完成 Maven 打包和 jar 解压，才有 `target/dependency` 目录。

## 7. 前端构建问题

### 问题：npm install 卡死

**现象**：
- 长时间无输出
- `MaxListenersExceededWarning` 警告

**修复**：在 Jenkinsfile 添加环境变量：

```bash
export NODE_OPTIONS="--max-old-space-size=4096"
export npm_config_registry=https://registry.npmmirror.com
export npm_config_fetch_retries=5
export npm_config_fetch_timeout=300000
```

## 快速定位指南

| 报错关键词 | 首先检查 |
|------------|----------|
| `activating` / `503` | Jenkins systemd 配置 |
| `error in libcrypto` | SSH 密钥格式 |
| `permission denied.*docker` | docker 组权限 |
| `Could not find artifact` | Maven 本地仓库 |
| `Failed to delete` | workspace 文件归属 |
| `git ls-remote.*128` | Gitee 凭证 |
| `COPY.*not found` | Docker build context |
