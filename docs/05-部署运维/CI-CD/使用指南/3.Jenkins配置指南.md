## 一、Jenkins环境准备

### 1.1 Jenkins安装

```bash
# Ubuntu/Debian
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt update
sudo apt install jenkins

# 启动Jenkins
sudo systemctl start jenkins
sudo systemctl enable jenkins

# 获取初始密码
sudo cat /var/lib/jenkins/secrets/initialAdminPassword
```

访问：`http://<jenkins-server>:8080`

### 1.2 必要插件安装

进入Jenkins → Manage Jenkins → Manage Plugins → Available

**必装插件**：
- [x] Pipeline
- [x] Git plugin
- [x] SSH Agent Plugin
- [x] Credentials Binding Plugin
- [x] Config File Provider Plugin
- [x] Qy Wechat Notification Plugin（企业微信通知）

**推荐插件**：
- [ ] Blue Ocean（更好的Pipeline可视化）
- [ ] JUnit Plugin（测试报告）
- [ ] SonarQube Scanner（代码质量）

## 二、凭证配置

### 2.1 SSH密钥配置

**步骤1：生成SSH密钥对**
```bash
ssh-keygen -t rsa -b 4096 -C "jenkins@metersphere" -f metersphere-deploy-key
```

**步骤2：在Jenkins中添加凭证**
```
Jenkins → Manage Jenkins → Manage Credentials
→ Global → Add Credentials

类型：SSH Username with private key
ID：metersphere-deploy-key
Username：deploy
Private Key：粘贴metersphere-deploy-key文件内容
```

**步骤3：在部署服务器添加公钥**
```bash
# 在部署服务器上
ssh deploy@test-server
mkdir -p ~/.ssh
cat >> ~/.ssh/authorized_keys
# 粘贴metersphere-deploy-key.pub内容
chmod 600 ~/.ssh/authorized_keys
```

### 2.2 企业微信Webhook配置

**步骤1：创建企业微信机器人**
```
1. 进入企业微信群
2. 右键 → 添加群机器人
3. 复制Webhook地址
```

**步骤2：在Jenkins中添加**
```
Manage Credentials → Add Credentials

类型：Secret text
ID：wechat-bot-webhook
Secret：粘贴Webhook URL
```

### 2.3 服务器IP配置

```
添加两个Secret text凭证：

1. test-server-ip
   Secret：192.168.1.100（测试服务器IP）

2. prod-server-ip
   Secret：192.168.1.200（生产服务器IP）
```

### 2.4 Maven settings.xml配置

**步骤1：准备settings.xml**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0">
    <localRepository>/var/jenkins_home/.m2/repository</localRepository>
    
    <mirrors>
        <mirror>
            <id>aliyun</id>
            <name>Aliyun Maven</name>
            <url>https://maven.aliyun.com/repository/public</url>
            <mirrorOf>central</mirrorOf>
        </mirror>
    </mirrors>
    
    <servers>
        <server>
            <id>fit2cloud-public</id>
            <username>your-username</username>
            <password>your-password</password>
        </server>
    </servers>
    
    <profiles>
        <profile>
            <id>metersphere</id>
            <repositories>
                <repository>
                    <id>fit2cloud-public</id>
                    <url>https://repository.fit2cloud.com/repository/public/</url>
                </repository>
            </repositories>
        </profile>
    </profiles>
    
    <activeProfiles>
        <activeProfile>metersphere</activeProfile>
    </activeProfiles>
</settings>
```

**步骤2：在Jenkins中添加**
```
Manage Jenkins → Managed files → Add a new Config
→ Global Maven settings.xml

ID：metersphere-maven
Content：粘贴上面的settings.xml
```

## 三、Pipeline任务配置

### 3.1 创建Pipeline任务

```
1. New Item
2. 输入名称：MeterSphere-CI/CD
3. 选择：Pipeline
4. 点击：OK
```

### 3.2 基本配置

**General**：
- Description：MeterSphere自动化构建与部署

**Build Triggers**：
- Poll SCM：`H/5 * * * *`（每5分钟检查一次，可选）
- 或不勾选（使用Jenkinsfile中的pollSCM配置）

**Pipeline**：
```
Definition：Pipeline script from SCM

SCM：Git
Repository URL：<your-git-repo-url>
Credentials：<your-git-credentials>
Branch Specifier：*/develop（或根据需要）

Script Path：Jenkinsfile.enhanced
```

### 3.3 参数化构建

在Jenkinsfile中已定义参数，Jenkins会自动识别：
- buildSdk
- deployToTest
- runTests
- frontendLink

## 四、Jenkins节点配置

### 4.1 配置构建节点

```
Manage Jenkins → Manage Nodes → New Node

Name：metersphere
Type：Permanent Agent

配置：
Remote root directory：/home/jenkins
Labels：metersphere
Launch method：Launch agent via SSH
Host：<build-server-ip>
Credentials：<jenkins-ssh-key>
```

### 4.2 节点环境准备

**在构建节点上安装**：
```bash
# Java 17
sudo apt install openjdk-17-jdk

# Maven 3.8.3
wget https://dlcdn.apache.org/maven/maven-3/3.8.3/binaries/apache-maven-3.8.3-bin.tar.gz
sudo tar -xzf apache-maven-3.8.3-bin.tar.gz -C /opt/

# Docker
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker jenkins

# Docker Buildx
docker buildx create --use
```

## 六、Jenkins优化

### 6.1 Maven缓存优化

```groovy
// Jenkinsfile中配置
environment {
    MAVEN_OPTS = '-Dmaven.repo.local=/var/jenkins_home/.m2/repository'
}
```

**效果**：
- 首次构建：15分钟
- 后续构建：3-5分钟
- 提升：70%

### 6.2 并行构建

```groovy
// 如果有多个Jenkins节点
stage('Parallel Build') {
    parallel {
        stage('Build Backend') {
            steps { ... }
        }
        stage('Build Frontend') {
            steps { ... }
        }
    }
}
```

### 6.3 构建结果归档

```groovy
// 归档构建产物
post {
    success {
        archiveArtifacts artifacts: '**/target/*.jar', fingerprint: true
```

## 八、故障排查

### 8.1 构建卡住

**可能原因**：
- Maven下载依赖超时
- Docker构建超时
- SSH连接挂起

**解决**：
- 中止构建
- 检查网络连接
- 重新触发

### 8.2 凭证失效

**现象**：SSH连接失败、推送镜像失败

**解决**：
```
1. Manage Credentials
2. 找到失效的凭证
3. Update更新内容
4. 重新构建
```
