> 版本：v2.10-lts | Java 17 | Spring Boot 3.2.12 | Spring Cloud 2023.0.1 | Vue 2.7

### 2.1 服务端口与注册信息

| 服务 | 端口 | 管理端口 | Eureka 名称 |

| Eureka | 8761 | 8752 | eureka |

| API Test | 8004 | 7004 | api |
| Test Track | 8005 | 7005 | track |
| Report Stat | 8006 | 7006 | report |
| Workstation | 8007 | 7007 | workstation |
| Workflow | 8008 | 7008 | workflow |
| Analytics Stat | 8009 | 7009 | analytics |

### 2.2 配置层级

1. **commons.properties** (`framework/sdk-parent/sdk/src/main/resources/`) — 所有服务共享的基础配置
2. **各服务 application.properties** (`{module}/backend/src/main/resources/`) — 服务端口、Eureka 注册、Flyway 等
3. **外部运行时配置** `/opt/metersphere/conf/metersphere.properties` — 数据库URL、Kafka、Redis 等环境相关配置
4. **Redisson 配置** `/opt/metersphere/conf/redisson.yml` — Redis 客户端配置

### 2.3 commons.properties 关键配置

- **HikariCP**：最大 200 连接，10 分钟空闲，15 分钟最大生命周期
- **Kafka**：Producer 最大请求 524MB，Consumer group `metersphere_group_id`
- **Redis/Session**：12 小时超时，Spring Session indexed 模式
- **文件上传**：最大 1024MB
- **Eureka**：心跳 5s，过期 15s
- **Flyway**：默认关闭，各服务单独启用

### Maven 模块依赖链

```
metersphere (root POM)
 └── framework
      ├── sdk-parent
      │    ├── domain           — MyBatis 生成的领域实体
      │    ├── sdk              — 共享SDK：Eureka client, MyBatis, Shiro, Redis, Kafka, MinIO, JMeter, Selenium, K8s client
      │    ├── xpack-interface  — XPack 扩展接口
      │    ├── jmeter           — JMeter 集成核心
      │    ├── frontend         — 共享 Vue.js 前端库
      │    ├── metersphere-plugin-sdk
      │    └── metersphere-platform-plugin-sdk
      ├── eureka                — 独立 Eureka 服务器
      └── gateway               — Spring Cloud Gateway
 └── api-test / performance-test / ... (各业务模块)
      ├── frontend              — Vue.js 应用，构建到 dist/，复制进后端
      └── backend               — Spring Boot 应用，依赖 sdk + xpack-interface
```

### 各服务后端构建流程

1. **前端构建**（`{module}/frontend/pom.xml`）：使用 frontend-maven-plugin 安装 Node v20.8.1 + npm 8.4.0，执行 `npm install` 和 `npm run build`，输出到 `dist/`
2. **后端构建**（`{module}/backend/pom.xml`）：使用 maven-antrun-plugin 将 `../frontend/dist/` 复制到 `src/main/resources/static/`（资源）和 `src/main/resources/public/`（HTML），然后 spring-boot-maven-plugin 打包 fat JAR

### Gateway 构建差异

Gateway 复制的是共享 SDK 前端（`framework/sdk-parent/frontend/dist/`）而非各服务自有前端。Gateway 是 UI 主入口。

## 4. 前端微前端架构

### 技术栈（所有模块统一）

- Vue 2.7.3 + Element UI 2.15.13
- Pinia 2.x（状态管理）
- vue-i18n（国际化）
- echarts（图表）
- @micro-zoe/micro-app（微前端框架）
- metersphere-frontend（共享SDK库，引用路径 `file:../../framework/sdk-parent/frontend`）

### 架构说明

- **主壳应用**：`framework/sdk-parent/frontend`，作为 Gateway 前端，使用 micro-app 加载子应用
- **子应用**：每个业务模块有独立前端（api-test, test-track 等）
- **共享库**：`metersphere-frontend` 封装通用组件、路由、i18n、Pinia Store，以 CommonJS 库形式构建，被所有子应用作为本地依赖引用

### 前端 dev 端口

| 模块 | Dev 端口 | 代理后端 |
|------|---------|---------|
| 共享 SDK 前端（Gateway 主壳） | 3000 | :8000 |
| API Test | 4004 | :8004 |

## 5. 基础设施

### 5.1 Docker

每个服务有独立 Dockerfile（基于 `alpine-openjdk17-jre`），构建模式：

1. 复制解压后的 JAR（BOOT-INF/lib, META-INF, BOOT-INF/classes）到 `/app/`
2. 复制 xpack JARs 到 `/app/lib/`
3. 复制前端静态资源到 `/app/static/` 和 `/app/public/`
4. 通过 `/deployments/run-java.sh` 启动

生产部署通过 **msctl** 脚本管理 Docker Compose，配置文件位于 `/opt/metersphere/`。

### 5.2 MySQL

- 驱动：`com.mysql:mysql-connector-j`
- 连接池：HikariCP（最大 200 连接）
- 迁移：Flyway，各服务有独立版本表（如 `api_version`, `track_version` 等）
- 迁移脚本位置：`{module}/backend/src/main/resources/db/migration/`
- 数据库 URL、用户名、密码通过 `/opt/metersphere/conf/metersphere.properties` 运行时注入

### 5.3 Redis / Redisson

- Redisson 客户端：`redisson-spring-boot-starter:3.25.0`
- 配置文件：`file:/opt/metersphere/conf/redisson.yml`
- Spring Session：`spring-session-data-redis`，repository-type=indexed
- Session 超时：43200s（12 小时）

### 5.4 Kafka

- Spring Kafka，bootstrap servers 从 `${kafka.bootstrap-servers}` 获取
- Consumer group：`metersphere_group_id`
- Producer：acks=1（api-test），最大请求 524MB
- Analytics 服务有额外的 Kafka 配置用于知识库处理（topic: `knowledge-file-processing`）

### 5.5 服务发现（Eureka）

- Eureka 服务器端口 8761（独立模式，自我保护关闭）
- 所有后端服务注册为 Eureka 客户端
- Gateway 通过 Eureka 发现服务（`spring.cloud.gateway.discovery.locator.enabled=true`）
- 心跳 5s，过期 15s，注册表拉取 10s

### 5.6 API Gateway（Spring Cloud Gateway）

- 端口 8000，基于 WebFlux（非 Servlet）
- 显式路由 analytics 服务（其他通过 Eureka 自动发现）
- 连接超时 30s，响应超时 5 分钟
- WebSocket 最大帧 200MB
- Swagger/OpenAPI 聚合所有服务
- 托管主前端壳（来自 `framework/sdk-parent/frontend/dist/`）

### 5.7 其他组件

| 组件 | 用途 | 说明 |
|------|------|------|
| MinIO | 对象存储 | 文件上传，版本 8.5.3 |
| JMeter 5.5 | 嵌入式测试引擎 | 安装在 `/opt/jmeter` |
| Flowable | 工作流引擎 | 内嵌于 workflow-service |
| Shiro 2.0.1 | 认证/授权 | Jakarta classifier |
| LDAP | 目录服务 | spring-boot-starter-data-ldap |
| Quartz | 任务调度 | 大部分服务启用 |
| K8s Client | 容器编排 | fabric8:kubernetes-client |
| Selenium 4.10.0 | UI 自动化测试 | selenium-java |
| Elasticsearch | 知识库搜索 | analytics 服务使用，配置通过外部 metersphere.properties |

## 6. 常用命令

### 后端构建

```bash
# 全量构建（含前端）
./mvnw clean package -DskipTests

# 仅构建后端（跳过前端）
./mvnw clean package -DskipTests -DskipAntRunForJenkins \
  -pl "!framework/sdk-parent/frontend,!api-test/frontend,!performance-test/frontend,\
!project-management/frontend,!report-stat/frontend,!system-setting/frontend,\
!test-track/frontend,!workstation/frontend"

# 启动单个服务（需要先启动 Eureka + MySQL + Redis + Kafka）
./mvnw spring-boot:run -pl api-test/backend
./mvnw spring-boot:run -pl system-setting/backend
```

### 前端开发

```bash
# 启动主壳（Gateway 前端）
cd framework/sdk-parent/frontend && npm install && npm run gateway   # port 3000 → proxy :8000

# 启动子应用（以 API Test 为例）
cd api-test/frontend && npm install && npm run api                   # port 4004 → proxy :8004

# 启动工作站前端
cd workstation/frontend && npm install && npm run workstation
```
