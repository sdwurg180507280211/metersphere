## 一、Figma Sites 是什么？

### 简单理解

**Figma Sites = 在 Figma 里设计 → 直接发布成网站！**

没有 Figma Sites 时：
```
设计师：（在 Figma 里做网站设计）
    ↓
设计师：给你设计稿
    ↓
开发者：（手动还原成代码）
    ↓
开发者：花了 1-2 周，网站上线了
```

有了 Figma Sites 时：
```
设计师：（在 Figma Sites 里做设计）
    ↓
设计师：点击"发布"
    ↓
网站：（直接上线了！）
```

### 一句话总结

Figma Sites 是一个**无代码网站构建工具**——让设计师在熟悉的 Figma 环境里，直接设计、原型、发布网站，**不需要写一行代码**！

### 1. 设计师也能做网站

## 四、Figma Sites vs Figma Design

| 维度 | Figma Design | Figma Sites |
|-----|-------------|-------------|
| **定位** | UI/UX 设计工具 | 网站构建工具 |
| **输出** | 设计稿（.fig 文件） | 真实网站（URL） |
| **目标用户** | 设计师 | 设计师 + 营销 + 产品 |
| **能不能写代码** | 不能 | 能（AI 代码层） |
| **能不能发布** | 不能 | 能，一键发布 |

**什么时候用哪个？**

- 做 App 设计 → Figma Design
- 做营销页 / 个人博客 → Figma Sites
- 做复杂的 Web 应用 → Figma Design + 开发
- 做快速原型 / 着陆页 → Figma Sites

## 五、Figma Sites 核心概念

### 1. Blocks（块）

Blocks 是 Figma Sites 的基本构建单元：
- 文本块
- 图片块
- 按钮块
- 视频块
- 表单块
- 等等...

就像搭积木一样，把 Blocks 拼起来就是网站！

### 2. Breakpoints（断点）

Breakpoints 让你的网站在不同设备上看起来都好：

| 断点 | 设备 | 宽度 |
|-----|------|------|
| Mobile | 手机 | < 768px |
| Tablet | 平板 | 768px - 1024px |
| Desktop | 桌面 | > 1024px |

在 Figma Sites 里，你可以：
- 为每个断点单独设计
- 一键预览效果
- 不需要写 CSS 媒体查询

### 3. Code Layers（代码层）✨

这是 Figma Sites 最酷的功能之一！

**什么是 Code Layers？**
- 在 Figma Sites 的设计里添加自定义代码
- 可以用 AI 帮你写代码
- 可以实现复杂的交互效果
- 可以集成第三方工具

**例子：**
- 添加评论功能
- 添加购物车
- 添加实时聊天
- 添加自定义动画

这就是为什么那篇文章标题叫 **"Create code layers in Figma Sites with AI"**！

## 七、快速上手：5 分钟做第一个网站

### 步骤 1：打开 Figma Sites

访问 Figma → 点击 "Sites" → 新建网站

### 步骤 2：选个模板

Figma Sites 提供好多模板：
- 个人作品集
- 公司官网
- 产品着陆页
- 博客
- 等等...

选一个喜欢的开始！

### 步骤 3：改改内容

- 把文字改成你的
- 把图片换成你的
- 调整一下颜色
- 就像在 Figma 里改设计一样

### 步骤 4：预览一下

点击 "Preview" 按钮，看看网站在不同设备上的效果：
- 手机端
- 平板端
- 桌面端

### 步骤 5：发布！

点击 "Publish" 按钮，你的网站就上线了！

你会得到一个类似这样的 URL：
```
https://your-name.figma.site
```
