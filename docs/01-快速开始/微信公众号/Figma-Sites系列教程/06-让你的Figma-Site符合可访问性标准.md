# 🔥 让你的 Figma Site 更友好：可访问性完整指南

你的网站很漂亮，但所有人都能用吗？这篇文章带你搞懂可访问性（Accessibility），让你的 Figma Site 对每个人都友好！

---

## 一、先搞懂：可访问性是什么？

### 简单理解

**可访问性 = 让所有人都能用你的网站！**

没有考虑可访问性时：
```
视障用户：（用屏幕阅读器）
屏幕阅读器："图片，图片，图片"
用户：这网站在说什么？？？

键盘用户：（按 Tab 键导航）
用户：焦点去哪儿了？我点不到按钮！

色盲用户：
用户：这个按钮是"可用"还是"不可用"？颜色看起来一样...
```

考虑了可访问性时：
```
视障用户：
屏幕阅读器："产品图片，一款时尚的无线耳机"
用户：明白了！

键盘用户：
用户：（Tab 键导航，焦点清晰可见）
用户：很好，我知道我在哪！

色盲用户：
用户：（按钮除了颜色，还有文字和图标提示）
用户：清楚了！
```

### 为什么可访问性很重要？

| 原因 | 说明 |
|-----|------|
| 🌍 **道德** | 让每个人都能平等使用互联网 |
| 📈 **商业** | 扩大你的用户群体 |
| 🔍 **SEO** | 搜索引擎喜欢可访问的网站 |
| ⚖️ **法律** | 很多国家要求网站可访问 |

## 三、可访问性 Checklist（照着做！）

### ✅ 1. 图片：添加 Alt Text

#### 什么是 Alt Text？

**Alt Text = 图片的文字描述，给屏幕阅读器用的！**

#### 怎么写好 Alt Text？

| 图片类型 | 好的例子 | 不好的例子 |
|---------|---------|-----------|
| **产品图** | "一款白色无线耳机，放在木质桌面上" | "图片1.jpg" |
| **图标** | "搜索图标" | "icon.png" |
| **装饰图** | ""（空，设为装饰性） | "装饰图片" |
| **信息图** | "2024年销售数据：Q1 100万，Q2 120万..." | "图表" |

#### 在 Figma Sites 中设置 Alt Text

1. 选中图片
2. 右侧面板找到 "Accessibility"
3. 填写 "Alt Text"
4. 如果是装饰性图片，勾选 "Decorative"

#### 实战例子

```html
<!-- 不好的做法 -->
<img src="headphone.jpg">

<!-- 好的做法 -->
<img src="headphone.jpg" alt="一款白色无线耳机，放在木质桌面上">

<!-- 装饰性图片 -->
<img src="decorative-line.jpg" alt="" role="presentation">
```

### ✅ 2. 文字：确保对比度足够

#### 什么是对比度？

**对比度 = 文字和背景的亮度差异！**

#### WCAG 标准

| 标准 | 普通文字 | 大文字（≥18px） |
|-----|---------|----------------|
| **AA（推荐）** | 4.5:1 | 3:1 |
| **AAA（最高）** | 7:1 | 4.5:1 |

#### 在 Figma Sites 中检查对比度

1. 选中文字
2. 右侧面板找到 "Accessibility"
3. 查看对比度评分
4. 如果不够，点击 "Auto-adjust" 自动调整

#### 常见错误和修复

**错误：**
```
浅灰色文字 (#999999) 在白色背景上
对比度：2.3:1 ❌
```

**修复：**
```
深灰色文字 (#444444) 在白色背景上
对比度：7.1:1 ✅
```

#### 颜色检查工具

Figma Sites 内置了检查器，你也可以用在线工具：
- WebAIM Contrast Checker
- Colorable

### ✅ 3. 链接：描述清楚目标

#### 什么是好的链接文字？

**链接文字应该告诉用户点击后会去哪里！**

| 不好的链接 | 好的链接 |
|-----------|---------|
| "点击这里" | "查看产品文档" |
| "了解更多" | "阅读我们的隐私政策" |
| "链接" | "下载用户手册" |

#### 为什么"点击这里"不好？

屏幕阅读器用户有时会只听所有链接列表：

**不好的体验：**
```
链接列表：
- 点击这里
- 点击这里
- 了解更多
- 点击这里
用户：这些链接都去哪？？？
```

**好的体验：**
```
链接列表：
- 查看产品文档
- 阅读隐私政策
- 下载用户手册
用户：清楚了！
```

#### 在 Figma Sites 中设置链接

1. 选中文字或按钮
2. 添加链接
3. 确保链接文字有意义

### ✅ 4. 标题：使用正确的层级

#### 标题层级是什么？

**标题层级 = 给内容分级，就像书的目录！**

```
H1：页面主标题（只有一个）
  ├─ H2：主要章节
  │   ├─ H3：子章节
  │   └─ H3：子章节
  └─ H2：主要章节
      └─ H3：子章节
```

#### 在 Figma Sites 中设置标题

1. 选中文字
2. 在右侧面板选择 Heading 类型
3. H1、H2、H3... 按层级来

#### 好的例子

```
H1: Figma Sites 教程
  H2: 什么是 Figma Sites？
    H3: 核心概念
    H3: 主要功能
  H2: 如何开始？
    H3: 注册账号
    H3: 创建第一个项目
```

#### 常见错误

**错误 1：用 H1 做样式（不是真正的标题）**
```
H1: 🔥 🔥 🔥 🔥 （只是装饰，不是标题）
```

**错误 2：跳过层级**
```
H1: 标题
  H3: 直接跳到 H3，跳过 H2 ❌
```

### ✅ 5. 按钮：可点击且有反馈

#### 按钮应该多大？

**最小点击区域：44x44px（苹果标准）**

在 Figma Sites 中：
- 按钮高度至少 44px
- 宽度足够放下文字
- 周围有足够留白

#### 按钮状态

按钮应该有这些状态：

| 状态 | 说明 |
|-----|------|
| **默认** | 正常状态 |
| **悬停** | 鼠标移上去时 |
| **焦点** | 用 Tab 键选中时 |
| **按下** | 点击时 |
| **禁用** | 不可用时 |

#### 在 Figma Sites 中设置按钮状态

Figma Sites 会自动处理大部分状态！你只需要：
1. 添加按钮
2. 设置默认样式
3. Figma Sites 自动添加悬停、焦点等效果

### ✅ 6. 表单：标签清晰

#### 每个输入框都要有标签

**不好的做法：**
```
[输入框] （placeholder 当标签用）
```

**好的做法：**
```
邮箱：
[输入框]
```

#### 在 Figma Sites 中添加表单

Figma Sites 的 Form Block 会自动处理可访问性！

1. 添加 Form Block
2. 每个字段都填写 Label
3. 必填字段标记为 Required
4. 添加提交按钮

### ✅ 7. 键盘：支持全键盘操作

#### 键盘导航顺序

用户应该能用 Tab 键依次访问所有交互元素：

1. 导航链接
2. 按钮
3. 表单输入框
4. 链接

#### 焦点可见

当用户用 Tab 键导航时，当前元素应该有明显的焦点指示：

**好的焦点样式：**
- 明显的边框
- 或者明显的背景变化
- 不要只靠颜色

Figma Sites 默认就有焦点样式，不用你额外设置！

#### 测试键盘操作

自己测试一下：
1. 打开 Preview
2. 按 Tab 键，看看能不能导航所有元素
3. 按 Enter 或 Space 键，看看能不能激活按钮/链接
4. 按 Esc 键，看看能不能关闭弹窗

### ✅ 8. 动画：尊重用户偏好

####  prefers-reduced-motion

有些用户不喜欢动画，甚至会因为动画感到眩晕。

**在 Figma Sites 中：**
Figma Sites 会自动检测用户的系统设置！
- 如果用户开启"减少动画"，动画会自动减弱或关闭
- 你不用额外做什么

#### 动画最佳实践

1. **不要自动播放**
   - 视频、轮播图不要自动播放
   - 让用户自己控制

2. **提供暂停选项**
   - 如果有自动播放的内容
   - 提供暂停/停止按钮

3. **动画不要太夸张**
   - 避免闪烁、快速滚动
   - 保持动画柔和、简短

## 四、实战：检查并修复一个页面

让我们用一个例子，把上面的知识用起来！

### 假设你的页面有这些问题

```
页面内容：
1. [图片] （没有 Alt Text）
2. "点击这里了解更多" （链接文字不好）
3. 浅灰色文字 （对比度不够）
4. 标题层级混乱
```

### 修复步骤

#### 步骤 1：检查图片

1. 选中图片
2. 添加 Alt Text："产品截图，展示 Figma Sites 编辑器界面"
3. 完成！

#### 步骤 2：改进链接

1. 选中链接文字
2. 把"点击这里了解更多"改成"查看 Figma Sites 文档"
3. 完成！

#### 步骤 3：修复对比度

1. 选中浅灰色文字
2. 打开 Accessibility 面板
3. 看到对比度警告：2.1:1 ❌
4. 点击 "Auto-adjust"
5. 自动调整为深灰色，对比度 5.2:1 ✅
6. 完成！

#### 步骤 4：整理标题层级

1. 找到主标题，设为 H1
2. 找到章节标题，设为 H2
3. 找到子章节，设为 H3
4. 确保顺序正确，不跳级
5. 完成！

#### 步骤 5：运行可访问性检查

1. 点击右侧 Accessibility 标签
2. 点击 "Run Check"
3. 查看是否还有警告
4. 逐个修复

#### 步骤 6：真实测试

1. 用键盘 Tab 一遍，确保能导航
2. 用屏幕阅读器试读（可选）
3. 在手机上看看效果
4. 完成！

## 六、常见问题

### Q: 可访问性会让网站变丑吗？

A: 不会！可访问性和美观不冲突。实际上，好的可访问性通常意味着更好的设计：
- 对比度足够 → 文字更清晰
- 留白足够 → 看起来更舒适
- 结构清晰 → 用户更容易理解

### Q: 我需要为可访问性做很多额外工作吗？

A: Figma Sites 已经帮你做了大部分！你只需要：
1. 给图片加 Alt Text
2. 确保链接文字有意义
3. 检查对比度
4. 用正确的标题层级

就这几件事，不难！

### Q: 可访问性只对残障用户重要吗？

A: 不是！可访问性让所有人受益：
- 老年人：视力、听力下降
- 暂时受伤：手臂骨折，用不了鼠标
- 环境影响：在阳光下看手机，需要高对比度
- 手机用户：用键盘操作有时更快

### Q: 怎么知道我的网站够不够好？

A: 三个层次的测试：
1. **Figma Sites 自动检查**：看有没有警告
2. **自己测试**：用键盘 Tab 一遍，看看顺不顺
3. **用户测试**：找真实用户试试，包括残障用户

## 七、最佳实践

### ✅ 应该做的

1. **从一开始就考虑可访问性**
   - 不要等做完了再补
   - 设计时就想着可访问性

2. **用 Figma Sites 的检查工具**
   - 经常运行可访问性检查
   - 看到警告就马上修

3. **测试键盘操作**
   - 自己用 Tab 键导航一遍
   - 确保所有功能都能用键盘完成

4. **写有意义的 Alt Text**
   - 描述图片内容
   - 不要只说"图片"

5. **保持简单**
   - 复杂的设计往往可访问性差
   - 简单清晰的设计对 everyone 都好

### ❌ 不应该做的

1. **不要用颜色作为唯一提示**
   - "红色按钮是错误，绿色是成功"
   - 还要加图标或文字提示

2. **不要用"点击这里"做链接文字**
   - 链接文字要说明去哪
   - 让屏幕阅读器用户也能理解

3. **不要把 placeholder 当标签**
   - placeholder 只是提示，不是标签
   - 每个输入框都要有真实的 label

4. **不要自动播放视频/音频**
   - 会吓到用户
   - 消耗流量
   - 让用户自己控制

5. **不要忽略键盘用户**
   - 不是所有人都用鼠标
   - 确保所有功能都能用键盘操作

---

## 八、总结

可访问性记住这几点：

1. **图片**：加 Alt Text
2. **文字**：对比度足够
3. **链接**：描述清楚
4. **标题**：层级正确
5. **键盘**：支持全键盘操作

Figma Sites 已经帮你做了很多，你只需要做好这几件简单的事！

记住：可访问性不是"额外工作"，而是"好设计的一部分"。
