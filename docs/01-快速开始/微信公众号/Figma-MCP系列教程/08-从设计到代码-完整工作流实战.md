```
1. 页面结构

3. 设计 Token（颜色、间距、字体）
```

AI 会通过 MCP 读取设计稿，然后给你分析：

```
好的，我分析了这个登录页设计：

## 页面结构
- 整体：垂直布局，居中对齐
- 从上到下：Logo → 邮箱输入框 → 密码输入框 → 登录按钮 → 注册链接

## 使用的组件
- Button (Primary variant)
- Input (Email variant)
- Input (Password variant)
- Text (Body variant)

## 设计 Token
- 主色: #1890FF (Primary Blue)
- 背景: #FFFFFF
- 边框: #E0E0E0
- 间距: 16px (md)

## 布局方式
- 使用 Auto Layout，垂直方向
- 元素间距 16px
- 整体居中，上下边距 40px
```

### AI 生成的代码

```tsx
// src/pages/login-page.tsx
import React, { useState } from 'react';
import { Button } from 'src/components/Button';
import { Input } from 'src/components/Input';
import { Text } from 'src/components/Text';

interface LoginFormData {
  email: string;
  password: string;
}

interface LoginPageProps {
  onLogin?: (data: LoginFormData) => void;
}

export const LoginPage: React.FC<LoginPageProps> = ({ onLogin }) => {
  const [formData, setFormData] = useState<LoginFormData>({
    email: '',
    password: '',
  });
  const [isLoading, setIsLoading] = useState(false);
  const [errors, setErrors] = useState<Partial<LoginFormData>>({});

  const validateForm = (): boolean => {
    const newErrors: Partial<LoginFormData> = {};

    if (!formData.email) {
      newErrors.email = '请输入邮箱';
    } else if (!/\S+@\S+\.\S+/.test(formData.email)) {
      newErrors.email = '邮箱格式不正确';
    }

    if (!formData.password) {
      newErrors.password = '请输入密码';
    } else if (formData.password.length < 6) {
      newErrors.password = '密码至少 6 位';
    }

    setErrors(newErrors);
    return Object.keys(newErrors).length === 0;
  };

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();

    if (!validateForm()) {
      return;
    }

    setIsLoading(true);
    try {
      await onLogin?.(formData);
    } finally {
      setIsLoading(false);
    }
  };

  return (
    <div className="login-page">
      <form onSubmit={handleSubmit} className="login-form">
        <Text variant="h2" align="center">
          登录
        </Text>

        <Input
          type="email"
          placeholder="请输入邮箱"
          value={formData.email}
          onChange={(e) => setFormData({ ...formData, email: e.target.value })}
          error={errors.email}
          disabled={isLoading}
        />

        <Input
          type="password"
          placeholder="请输入密码"
          value={formData.password}
          onChange={(e) => setFormData({ ...formData, password: e.target.value })}
          error={errors.password}
          disabled={isLoading}
        />

        <Button
          variant="primary"
          size="md"
          type="submit"
          disabled={isLoading}
          loading={isLoading}
        >
          {isLoading ? '登录中...' : '登录'}
        </Button>

        <Text variant="body" align="center">
          还没有账号？<a href="/register">立即注册</a>
        </Text>
      </form>
    </div>
  );
};
```

完美！这完全像是你们团队自己写的代码！

## 七、常见问题

### Q: 这个工作流需要多长时间？

A: 取决于页面复杂度：
- 简单页面（登录页）：10-15 分钟
- 中等页面（列表页）：20-30 分钟
- 复杂页面（Dashboard）：1 小时左右

以前手动写可能要半天，现在 AI 帮你加速 5-10 倍！

### Q: AI 生成的代码需要改吗？

A: 取决于你的 Prompt 写得好不好：
- 描述得详细 → 几乎不用改
- 描述得简单 → 可能需要小调整

建议：先生成，再微调。

### Q: 设计师不懂技术怎么办？

A: 没关系！设计师只要：
1. 正常在 Figma 里设计
2. 用 Components 和 Variables
3. 设置好 Auto Layout

剩下的交给你和 AI！

### Q: 团队协作时怎么同步？

A: 把这些文件提交到 Git：
- Code Connect 配置
- Skills 文件
- 生成的代码（当然）

团队成员都能用同样的工作流！

## 八、总结

这个完整工作流的核心是：
1. **Figma MCP**：让 AI 能"看"懂设计
2. **Code Connect**：让 AI 知道用哪个组件
3. **Skills**：让 AI 知道怎么写代码

三者配合，效果翻倍：
- 以前：设计 → 手动量像素 → 写代码 → 调整 → 再调整
- 现在：设计 → AI 生成 → 微调 → 完成

快去试试这个工作流吧！你会爱上这种效率的！
