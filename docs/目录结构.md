## 🎯 目录设计原则

### 1. 按角色分类
- **新手开发者**：01-快速开始
- **功能开发者**：02-开发指南 + 03-功能模块
- **运维工程师**：05-部署运维
- **架构师**：04-技术架构 + 06-数据库

### 2. 按生命周期
```
开发 → 测试 → 部署 → 运维
 ↓       ↓       ↓       ↓
02-03   03     05      05
```

### 3. 扁平化结构
- 最多2-3层嵌套
- 避免过深的目录层级
- 便于快速定位文档

### 4. 资源集中
- SQL脚本统一在 `06-数据库/SQL脚本/`
- CI/CD文档统一在 `05-部署运维/CI-CD/`
- 权限相关统一在 `03-功能模块/权限管理/`

## 📝 文档命名规范

### 中文命名
- 使用清晰的中文描述
- 技术术语保持英文（Maven、Docker、JVM等）
- 使用连字符分隔多个概念

### 示例
✅ 好的命名：
- `缺陷管理页面加载优化.md`
- `Maven-在MeterSphere项目中的使用与常见问题.md`
- `page.condition核心机制-Part1-概述与filters.md`

❌ 不好的命名：
- `doc1.md`
- `temp.md`
- `新建文档.md`
