# 功能用例模块树递归机制详解

## 一、概述

功能用例页面左侧的模块树是一个**递归树形结构**，用于组织和管理测试用例。本文档详细说明模块树的数据结构、递归逻辑、前后端交互流程以及统计机制。

---

## 二、数据库表结构

### 2.1 核心表：test_case_node

模块树的数据存储在 `test_case_node` 表中：

```sql
CREATE TABLE `test_case_node` (
  `id` varchar(50) NOT NULL COMMENT '模块ID',
  `project_id` varchar(50) NOT NULL COMMENT '所属项目ID',
  `name` varchar(100) NOT NULL COMMENT '模块名称',
  `parent_id` varchar(50) DEFAULT NULL COMMENT '父模块ID（顶级模块为NULL）',
  `level` int DEFAULT 1 COMMENT '模块层级（1=顶级，2=二级，以此类推）',
  `create_time` bigint NOT NULL COMMENT '创建时间（毫秒时间戳）',
  `update_time` bigint NOT NULL COMMENT '更新时间（毫秒时间戳）',
  `pos` double DEFAULT NULL COMMENT '排序位置（用于同级模块排序）',
  `create_user` varchar(100) DEFAULT NULL COMMENT '创建人',
  PRIMARY KEY (`id`),
  KEY `idx_project_id` (`project_id`),
  KEY `idx_name` (`name`),
  KEY `idx_parent_id` (`parent_id`),
  KEY `idx_level` (`level`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 2.2 关键字段说明

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | varchar(50) | 模块唯一标识 | `09756589-a135-482b-bd6d-b9b9e1c69597` |
| `project_id` | varchar(50) | 所属项目ID | `1ad3f2bb-2cd9-4ca4-990d-d44824a57ffd` |
| `name` | varchar(100) | 模块名称 | `个险保全` |
| `parent_id` | varchar(50) | 父模块ID（顶级模块为NULL） | `NULL` 或 `f08ff4b6-6677-488d-8b0e-8b708fe76534` |
| `level` | int | 模块层级（1=顶级） | `1`, `2`, `3` |
| `pos` | double | 同级排序位置 | `65536.0`, `131072.0` |

### 2.3 树形结构示例

```
项目A
├── 个险保全 (level=1, parent_id=NULL)
│   ├── ICBS-POS-20251212-5134 (level=2, parent_id=个险保全ID)
│   └── 保全业务 (level=2, parent_id=个险保全ID)
│       └── 保单变更 (level=3, parent_id=保全业务ID)
├── 个险契约 (level=1, parent_id=NULL)
└── 出单中心 (level=1, parent_id=NULL)
```

#### 3.1.2 SQL查询（ExtTestCaseNodeMapper.xml）

```xml
<select id="getNodeTreeByProjectId" resultType="io.metersphere.dto.TestCaseNodeDTO">
    SELECT
        id, project_id, name, parent_id, level, 
        create_time, update_time, pos, create_user
    FROM test_case_node
    WHERE project_id = #{projectId}
    ORDER BY pos ASC  -- 按排序位置升序排列
```

**关键点**：
- 查询返回的是**扁平列表**，不是树形结构
- 按 `pos` 字段排序，确保同级模块按正确顺序显示
- 树形结构的构建在**Java代码中完成**

### 3.2 Service层递归构建树（TestCaseNodeService.java）

#### 3.2.1 入口方法

```java
/**
 * 获取项目的模块树（带用例数量统计）
 * @param projectId 项目ID
 * @param request 查询条件（用于过滤用例）
 * @return 树形结构的模块列表
 */
public List<TestCaseNodeDTO> getNodeTreeByProjectId(String projectId, QueryTestCaseRequest request) {
    // 1. 检查并创建默认模块（如果不存在）
    checkDefaultNode(projectId);
    
    // 2. 设置查询条件
    request.setProjectId(projectId);
    request.setUserId(SessionUtils.getUserId());
    request.setNodeIds(null);
    
    // 3. 查询每个模块的用例数量（带过滤条件）
    List<TestCaseNodeDTO> countNodes = extTestCaseMapper.getCountNodes(request);
    
    // 4. 查询所有模块节点（扁平列表）
    List<TestCaseNodeDTO> testCaseNodes = extTestCaseNodeMapper.getNodeTreeByProjectId(projectId);
    
    // 5. 构建树形结构并填充用例数量
    return getNodeTrees(testCaseNodes, getCountMap(countNodes));
}
```

#### 3.2.2 递归构建树的核心方法（继承自NodeTreeService）

```java
/**
 * 将扁平列表转换为树形结构
 * @param nodes 扁平的节点列表
 * @return 树形结构的根节点列表
 */
public List<TestCaseNodeDTO> getNodeTrees(List<TestCaseNodeDTO> nodes) {
    if (CollectionUtils.isEmpty(nodes)) {
        return null;
    }
    
    // 1. 找出所有顶级节点（level=1 或 parent_id=null）
    List<TestCaseNodeDTO> nodeTreeList = new ArrayList<>();
    Map<Integer, List<TestCaseNodeDTO>> nodeLevelMap = new HashMap<>();
    
    // 按层级分组
    nodes.forEach(node -> {
        Integer level = node.getLevel();
        if (nodeLevelMap.containsKey(level)) {
            nodeLevelMap.get(level).add(node);
        } else {
            List<TestCaseNodeDTO> testCaseNodes = new ArrayList<>();
            testCaseNodes.add(node);
            nodeLevelMap.put(level, testCaseNodes);
        }
    });
    
    // 2. 递归构建树：从顶级节点开始，逐层向下查找子节点
    List<TestCaseNodeDTO> rootNodes = Optional.ofNullable(nodeLevelMap.get(1))
            .orElse(new ArrayList<>());
    
    rootNodes.forEach(rootNode -> {
        // 递归构建子树
        buildNodeTree(nodeLevelMap, rootNode, 1);
        nodeTreeList.add(rootNode);
    });
    
    return nodeTreeList;
}

/**
 * 递归构建子树
 * @param nodeLevelMap 按层级分组的节点Map
 * @param pNode 父节点
 * @param level 当前层级
 */
private void buildNodeTree(Map<Integer, List<TestCaseNodeDTO>> nodeLevelMap, 
                          TestCaseNodeDTO pNode, int level) {
    // 获取下一层级的所有节点
    List<TestCaseNodeDTO> levelNodes = nodeLevelMap.get(level + 1);
    if (levelNodes == null) {
        return;
    }
    
    // 找出当前节点的所有子节点
    List<TestCaseNodeDTO> children = new ArrayList<>();
    levelNodes.forEach(item -> {
        if (item.getParentId() != null && item.getParentId().equals(pNode.getId())) {
            children.add(item);
            // 递归构建子节点的子树
            buildNodeTree(nodeLevelMap, item, level + 1);
        }
    });
    
    // 设置子节点列表
    pNode.setChildren(children);
}
```

## 四、用例数量统计机制

### 4.1 统计查询（ExtTestCaseMapper.xml）

```xml
<select id="getCountNodes" resultType="io.metersphere.dto.TestCaseNodeDTO">
    SELECT 
        node_id AS id,
        COUNT(id) AS caseNum
    FROM test_case
    WHERE project_id = #{request.projectId}
      AND status != 'Trash'  -- 排除回收站用例
      <!-- 其他过滤条件：创建人、标签、自定义字段等 -->
    GROUP BY node_id
</select>
```

### 4.2 统计结果填充

```java
/**
 * 将用例数量填充到模块树中
 * @param nodes 模块树
 * @param countMap 模块ID -> 用例数量的映射
 * @return 填充后的模块树
 */
public List<TestCaseNodeDTO> getNodeTrees(List<TestCaseNodeDTO> nodes, 
                                          Map<String, Integer> countMap) {
    List<TestCaseNodeDTO> nodeTreeList = getNodeTrees(nodes);
    
    // 递归填充用例数量
    nodeTreeList.forEach(node -> {
        fillNodeCount(node, countMap);
    });
    
    return nodeTreeList;
}

/**
 * 递归填充节点的用例数量
 */
private void fillNodeCount(TestCaseNodeDTO node, Map<String, Integer> countMap) {
    // 设置当前节点的用例数量
    Integer count = countMap.get(node.getId());
    node.setCaseNum(count == null ? 0 : count);
    
    // 递归处理子节点
    List<TestCaseNodeDTO> children = node.getChildren();
    if (children != null && !children.isEmpty()) {
        children.forEach(child -> fillNodeCount(child, countMap));
    }
}
```

## 五、SQL递归查询（WITH RECURSIVE）

### 5.1 递归CTE统计顶级模块及所有子模块的用例数

```sql
-- 使用递归CTE遍历模块树的所有层级
WITH RECURSIVE module_tree AS (
    -- 锚点查询：选择所有顶级模块（level = 1）
    SELECT 
        id, 
        project_id, 
        name, 
        parent_id, 
        level, 
        id AS top_module_id,           -- 标记顶级模块ID
        name AS top_module_name         -- 标记顶级模块名称
    FROM test_case_node
    WHERE level = 1
    
    UNION ALL
    
    -- 递归查询：通过parent_id关联找出所有子模块
    SELECT 
        tcn.id, 
        tcn.project_id, 
        tcn.name, 
        tcn.parent_id, 
        tcn.level, 
        mt.top_module_id,               -- 继承顶级模块ID
        mt.top_module_name              -- 继承顶级模块名称
    FROM test_case_node tcn
    INNER JOIN module_tree mt ON tcn.parent_id = mt.id
)
-- 最终统计：按顶级模块分组，统计所有子孙模块的用例数
SELECT 
    w.name AS '工作空间名称',
    p.name AS '项目名称',
    mt.top_module_name AS '顶级模块名称',
    COUNT(DISTINCT tc.id) AS '功能用例数量（含所有子模块）'
FROM 
    workspace w
    INNER JOIN project p ON w.id = p.workspace_id
    LEFT JOIN module_tree mt ON mt.project_id = p.id
    LEFT JOIN test_case tc ON mt.id = tc.node_id AND tc.project_id = p.id
WHERE 
    w.name = '默认工作空间'
GROUP BY 
    w.name, p.name, mt.top_module_name
ORDER BY 
    p.name, mt.top_module_name;
```

### 5.2 递归CTE工作原理

```
执行步骤：

1. 锚点查询（Anchor Query）
   ┌─────────────────────────────────────────────────┐
   │ SELECT * FROM test_case_node WHERE level = 1    │
   │ 结果：个险保全(L1), 个险契约(L1), 出单中心(L1)  │
   └─────────────────────────────────────────────────┘
                        ↓
2. 第一次递归（找Level 2节点）
   ┌─────────────────────────────────────────────────┐
   │ SELECT * FROM test_case_node                    │
   │ WHERE parent_id IN (个险保全ID, 个险契约ID, ...) │
   │ 结果：ICBS-POS(L2), 保全业务(L2), ...           │
   └─────────────────────────────────────────────────┘
                        ↓
3. 第二次递归（找Level 3节点）
   ┌─────────────────────────────────────────────────┐
   │ SELECT * FROM test_case_node                    │
   │ WHERE parent_id IN (ICBS-POS ID, 保全业务ID, ...)│
   │ 结果：保单变更(L3), ...                         │
   └─────────────────────────────────────────────────┘
                        ↓
4. 递归终止（没有更多子节点）
   ┌─────────────────────────────────────────────────┐
   │ 所有节点都已遍历完成                            │
   │ module_tree表包含所有层级的节点                 │
   │ 每个节点都标记了所属的顶级模块ID                │
   └─────────────────────────────────────────────────┘
                        ↓
5. 最终统计
   ┌─────────────────────────────────────────────────┐
   │ GROUP BY top_module_name                        │
   │ 统计每个顶级模块及其所有子孙模块的用例总数     │
   └─────────────────────────────────────────────────┘
```

## 七、关键技术点总结

### 7.1 为什么使用递归？

1. **树形结构的本质**：模块树是典型的树形数据结构，每个节点可以有多个子节点
2. **层级不固定**：模块层级深度不固定（可能是2层、3层、甚至10层）
3. **递归是最自然的解决方案**：通过递归可以优雅地处理任意深度的树

### 7.2 性能优化策略

| 优化点 | 实现方式 | 效果 |
|--------|---------|------|
| **单次查询** | 一次性查询所有节点，避免N+1问题 | 减少数据库查询次数 |
| **按层级分组** | 使用Map按level分组，避免重复遍历 | 时间复杂度O(n) |
| **pos排序** | 数据库层面排序，前端直接使用 | 避免前端排序计算 |
| **懒加载** | 前端el-tree支持懒加载子节点 | 减少初始渲染数据量 |

### 7.3 数据一致性保证

1. **默认模块检查**：每次查询前检查并创建默认模块（"未规划用例"）
2. **分布式锁**：使用Redisson分布式锁防止并发创建默认模块
3. **事务保证**：模块的增删改操作使用`@Transactional`保证原子性

```java
// 使用分布式锁创建默认模块
private static final String TEST_CASE_DEFAULT_NODE_CREATE_KEY = "TEST_CASE:DEFAULT_NODE:CREATE";

public TestCaseNode checkDefaultNode(String projectId) {
    TestCaseNode defaultNode = getDefaultNode(projectId);
    if (defaultNode == null) {
        RLock lock = redissonClient.getLock(TEST_CASE_DEFAULT_NODE_CREATE_KEY + ":" + projectId);
        try {
            lock.lock();
            // 双重检查
            defaultNode = getDefaultNode(projectId);
            if (defaultNode == null) {
                return createDefaultNode(projectId);
            }
        } finally {
            lock.unlock();
        }
    }
    return defaultNode;
}
```

## 八、常见问题与解决方案

### 8.1 模块树显示不全

**原因**：parent_id或level字段数据异常

**解决方案**：
```sql
-- 检查孤儿节点（parent_id指向不存在的节点）
SELECT * FROM test_case_node tcn
WHERE parent_id IS NOT NULL
  AND NOT EXISTS (
      SELECT 1 FROM test_case_node parent 
      WHERE parent.id = tcn.parent_id
  );

-- 修复：将孤儿节点挂到默认模块下
UPDATE test_case_node 
SET parent_id = (SELECT id FROM test_case_node WHERE name = '未规划用例' AND level = 1)
WHERE parent_id IS NOT NULL
  AND NOT EXISTS (SELECT 1 FROM test_case_node parent WHERE parent.id = test_case_node.parent_id);
```

### 8.2 用例数量统计不准确

**原因**：回收站用例未排除，或过滤条件未生效

**解决方案**：
```java
// 确保查询时排除回收站用例
request.setStatus("!Trash");

// 确保自定义字段过滤条件正确设置
ServiceUtils.setBaseQueryRequestCustomMultipleFields(request);
```

### 8.3 模块树加载慢

**原因**：用例数量统计查询慢

**优化方案**：
1. 为 `test_case.node_id` 添加索引
2. 使用缓存存储模块树结构（Redis）
3. 前端使用虚拟滚动（大量模块时）

```sql
-- 添加索引
CREATE INDEX idx_node_id ON test_case(node_id);
CREATE INDEX idx_project_status ON test_case(project_id, status);
```

## 十、总结

功能用例模块树的递归机制核心要点：

1. **数据结构**：通过 `parent_id` 和 `level` 字段构建树形关系
2. **查询策略**：一次性查询所有节点（扁平列表），在Java代码中递归构建树
3. **递归逻辑**：按层级分组，从顶级节点开始逐层向下查找子节点
4. **统计机制**：单独查询用例数量，递归填充到树节点中
5. **SQL递归**：使用 `WITH RECURSIVE` CTE 可以在数据库层面实现递归统计
6. **前端渲染**：使用 el-tree 组件自动渲染树形结构

这种设计既保证了查询效率（单次查询），又保持了代码的清晰性（递归构建），是处理树形数据的经典模式。
