# 🔥 Remote vs Desktop：两种 Figma MCP 服务器怎么选？

Figma 提供了两种 MCP 服务器：**Remote** 和 **Desktop**。到底应该用哪一个？这篇文章帮你彻底搞懂！

---

## 一、一句话总结

| 维度 | Remote MCP | Desktop MCP |
|-----|-----------|------------|
| **推荐度** | ⭐⭐⭐⭐⭐ | ⭐⭐ |
| **配置难度** | 简单 | 中等 |
| **功能完整性** | 最全 | 部分 |
| **数据位置** | Figma 服务器 | 本地 |
| **适合人群** | 大多数人 | 企业/隐私需求 |

**90% 的人应该用 Remote MCP！**

---

## 二、Remote MCP 详细介绍

### 是什么？

Remote MCP 是 **Figma 官方托管的云端服务器**。

就像：
- 你用 Gmail，邮件存在 Google 服务器
- 你用 Remote MCP，MCP 服务在 Figma 服务器

### 优点

✅ **配置超级简单**
```
claude plugin install figma@claude-plugins-official
```
一行命令搞定！

✅ **功能最全**
- 读取设计稿 ✓
- 生成代码 ✓
- **写入画布** ✓（AI 直接在 Figma 里画图）
- **网页抓图** ✓（把网页变成 Figma 设计稿）

✅ **官方维护**
- 自动更新
- 问题修复快
- 新功能第一时间用上

✅ **跨设备同步**
- 公司电脑、家里电脑都能用
- 配置一次，到处生效

### 缺点

❌ **需要联网**
- 必须能访问 Figma 服务器
- 断网就不能用了

❌ **数据经过 Figma 服务器**
- 设计稿数据会经过 Figma 服务器
- （但 Figma 有完善的隐私保护）

---

## 三、Desktop MCP 详细介绍

### 是什么？

Desktop MCP 在 **你的本地 Figma 桌面端**运行。

就像：
- 你用本地 Excel，文件存在你电脑
- 你用 Desktop MCP，服务在你本地

### 优点

✅ **完全本地**
- 数据不离开你的电脑
- 隐私保护最好

✅ **不需要联网**
- 断网也能用
- 适合内网环境

✅ **企业友好**
- 可以完全控制数据流向
- 适合有安全合规要求的公司

### 缺点

❌ **配置复杂**
- 需要安装 Figma 桌面端
- 需要手动配置
- 步骤更多

❌ **功能不全**
- 读取设计稿 ✓
- 生成代码 ✓
- **写入画布** ✗（不支持）
- **网页抓图** ✗（不支持）

❌ **需要 Figma 桌面端**
- 不能用网页版 Figma
- 必须下载桌面客户端

❌ **手动更新**
- 不会自动更新
- 需要手动维护

## 五、写入画布是什么？为什么重要？

### 写入画布 = AI 直接在 Figma 里画图

没有写入画布时：
```
你：AI，帮我设计一个登录页
AI：好的，我描述一下：
    - 顶部一个 Logo
    - 中间两个输入框
    - 底部一个按钮
    - ...
你：（自己在 Figma 里画，花了 30 分钟）
```

有了写入画布时：
```
你：AI，帮我在 Figma 里设计一个登录页
AI：（直接在 Figma 里创建图层、组件、样式...）
你：（10 秒后看到完整的设计稿）
```

### 实际例子

**例子 1：创建组件**
```
在 Figma 里创建一个按钮组件：
- 主按钮：蓝色背景，白色文字
- 次要按钮：白色背景，灰色边框
- 悬停状态：颜色加深 10%
```

**例子 2：重构设计**
```
把这个登录页改成暗黑模式：
- 背景改成 #141414
- 文字改成 #FFFFFF
- 输入框背景改成 #1F1F1F
```

**例子 3：从代码生成设计**
```
把这个 React 组件转成 Figma 设计稿：
<Button variant="primary">Click me</Button>
```

## 六、网页抓图是什么？为什么重要？

### 网页抓图 = 把浏览器里的网页变成 Figma 设计稿

没有网页抓图时：
```
你：（看到一个网页想参考）
你：（截图 → 导入 Figma → 手动临摹 → 花了 1 小时）
```

有了网页抓图时：
```
你：把 localhost:3000 的页面抓到 Figma 里
AI：（打开浏览器 → 捕获页面 → 自动生成 Figma 图层）
你：（30 秒后看到可编辑的 Figma 设计稿）
```

### 实际例子

**例子 1：参考竞品**
```
把 https://competitor.com 的首页抓到 Figma 里，
我想分析一下他们的布局。
```

**例子 2：还原现有页面**
```
把我们线上的登录页抓到 Figma 里，
然后更新一下设计。
```

**例子 3：多状态捕获**
```
打开 https://my-app.com，然后：
1. 捕获首页状态
2. 点击登录按钮，捕获登录表单
3. 输入错误密码，捕获错误状态
```

## 七、谁应该用哪个？

### ✅ 用 Remote MCP，如果你是：

- **个人开发者**：配置简单，功能全
- **前端开发**：需要写入画布、网页抓图
- **快速迭代**：想要最新功能
- **多设备工作**：公司家里都要用
- **大多数人**：90% 的场景都适合

### ✅ 用 Desktop MCP，如果你是：

- **企业开发**：有严格的安全合规要求
- **内网环境**：不能访问外网
- **隐私优先**：数据绝对不能离开本地
- **特定场景**：只需要读取设计稿，不需要其他功能

## 八、能不能同时用？

**可以！** 你可以同时配置两个，根据需要切换。

配置示例：

```json
{
  "mcp": {
    "servers": {
      "figma-remote": {
        "command": ["..."],
        "enabled": true
      },
      "figma-desktop": {
        "command": ["..."],
        "enabled": false
      }
    }
  }
}
```

平时用 `figma-remote`，需要时切换到 `figma-desktop`。

## 九、常见问题

### Q: Remote MCP 安全吗？

A: 安全！Figma 有完善的隐私保护：
- 数据加密传输
- 符合 GDPR、CCPA 等法规
- 不会滥用你的数据

### Q: Desktop MCP 能解锁 Remote 的功能吗？

A: 不能！写入画布和网页抓图只在 Remote MCP 可用。

### Q: 我可以先试试 Remote，不行再换 Desktop 吗？

A: 当然可以！配置很简单，随时可以切换。

### Q: 免费版能用 Remote MCP 吗？

A: 可以，但需要 Dev 或 Full seat（付费计划）。
