## 一、先搞懂：MCP 是什么？

### MCP = Model Context Protocol

中文叫 **模型上下文协议**。

简单理解：**MCP 就是 AI 和各种工具之间的"通用翻译官"**。

没有 MCP 时：
- AI 只会说"AI 语言"
- 你的 Figma 只会说"Figma 语言"
- 你的数据库只会说"数据库语言"
- 结果就是：它们互相听不懂！

有了 MCP 后：
- AI 说："帮我看看这个设计稿"
- MCP 翻译给 Figma："把那个设计稿的信息给我"
- Figma 把数据给 MCP
- MCP 翻译成 AI 能懂的格式
- AI 说："好的，我来生成代码！"

---

## 二、那 Figma MCP 又是什么？

**Figma MCP** 就是专门让 AI 和 Figma 对话的翻译官！

### 它能做什么？

| 功能 | 说明 |
|-----|------|
| 👀 **读取设计稿** | 让 AI"看"懂 Figma 的图层、颜色、间距... |
| 💻 **生成代码** | 从设计稿直接生成 React/Vue/HTML 代码 |
| ✏️ **写入画布** | 让 AI 直接在 Figma 里画图、改设计 |
| 🌐 **网页抓图** | 把浏览器里的网页抓成 Figma 设计稿 |
| 🧩 **Code Connect** | 用你项目里的组件生成代码 |

## 五、两种连接方式

Figma 提供两种 MCP 服务器，你可以根据需要选择：

### 1. Remote MCP（推荐 ⭐⭐⭐⭐⭐）

- **是什么**：Figma 官方托管的服务器
- **优点**：
  - 配置最简单（一行命令搞定）
  - 功能最全（支持写入画布、网页抓图）
  - 官方维护，更新及时
- **缺点**：需要联网

### 2. Desktop MCP（备用）

- **是什么**：在你本地 Figma 桌面端运行
- **优点**：
  - 完全本地，数据不离开你的电脑
  - 适合企业内部使用
- **缺点**：
  - 配置复杂一点
  - 功能较少（不支持写入画布、网页抓图）

## 七、什么时候用 Figma MCP？

### ✅ 适合用的场景

1. **前端开发**：从设计稿生成代码
2. **UI 还原**：确保 1:1 还原设计稿
3. **设计迭代**：快速把设计更新反映到代码
4. **组件库建设**：让 AI 用你现有的组件
5. **设计系统维护**：保持设计和代码一致

### ❌ 不需要用的场景

1. **只是想看看设计稿**：直接用 Figma 就好
2. **简单的页面**：手动写更快
3. **完全从零开始**：不需要参考设计稿

## 八、常见问题

### Q: Figma MCP 要钱吗？

A: Beta 期间完全免费！以后可能会变成按使用量付费，但现在赶紧用！

### Q: 免费版 Figma 能用吗？

A: 可以，但需要 Dev 或 Full seat（付费计划）。免费版有使用限制。

### Q: 我的设计稿数据安全吗？

A:
- Remote MCP：通过 Figma 官方服务器传输
- Desktop MCP：完全在本地，数据不离开你的电脑

### Q: 只能用 Claude Code 吗？

A: 不是！支持 Cursor、VS Code、Codex、Warp 等好多工具。
