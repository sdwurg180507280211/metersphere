设计完成了，现在要让全世界看到！这篇文章带你完整走一遍 Figma Site 发布流程：预览、发布、更新、自定义域名... 全都有！

## 一、先搞懂：发布流程概览

### 简单理解

**发布 = 把你的设计变成一个可以访问的网址！**

没有 Figma Sites 时：
```
你：设计完成了
    ↓
你：找开发者写代码
    ↓
开发者：写代码（1-2周）
    ↓
你：买服务器、配置域名
    ↓
你：部署上线
    ↓
网站：终于上线了！（半个月过去了）
```

有了 Figma Sites 时：
```
你：设计完成了
    ↓
你：点击"Publish"按钮
    ↓
网站：上线了！（几秒钟）
```

### 发布后你会得到什么？

| 内容 | 说明 |
|-----|------|
| **免费网址** | `your-name.figma.site` |
| **HTTPS** | 自动配置，安全访问 |
| **CDN** | 全球加速，访问快 |
| **SEO 友好** | 搜索引擎可以收录 |

## 三、正式发布：点击一下就上线！

### 步骤 1：最后一次预览

1. 点击右上角的 **"Preview"** 按钮
2. 在打开的浏览器标签页里，完整浏览一遍
3. 点击所有链接，测试所有交互
4. 切换不同设备视图检查

### 步骤 2：点击 Publish

确认没问题后：

1. 回到 Figma Sites 编辑器
2. 点击右上角的 **"Publish"** 按钮
3. 会弹出一个确认对话框

### 步骤 3：配置发布选项

在发布对话框中，你可以设置：

#### Site Name（网站名称）

这会出现在浏览器标签页上，也会影响默认 URL。

**好的例子：**
- "小明的作品集"
- "ABC 公司官网"
- "产品着陆页"

#### URL（网址）

默认会生成一个：
```
你的名字.figma.site
```

你可以修改前面的部分：
```
xiaoming.figma.site
abccorp.figma.site
product-landing.figma.site
```

**注意：**
- 只能用字母、数字、连字符（-）
- 不能用空格、特殊字符
- 不能和别人重复

#### Description（描述）

这会用于 SEO，显示在搜索结果里。

**好的描述例子：**
> "小明的产品设计作品集，展示 UI/UX 设计作品"
> "ABC 公司官网，提供企业级解决方案"
> "产品 X 着陆页，了解更多产品信息"

### 步骤 4：确认发布

1. 检查所有设置
2. 点击 **"Publish"** 确认
3. 等待几秒钟...
4. 完成！🎉

### 步骤 5：查看你的网站

发布成功后，会显示你的网站 URL：
```
https://your-name.figma.site
```

点击链接，就能看到你的网站了！

## 四、更新网站：重新发布就好！

### 想要修改网站怎么办？

很简单！修改 → 预览 → 重新发布！

#### 步骤 1：修改内容

在 Figma Sites 编辑器中：
1. 修改文字、图片、布局...
2. 想改什么就改什么

#### 步骤 2：预览

1. 点击 **"Preview"**
2. 检查修改效果
3. 确认没问题

#### 步骤 3：重新发布

1. 点击 **"Publish"**
2. 这时候按钮可能显示为 **"Republish"** 或 **"Update"**
3. 确认发布
4. 几秒钟后，更新就上线了！

### 版本历史

Figma Sites 会保留发布历史：

1. 点击 **"Publish"** 按钮旁边的箭头
2. 选择 **"Version history"**
3. 可以看到之前的所有版本
4. 可以恢复到任意版本！

### 不想立即发布？保存草稿！

如果你还没准备好发布：
- Figma Sites 会自动保存你的修改
- 不用手动保存
- 下次打开继续编辑
- 准备好了再发布

## 五、自定义域名：用你自己的网址！

### 为什么要用自定义域名？

| 对比 | Figma 子域名 | 自定义域名 |
|-----|-------------|-----------|
| URL | `your-name.figma.site` | `www.your-domain.com` |
| 专业度 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 品牌感 | 一般 | 很强 |
| 费用 | 免费 | 域名费用（通常几十元/年） |

### 前提条件

使用自定义域名需要：
1. ✅ Figma Sites 付费版（Professional 或 Enterprise）
2. ✅ 你有一个域名（可以在阿里云、腾讯云、Namecheap 等购买）

### 步骤 1：在 Figma Sites 中添加域名

1. 打开你的 Figma Site
2. 点击右上角的 **"Share"** 或 **"Settings"**
3. 找到 **"Custom domain"** 部分
4. 点击 **"Add domain"**
5. 输入你的域名，例如：`www.your-domain.com`

### 步骤 2：配置 DNS

添加域名后，Figma Sites 会给你 DNS 配置信息，类似：

| 类型 | 主机记录 | 值 |
|-----|---------|----|
| CNAME | www | `your-name.figma.site` |
| 或者 A | @ | `192.0.2.1` |

你需要去域名提供商那里配置：

**以阿里云为例：**
1. 登录阿里云控制台
2. 进入"域名" → "解析"
3. 添加记录，填写 Figma 给你的信息
4. 保存

**以 Namecheap 为例：**
1. 登录 Namecheap
2. 进入 "Manage" → "Advanced DNS"
3. 添加记录
4. 保存

### 步骤 3：验证域名

配置完 DNS 后：
1. 回到 Figma Sites
2. 点击 **"Verify"** 验证域名
3. 等待 DNS 生效（可能需要几分钟到几小时）
4. 验证成功后，HTTPS 会自动配置

### 步骤 4：完成！

现在访问 `www.your-domain.com` 就能看到你的网站了！

## 六、网站设置：SEO、隐私、更多

### SEO 设置

让搜索引擎更容易发现你的网站：

#### 1. Title（标题）

浏览器标签页显示的标题，也是搜索结果的标题。

**好的标题：**
```
小明的设计作品集 | UI/UX Designer
ABC 公司官网 | 企业解决方案
产品 X - 让工作更高效
```

**设置位置：**
Settings → SEO → Site title

#### 2. Meta Description（描述）

搜索结果中显示的描述。

**好的描述：**
```
小明的产品设计作品集，展示 UI/UX 设计作品，欢迎合作。
ABC 公司提供企业级解决方案，帮助客户实现数字化转型。
产品 X 是一款高效的协作工具，让团队工作更顺畅。
```

**设置位置：**
Settings → SEO → Meta description

#### 3. Social Image（社交图片）

分享到微信、Twitter、Facebook 时显示的图片。

**建议：**
- 尺寸：1200 x 630px
- 包含品牌、标题
- 简洁醒目

**设置位置：**
Settings → SEO → Social image

### 隐私设置

#### 1. Search Indexing（搜索索引）

是否允许搜索引擎收录：

| 设置 | 效果 |
|-----|------|
| **允许** | 搜索引擎可以收录，别人能搜索到 |
| **不允许** | 搜索引擎不收录，只能通过直接链接访问 |

**什么时候不允许？**
- 测试网站
- 内部网站
- 还没准备好公开

**设置位置：**
Settings → Privacy → Allow search indexing

#### 2. Password Protection（密码保护）

付费版功能，给网站加密码：

1. 启用密码保护
2. 设置密码
3. 访问者需要输入密码才能看

**适用场景：**
- 客户预览
- 内部项目
- 还没准备好公开

### 其他设置

#### Favicon（网站图标）

浏览器标签页上的小图标：

- 尺寸：32x32px 或 64x64px
- 格式：PNG、ICO
- 建议：用品牌 Logo

**设置位置：**
Settings → General → Favicon

## 七、网站分析：看看谁在访问

### Figma Sites 内置统计

免费版和付费版都有基础统计：

| 数据 | 说明 |
|-----|------|
| **页面浏览量** | 网站被看了多少次 |
| **独立访客** | 有多少不同的人访问 |
| **热门页面** | 哪些页面最受欢迎 |
| **来源** | 用户从哪来的（搜索引擎、直接访问等） |

**查看位置：**
Dashboard → Analytics

### 集成 Google Analytics（付费版）

如果你需要更详细的分析，可以集成 Google Analytics：

#### 步骤 1：获取 GA ID

1. 去 https://analytics.google.com/
2. 创建账号（如果没有）
3. 创建媒体资源
4. 获取 Measurement ID（格式：G-XXXXXXXXXX）

#### 步骤 2：在 Figma Sites 中添加

1. 打开 Settings
2. 找到 Integrations
3. 找到 Google Analytics
4. 输入你的 GA ID
5. 保存

#### 步骤 3：完成！

现在可以在 Google Analytics 中看到详细数据了！

## 八、常见问题

### Q: 发布后多久能看到？

A: 通常几秒钟！最多 1-2 分钟。

### Q: 可以取消发布吗？

A: 可以！在 Settings 中找到 "Unpublish"，点击后网站就不能访问了。

### Q: 修改后重新发布，旧版本还在吗？

A: 在版本历史里还能看到，也可以恢复！

### Q: 免费版可以用自定义域名吗？

A: 不行，自定义域名是付费版功能。

### Q: 可以把网站导出吗？

A: 目前 Figma Sites 不支持导出 HTML/CSS，但你可以用 Code Layer 添加自定义代码。

### Q: 网站备份怎么办？

A: Figma Sites 会自动保存，还有版本历史，你也可以手动复制重要内容。

### Q: 访问速度怎么样？

A: 很快！Figma Sites 用 CDN 全球加速，世界各地访问都很快。

### Q: 网站会挂吗？

A: Figma Sites 托管在 Figma 的基础设施上，可用性很高！

## 十、最佳实践

### ✅ 应该做的

1. **发布前充分测试**
   - Preview 里看一遍
   - 真实浏览器试一遍
   - 手机上看一遍

2. **从小步迭代开始**
   - 先发布第一个版本
   - 有问题再改
   - 不要追求完美

3. **用好版本历史**
   - 大改动前，先记一下当前版本
   - 改坏了可以恢复
   - 不用担心

4. **配置好 SEO**
   - 写好标题和描述
   - 加好社交图片
   - 让搜索引擎更容易找到

5. **监控访问数据**
   - 定期看 Analytics
   - 了解用户行为
   - 持续优化

### ❌ 不应该做的

1. **不要直接发布未经测试的版本**
   - 一定要先 Preview
   - 确认没问题再发布
   - 不然用户会看到有问题的网站

2. **不要怕更新**
   - 有问题就改
   - 改完重新发布
   - 很简单的

3. **不要忽略 SEO**
   - 网站做得好
   - 也要让别人能找到
   - SEO 很重要

4. **不要忘记监控**
   - 发布不是结束
   - 看看谁在访问
   - 持续改进
