# 🔥 Figma Sites 响应式设计：Breakpoints、Blocks 和预览

在 Figma Sites 里做网站，最爽的就是响应式设计超级简单！这篇文章带你搞懂三个核心概念：Breakpoints（断点）、Blocks（模块）和 Preview（预览）。

---

## 一、先搞懂：响应式设计是什么？

### 简单理解

**响应式设计 = 同一个网站，在不同设备上都好看！**

没有响应式时：
```
手机打开你的网站：
- 字太小看不清
- 按钮点不到
- 需要左右滑动
- 用户：（直接关掉）
```

有了响应式时：
```
手机打开你的网站：
- 字大小刚好
- 按钮好点击
- 自动调整布局
- 用户：（体验太棒了！）
```

---

## 二、Breakpoints（断点）：让网站适配不同设备

### 什么是 Breakpoints？

**Breakpoints = 决定在什么屏幕宽度下切换布局的"分界点"！**

就像换衣服：
- 天气热 → 穿短袖（小屏幕）
- 天气凉 → 穿外套（中等屏幕）
- 天气冷 → 穿羽绒服（大屏幕）

### Figma Sites 的三个默认断点

| 断点名称 | 设备类型 | 屏幕宽度 |
|---------|---------|---------|
| **Mobile** | 手机 | < 768px |
| **Tablet** | 平板 | 768px - 1024px |
| **Desktop** | 桌面 | > 1024px |

### 怎么用 Breakpoints？

#### 步骤 1：切换断点

在 Figma Sites 顶部，你会看到三个按钮：
- 📱 Mobile
- 📱 Tablet
- 🖥️ Desktop

点击不同的按钮，就能在不同视图之间切换！

#### 步骤 2：为每个断点单独设计

**重要概念：每个断点可以有不同的布局！**

例子：

**Desktop（桌面端）：**
```
[图片] [文字] [图片] [文字]
```

**Tablet（平板端）：**
```
[图片] [文字]
[图片] [文字]
```

**Mobile（手机端）：**
```
[图片]
[文字]
[图片]
[文字]
```

#### 步骤 3：断点继承机制

Figma Sites 的断点有个超酷的特性：**继承**！

```
Desktop（基础）
    ↓
Tablet（继承 Desktop，只改不同的）
    ↓
Mobile（继承 Tablet，只改不同的）
```

这意味着：
- 你在 Desktop 做的修改，会自动同步到 Tablet 和 Mobile
- 你在 Tablet 做的修改，会自动同步到 Mobile
- 你只需要改"不一样"的地方，不用重复劳动！

### 实战例子：导航栏

#### Desktop 视图
```
[Logo]  [首页] [产品] [关于] [联系]
```

#### Mobile 视图
```
[Logo]              [☰ 菜单]
```

**操作步骤：**
1. 在 Desktop 视图设计完整导航栏
2. 切换到 Mobile 视图
3. 隐藏不需要的链接
4. 添加汉堡菜单按钮
5. 完成！Tablet 视图可以根据需要调整

## 三、Blocks（块）：网站的积木

### 什么是 Blocks？

**Blocks = 构建网站的基本单元！**

就像搭乐高：
- 每块积木是一个 Block
- 把 Blocks 拼起来就是一个完整的网站

### Figma Sites 常用 Blocks

| Block 类型 | 用途 | 例子 |
|-----------|------|------|
| **Text** | 显示文字 | 标题、段落、列表 |
| **Image** | 显示图片 | 产品图、背景图 |
| **Button** | 可点击按钮 | "立即购买"、"了解更多" |
| **Video** | 嵌入视频 | YouTube、Vimeo |
| **Form** | 收集用户信息 | 联系表单、订阅表单 |
| **Section** | 分组容器 | 把相关内容放一起 |
| **Navigation** | 导航栏 | 网站顶部菜单 |
| **Footer** | 页脚 | 网站底部信息 |
| **Code Layer** | 自定义代码 | 用 AI 加高级功能 |

### 怎么用 Blocks？

#### 步骤 1：添加 Block

1. 点击左侧的 "Add" 按钮
2. 选择你想要的 Block 类型
3. 拖到画布上！

#### 步骤 2：编辑 Block

每个 Block 都有自己的设置：

**Text Block：**
- 修改文字内容
- 选择字体、字号
- 设置颜色、对齐方式

**Image Block：**
- 上传图片
- 调整尺寸
- 设置圆角、阴影

**Button Block：**
- 修改按钮文字
- 设置链接地址
- 调整样式（颜色、圆角等）

#### 步骤 3：排列 Blocks

**水平排列：**
```
[Block 1] [Block 2] [Block 3]
```

**垂直排列：**
```
[Block 1]
[Block 2]
[Block 3]
```

**网格排列：**
```
[Block 1] [Block 2]
[Block 3] [Block 4]
```

### 技巧：Section Block 是你的好朋友

**Section Block = 把其他 Blocks 打包的"盒子"！**

为什么要用 Section？
1. **组织内容**：把相关内容放一起
2. **统一背景**：给一组内容设置相同背景
3. **批量调整**：移动 Section 就移动了里面所有内容

**例子：**
```
[Section: 英雄区域]
  ├── [标题]
  ├── [副标题]
  └── [按钮]

[Section: 产品展示]
  ├── [产品图 1]
  ├── [产品图 2]
  └── [产品图 3]
```

## 四、Preview（预览）：看看网站真实效果

### 什么是 Preview？

**Preview = 在浏览器里看看网站实际长什么样！**

在 Figma Sites 编辑器里看：
- 能看到所有辅助线
- 能看到选中状态
- 可能和真实效果有点区别

在 Preview 里看：
- 就是用户看到的样子
- 能测试交互
- 能测试响应式

### 怎么用 Preview？

#### 步骤 1：打开 Preview

点击右上角的 **"Preview"** 按钮！

会打开一个新的浏览器标签页，显示你的网站。

#### 步骤 2：测试响应式

在 Preview 页面顶部，你会看到设备切换按钮：

| 设备 | 说明 |
|-----|------|
| 📱 Mobile | 手机视图 |
| 📱 Tablet | 平板视图 |
| 🖥️ Desktop | 桌面视图 |
| 🔄 Rotate | 旋转屏幕（横屏/竖屏）|

点击不同的设备，看看你的网站在各种屏幕上的效果！

#### 步骤 3：测试交互

在 Preview 里你可以：
- 点击按钮，看看链接是否正常
- 填写表单，看看提交是否正常
- 滚动页面，看看动画是否正常
- 悬停元素，看看交互效果

### 预览小技巧

#### 技巧 1：用真实浏览器大小

如果你想看看在你自己的电脑上的效果：
1. 打开 Preview
2. 调整浏览器窗口大小
3. 网站会自动响应！

#### 技巧 2：分享预览链接

Preview 页面的 URL 可以分享给别人：
- 复制浏览器地址栏的链接
- 发给朋友或同事
- 他们可以直接看到你的网站（不需要 Figma 账号！）

#### 技巧 3：检查可访问性

Preview 模式下，你可以：
- 用 Tab 键导航，看看焦点是否清晰
- 放大文字，看看可读性
- 测试键盘操作是否完整

## 五、完整工作流示例

让我们用一个完整的例子，把这三个概念串起来！

### 任务：做一个产品着陆页

#### 阶段 1：在 Desktop 视图设计

1. 添加 Navigation Block（导航栏）
2. 添加 Section Block（英雄区域）
   - 加标题
   - 加副标题
   - 加按钮
3. 添加 Section Block（产品特性）
   - 三个特性卡片，水平排列
4. 添加 Section Block（用户评价）
5. 添加 Footer Block（页脚）

#### 阶段 2：切换到 Tablet 视图

1. 检查产品特性卡片
   - 如果三个太挤，改成两个一行
2. 调整文字大小
   - 如果字号太大，改小一点
3. 检查间距
   - 确保留白合适

#### 阶段 3：切换到 Mobile 视图

1. 修改导航栏
   - 隐藏链接，改成汉堡菜单
2. 重新排列产品特性
   - 改成垂直排列（一个占一行）
3. 调整按钮大小
   - 改成全宽，方便点击
4. 调整所有文字
   - 确保在手机上清晰可读

#### 阶段 4：预览和测试

1. 点击 Preview
2. 依次查看三个断点
3. 测试所有按钮和链接
4. 调整浏览器窗口大小
5. 分享给朋友看看

#### 阶段 5：迭代优化

根据预览反馈：
- 如果某个断点效果不好，回去调整
- 如果交互有问题，修复一下
- 满意后就可以发布了！

## 六、常见问题

### Q: 我需要为每个断点重新设计一遍吗？

A: 不需要！因为有继承机制：
- Desktop 是基础
- Tablet 继承 Desktop
- Mobile 继承 Tablet
- 你只需要改"不一样"的地方

### Q: 可以自定义断点吗？

A: 目前 Figma Sites 用的是三个标准断点（Mobile/Tablet/Desktop），这三个已经能覆盖 99% 的场景了！

### Q: 预览链接别人能编辑我的网站吗？

A: 不能！预览链接是只读的，别人只能看，不能改。

### Q: 可以在 Preview 里测试 Code Layer 吗？

A: 可以！Preview 会加载所有 Code Layer，你可以测试自定义功能。

## 七、最佳实践

### ✅ 应该做的

1. **从 Desktop 开始**
   - 先设计桌面端
   - 再往小屏幕适配

2. **用 Section 组织内容**
   - 每个部分用 Section 包起来
   - 调整起来更方便

3. **经常预览**
   - 改一点就预览一下
   - 不要等全部做完才预览

4. **测试真实设备**
   - 如果有手机，在真实手机上看看
   - 模拟器和真机可能有区别

### ❌ 不应该做的

1. **不要先设计 Mobile**
   - Desktop 到 Mobile 更容易
   - 反过来比较麻烦

2. **不要用太多 Block**
   - 保持结构简单
   - 太多 Block 会难维护

3. **不要忽略预览**
   - 编辑器里看着好 ≠ 实际效果好
   - 一定要在 Preview 里验证

## 八、总结

这三个核心概念：

1. **Breakpoints** - 让网站在不同设备上都好看
2. **Blocks** - 搭网站的积木
3. **Preview** - 看看真实效果

掌握这三个，你就能在 Figma Sites 里做出专业的响应式网站了！
