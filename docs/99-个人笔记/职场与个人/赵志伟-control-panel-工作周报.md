## 一、项目整体定位

`metersphere-control-panel` 是围绕 MeterSphere 多模块研发场景建设的本地控制台工具。MeterSphere 主项目模块多、启动链路长、前后端构建步骤分散，日常开发中容易遇到以下问题：

- 多个 Spring Boot 服务启动顺序、端口、健康检查和进程状态难以统一管理；
- 前端模块数量较多，SDK 依赖、业务模块依赖和批量构建顺序需要人工判断；
- 打包脚本、镜像版本、构建日志和失败原因分散在终端中，排查成本高；
- 本地配置、Redis、SSH 隧道、JVM 参数、Node/npm 路径等环境差异会影响构建和启动稳定性；
- 服务日志、命令日志、构建日志缺少统一的实时查看和历史追踪入口。

该控制台的核心目标是把 MeterSphere 的“启动、构建、打包、诊断、日志”流程产品化，降低本地开发和联调成本。
