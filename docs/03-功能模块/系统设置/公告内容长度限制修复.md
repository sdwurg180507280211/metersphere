# 公告内容长度限制修复

## 问题描述

在系统参数设置-公告设置页面，当输入过长的中文字符时会报错，无法保存。

## 问题根因

### 数据库层面
- `system_parameter` 表的 `param_value` 字段类型为 `varchar(255)`
- 在 UTF-8 编码下，一个中文字符占用 3-4 字节
- 255 字节只能存储约 **85 个中文字符**
- 当前公告内容已有 191 个字符（429 字节），远超限制

### 前端层面
- `AnnouncementSetting.vue` 组件中 `el-input` 设置了 `maxlength="500"`
- 虽然前端限制为 500 字符，但数据库无法存储超过 85 个中文字符的内容

## 解决方案

### 1. 数据库迁移脚本

创建迁移脚本 `V148__2.10.23_fix_system_parameter_length.sql`：

```sql
SET SESSION innodb_lock_wait_timeout = 7200;

-- 修复 system_parameter 表 param_value 字段长度限制
-- 将 param_value 字段从 varchar(255) 改为 text 类型
-- text 类型支持最多 65535 字节（约 21845 个中文字符）

ALTER TABLE system_parameter 
MODIFY COLUMN param_value TEXT COMMENT '参数值';

SET SESSION innodb_lock_wait_timeout = DEFAULT;
```

**修改说明**：
- **修改前**：`varchar(255)` - 最多 255 字节（约 85 个中文字符）
- **修改后**：`text` - 最多 65535 字节（约 21845 个中文字符）
- **影响范围**：所有系统参数，不仅限于公告内容

### 2. 前端组件修改

修改 `system-setting/frontend/src/business/system/setting/AnnouncementSetting.vue`：

```vue
<!-- 修改前 -->
<el-input
  type="textarea"
  v-model="form.content"
  :rows="4"
  :placeholder="$t('announcement.content_placeholder')"
  maxlength="500"
  show-word-limit
/>

<!-- 修改后 -->
<el-input
  type="textarea"
  v-model="form.content"
  :rows="4"
  :placeholder="$t('announcement.content_placeholder')"
  maxlength="2000"
  show-word-limit
/>
```

**修改说明**：
- **修改前**：`maxlength="500"` - 前端限制 500 字符
- **修改后**：`maxlength="2000"` - 前端限制 2000 字符
- **设计考虑**：虽然数据库支持 21845 个中文字符，但考虑到公告栏的实际使用场景，2000 字符已经足够

## 注意事项

1. **数据库迁移**：
   - 迁移脚本会自动执行，无需手动干预
   - `varchar(255)` 改为 `text` 是安全的，不会丢失数据
   - 迁移过程中会锁表，但时间很短（毫秒级）

2. **前端限制**：
   - 前端限制为 2000 字符是为了避免公告内容过长影响用户体验
   - 如果需要更长的内容，可以调整 `maxlength` 值
   - 建议不要超过 5000 字符，否则公告栏显示效果会很差

3. **其他系统参数**：
   - 此次修改影响所有系统参数，不仅限于公告内容
   - 如果其他系统参数也需要存储长文本，现在可以支持了
