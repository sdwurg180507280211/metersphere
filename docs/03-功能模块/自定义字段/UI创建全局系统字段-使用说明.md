# UI创建全局系统字段 - 使用说明

## 📋 功能概述

现在可以通过UI界面创建全局系统字段，无需编写SQL脚本和修改代码！

### ✅ 核心特性

| 特性 | 说明 |
|------|------|
| **权限控制** | 只有系统管理员可以创建全局系统字段 |
| **自动关联** | 自动关联到所有全局模板和项目级模板 |
| **删除保护** | 系统字段不能删除（按钮自动禁用） |
| **半自动国际化** | 自动生成i18n key，但不填充翻译（显示为空） |
| **即时生效** | 创建后立即在所有项目可见 |

---

## 🚀 使用流程

### 步骤1：登录系统

使用**系统管理员**账号登录 MeterSphere

⚠️ 只有系统管理员才能看到"全局系统字段"选项

---

### 步骤2：进入自定义字段管理

```
路径：项目设置 → 模版管理 → 自定义字段
```

---

### 步骤3：创建字段

点击"创建"按钮，填写字段信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| **字段名称** | 中文名称 | 手机 |
| **字段备注** | 说明信息（可选） | 联系人手机号 |
| **使用场景** | TEST_CASE/ISSUE/API | ISSUE（缺陷） |
| **字段类型** | input/select/date等 | input（文本框） |
| **字段范围** | 项目字段 / 全局系统字段 | ⭐ **全局系统字段** |

---

### 步骤4：选择"全局系统字段"

在"字段范围"选项中，选择：

```
◉ 全局系统字段（所有项目可见，不可删除）
```

⚠️ 会显示警告提示：
```
⚠️ 全局系统字段将自动关联到所有项目，并且无法删除，请谨慎创建！
```

---

### 步骤5：确认创建

点击"确定"按钮

系统会自动：
1. ✅ 创建全局系统字段
2. ✅ 关联到 Local、Jira、Zentao、TAPD、Azure Devops 全局模板
3. ✅ 关联到所有项目级模板
4. ✅ 生成 i18n key（存储在备注字段）

---

### 步骤6：验证效果

切换到任意项目 → 缺陷管理 → 创建缺陷

应该能看到刚创建的字段！

---

## ⚠️ 注意事项

### 1. 权限限制

❌ **非系统管理员**：看不到"字段范围"选项，只能创建项目字段

✅ **系统管理员**：可以创建全局系统字段

---

### 2. 删除保护

全局系统字段**不能删除**：

```
字段列表：
┌─────┬──────┬────────┬──────┬────────┐
│ 名称│ 场景 │ 字段类型│系统字段│ 操作  │
├─────┼──────┼────────┼──────┼────────┤
│ 手机│ 缺陷 │ 文本框  │ 是   │ [编辑] │  ← 没有删除按钮
│ 地址│ 缺陷 │ 文本框  │ 否   │ [编辑][删除] │  ← 有删除按钮
└─────┴──────┴────────┴──────┴────────┘
```

---

### 3. 半自动国际化

**创建时**：
- 系统自动生成 i18n key
- 格式：`test_track.issue_custom_时间戳`
- 存储在"字段备注"中

**显示效果**：
- 未配置翻译时：显示为**空**（不是中文字段名）
- 需要手动在 i18n 文件中添加翻译

**举例**：

```javascript
// 创建"手机"字段时
// 自动生成 i18n key: test_track.issue_custom_123456

// 未配置翻译前：
显示: ""（空白）

// 配置翻译后（需要手动添加到 zh-CN.js）：
test_track: {
  issue: {
    issue_custom_123456: "手机"
  }
}

显示: "手机"
```

---

### 4. 字段显示顺序

新创建的全局字段默认显示在**最后**（order=100）

如需调整显示顺序：
```sql
-- 修改显示顺序
UPDATE custom_field_template 
SET `order` = 3  -- 3表示显示在第3位
WHERE field_id = '字段ID';
```

---

### 5. 现有项目自动包含

✅ 所有现有项目**立即**可见新字段

✅ 新建项目**自动**包含新字段

## 🔧 故障排查

### 问题1：看不到"字段范围"选项

**原因**：当前用户不是系统管理员

**解决**：
1. 退出登录
2. 使用 admin 账号登录
3. 重新创建字段

---

### 问题2：创建后在项目中看不到字段

**原因1**：浏览器缓存

**解决**：清除浏览器缓存（Ctrl+Shift+Delete）

**原因2**：国际化未配置（显示为空）

**解决**：
```javascript
// 在 zh-CN.js 中添加翻译
test_track: {
  issue: {
    issue_custom_123456: "手机"  // 使用备注中的i18n key
  }
}
```

---

### 问题3：提示"权限不足"

**原因**：非系统管理员创建全局字段

**解决**：使用系统管理员账号

---

### 问题4：创建失败提示"字段已存在"

**原因**：同名全局系统字段已存在

**解决**：
1. 检查字段列表
2. 使用不同的字段名
3. 或编辑现有字段

在创建全局系统字段前，请确认：

- [ ] 是否真的需要全局字段？

## 💡 常见问题

### Q1: 普通用户能创建全局字段吗？

**A**: 不能。只有系统管理员（admin）可以创建全局系统字段。

---

### Q2: 创建后能修改吗？

**A**: 可以修改字段备注和选项值，但不能修改字段名称、类型和范围。

---

### Q3: 能删除全局系统字段吗？

**A**: 不能。系统字段不可删除，这是为了保护数据完整性。

---

### Q4: 国际化怎么处理？

**A**: 半自动处理：
1. 系统自动生成 i18n key
2. 显示时如果没有翻译，会显示为空
3. 需要手动在 zh-CN.js / en-US.js / zh-TW.js 中添加翻译

---

### Q5: 新建项目会自动包含新字段吗？

**A**: 会的！因为新项目创建时会基于全局模板，而全局模板已经关联了新字段。

---

### Q6: 能批量创建多个字段吗？

**A**: 暂不支持批量创建，需要逐个创建。
