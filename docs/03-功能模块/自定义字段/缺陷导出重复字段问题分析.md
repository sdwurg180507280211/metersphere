# 缺陷导出重复字段问题分析与修复

## 问题现象

缺陷导出时出现以下错误：

```
Duplicate key d392af07-fdfe-4475-a459-87d59f0b1626 
(attempted merging values 
  CustomFieldDao(required=true, order=15000, defaultValue="P0", ..., key=B) 
  and 
  CustomFieldDao(required=true, order=30000, defaultValue="P0", ..., key=E))
```

## 问题分析

### 1. 数据库层面

**问题根源**：`custom_field_template` 表中存在大量重复数据

查询发现，多个系统字段在多个模板中被重复关联：

```sql
SELECT 
    field_id,
    template_id,
    COUNT(*) as count,
    GROUP_CONCAT(`order` ORDER BY `order`) as orders,
    GROUP_CONCAT(`key` ORDER BY `order`) as key_list
FROM custom_field_template
GROUP BY field_id, template_id
HAVING COUNT(*) > 1;
```

### 2. 代码层面

**调用链路**：

```
IssuesController.downloadImportTemplate()
  └─> IssuesService.issueImportTemplate()
      └─> IssueTemplateService.getTemplate()
          └─> CustomFieldService.getCustomFieldByTemplateId()  <- 出错位置
```

**原代码问题**：

```java
public List<CustomFieldDao> getCustomFieldByTemplateId(String templateId) {
    List<CustomFieldTemplate> customFields = extCustomFieldTemplateMapper.getCustomFields(templateId);
    List<String> fieldIds = customFields.stream()
            .map(CustomFieldTemplate::getFieldId)
            .collect(Collectors.toList());

    List<CustomField> fields = getFieldByIds(fieldIds);
    Map<String, CustomField> fieldMap = fields.stream()
            .collect(Collectors.toMap(CustomField::getId, item -> item));  // <- 这里会抛出异常

    // ...
}
```

**问题原因**：
- `getCustomFields(templateId)` 返回的列表中，同一个 `field_id` 出现了多次
- 转换为 `Map<String, CustomField>` 时，`Collectors.toMap()` 不允许重复的 key
- 抛出 `IllegalStateException: Duplicate key`

**原去重逻辑的问题**：

```java
return result.stream().distinct().collect(Collectors.toList());
```

- `distinct()` 依赖对象的 `equals()` 和 `hashCode()` 方法
- `CustomFieldDao` 类可能没有正确实现这两个方法
- 即使实现了，也是在最后才去重，前面的 `toMap()` 已经抛出异常了

## 修复方案

### 方案一：清理数据库重复数据（推荐）

**优点**：
- 从根源解决问题
- 数据更干净，避免其他潜在问题
- 一次性解决所有重复数据

**步骤**：

1. 备份数据库（**必须！**）：
```bash
mysqldump -h 127.0.0.1 -u root -p metersphere_dev custom_field_template > backup_custom_field_template_$(date +%Y%m%d_%H%M%S).sql
```

2. 执行清理脚本：

**方式一：使用一键清理脚本（推荐）**
```bash
mysql -h 127.0.0.1 -u root -p metersphere_dev < docs/功能开发/自定义字段/执行清理重复字段.sql
```

**方式二：使用详细分析脚本（适合需要详细了解的情况）**
```bash

```

3. 验证清理结果：
```sql
SELECT field_id, template_id, COUNT(*) as count
FROM custom_field_template
GROUP BY field_id, template_id
HAVING COUNT(*) > 1;
```

如果返回空结果，说明清理成功。

**清理策略**：
- 保留每个 (field_id, template_id) 组合中的第一条记录
- 判断标准：
  - 如果 `order` 不为 NULL，保留 `order` 最小的记录
  - 如果 `order` 都为 NULL，保留 `id` 最小的记录

**预期清理结果**：
- 删除 14 条重复记录
- 保留 14 条有效记录（每个 field_id + template_id 组合保留一条）

### 方案二：修改代码增加容错（已实施）

**优点**：
- 不需要修改数据库
- 对历史数据兼容
- 防止未来再次出现类似问题

**修改内容**：

```java
public List<CustomFieldDao> getCustomFieldByTemplateId(String templateId) {
    List<CustomFieldTemplate> customFields = extCustomFieldTemplateMapper.getCustomFields(templateId);
    List<String> fieldIds = customFields.stream()
            .map(CustomFieldTemplate::getFieldId)
            .collect(Collectors.toList());

    List<CustomField> fields = getFieldByIds(fieldIds);
    Map<String, CustomField> fieldMap = fields.stream()
            .collect(Collectors.toMap(CustomField::getId, item -> item));

    // 使用 LinkedHashMap 去重，保留第一个出现的字段（order 最小的）
    // 目的：防止同一个 field_id 在同一个 template_id 中被关联多次导致的 Duplicate key 错误
    Map<String, CustomFieldDao> resultMap = new LinkedHashMap<>();
    if (CollectionUtils.isNotEmpty(customFields)) {
        customFields.forEach((item) -> {
            // 如果该字段已经存在，跳过（保留第一个，即 order 最小的）
            if (!resultMap.containsKey(item.getFieldId())) {
                CustomFieldDao customFieldDao = new CustomFieldDao();
                CustomField customField = fieldMap.get(item.getFieldId());
                BeanUtils.copyBean(customFieldDao, customField);
                BeanUtils.copyBean(customFieldDao, item);
                customFieldDao.setId(item.getFieldId());
                resultMap.put(item.getFieldId(), customFieldDao);
            }
        });
    }
    return new ArrayList<>(resultMap.values());
}
```

**关键改动**：
1. 使用 `LinkedHashMap` 替代 `ArrayList`，以 `field_id` 为 key
2. 在添加前检查 `containsKey()`，如果已存在则跳过
3. 由于 SQL 查询已经按 `order` 排序，第一个出现的就是 `order` 最小的
4. 最后转换为 `ArrayList` 返回

**为什么保留 order 最小的**：
- `order` 字段表示字段在表单中的显示顺序
- `order` 越小，字段越靠前，通常是更重要的字段
- 保留 `order` 最小的记录，符合业务逻辑

## 根本原因分析

### 为什么会产生重复数据？

可能的原因：

1. **模板复制功能**：复制模板时，可能没有正确处理字段关联
2. **数据迁移**：从旧版本升级时，迁移脚本可能有问题
3. **并发操作**：多个用户同时编辑模板，可能产生重复数据
4. **手动 SQL 操作**：运维人员手动执行 SQL 时，可能插入了重复数据

### 如何防止未来再次出现？

1. **数据库约束**：添加唯一索引
```sql
ALTER TABLE custom_field_template 
ADD UNIQUE INDEX uk_field_template (field_id, template_id);
```

2. **代码层面**：在插入前检查是否已存在
```java
// 在 CustomFieldTemplateService 中添加检查
public void addFieldToTemplate(CustomFieldTemplate fieldTemplate) {
    // 检查是否已存在
    CustomFieldTemplateExample example = new CustomFieldTemplateExample();
    example.createCriteria()
            .andFieldIdEqualTo(fieldTemplate.getFieldId())
            .andTemplateIdEqualTo(fieldTemplate.getTemplateId());
    long count = customFieldTemplateMapper.countByExample(example);
    if (count > 0) {
        throw new MSException("字段已关联到该模板");
    }
    customFieldTemplateMapper.insert(fieldTemplate);
}
```

3. **定期检查**：添加定时任务，定期检查并清理重复数据

## 总结

- **问题**：`custom_field_template` 表中存在重复数据，导致导出模板时出现 Duplicate key 错误
- **原因**：同一个 `field_id` 在同一个 `template_id` 中被关联了多次
- **修复**：修改 `CustomFieldService.getCustomFieldByTemplateId()` 方法，使用 `LinkedHashMap` 去重
- **建议**：清理数据库中的重复数据，并添加唯一索引防止未来再次出现
