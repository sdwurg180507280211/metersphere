## ⚠️ 为什么需要编译那么多模块？

### 模块依赖关系

```
test-track
  ├── 依赖 framework/sdk-parent/sdk
  ├── 依赖 framework/sdk-parent/domain
  ├── 依赖 framework/sdk-parent/xpack-interface
  └── 依赖 framework/sdk-parent/frontend（包含国际化）

framework/sdk-parent/sdk
  ├── 依赖 framework/sdk-parent/domain
  └── 依赖 framework/sdk-parent/xpack-interface
```

**如果只编译 frontend 和 test-track，会报错！** ❌

**错误示例**：
```
[ERROR] Failed to execute goal on project test-track: 
Could not resolve dependencies for project io.metersphere:test-track:jar:2.10
Missing: io.metersphere:sdk:jar:2.10
```

**解决方案**：必须按照依赖顺序编译 ✅
```
1. domain（最底层）
2. xpack-interface（底层）
3. sdk（中层，依赖domain和xpack-interface）
4. frontend（前端层）
5. test-track（业务层，依赖以上所有）
```

## 🐳 Docker部署打包（生产环境）

### 📦 正确的打包步骤（⚠️ 顺序很重要！）

```bash
# 设置环境变量
cd /Users/edy/ideaProjects/metersphere
imageVersion=v2.10.23.01-lts

# ============================================
# ✅ 推荐方案：完整打包（10-15分钟）
# ============================================

# 1. 初始化根pom
mvn install -N

# 2. ⭐ 编译框架和前端SDK（必须第一步！）
mvn clean install -pl framework/sdk-parent/frontend -DskipTests

# 3. ⭐ 打包 test-track（会自动构建frontend和backend）
cd test-track
mvn clean install -DskipTests
cd ..

# 重要：Maven会按照pom.xml中<modules>的顺序自动：
#   a. 先构建 test-track/frontend（生成 dist 目录）
#   b. 再构建 test-track/backend（自动复制 dist 到 resources/static）

# 4. 打包 project-management（同理）
cd project-management
mvn clean install -DskipTests
cd ..

# 5. 解压jar以便Docker构建（必须！）
mkdir -p test-track/backend/target/dependency
(cd test-track/backend/target/dependency && jar -xf ../*.jar)

mkdir -p project-management/backend/target/dependency
(cd project-management/backend/target/dependency && jar -xf ../*.jar)

# 6. 构建 Docker 镜像
docker build --build-arg MS_VERSION=${imageVersion} \
  -t registry.fit2cloud.com/north/test-track:${imageVersion} \
  -f test-track/Dockerfile .

docker build --build-arg MS_VERSION=${imageVersion} \
  -t registry.fit2cloud.com/north/project-management:${imageVersion} \
  -f project-management/Dockerfile .

# 7. 保存镜像（可选）
docker save registry.fit2cloud.com/north/test-track:${imageVersion} \
  -o test-track-${imageVersion}.tar
  
docker save registry.fit2cloud.com/north/project-management:${imageVersion} \
  -o project-management-${imageVersion}.tar
```

### ⚠️ 常见错误及解决方案

#### 错误1：`class path resource [static/index.html] cannot be opened`

**原因**：前端产物未正确复制到后端 JAR 包中

**解决方案**：
```bash
# ❌ 错误做法：单独打包backend
cd test-track/backend
mvn clean install  # 这样不会触发frontend构建！

# ✅ 正确做法：在父目录打包
cd test-track
mvn clean install  # Maven会按顺序构建frontend和backend
```

#### 错误2：frontend/dist 目录为空

**原因**：frontend-maven-plugin未执行或npm依赖未安装

**解决方案**：
```bash
# 检查frontend/dist是否存在
ls -la test-track/frontend/dist

# 如果为空，手动构建前端
cd test-track/frontend
npm install
npm run build
cd ../..

# 然后重新打包backend
cd test-track/backend
mvn clean install -DskipTests
```

#### 错误3：Docker构建失败

**原因**：未解压jar文件

**解决方案**：
```bash
# 必须先解压jar
mkdir -p test-track/backend/target/dependency
cd test-track/backend/target/dependency
jar -xf ../*.jar
cd ../../../..

# 然后再构建镜像
docker build -f test-track/Dockerfile .
```

## ⚠️ 打包注意事项

### 1. 前端构建时间
- 前端构建需要较长时间（5-10分钟）
- 首次构建需要下载 npm 依赖，时间更长

### 2. 确保 Node.js 版本
- 检查 `pom.xml` 中配置的 Node.js 版本
- 确保系统有足够的内存（建议 4GB+）

### 3. 清理缓存
如果遇到构建问题，可以清理：
```bash
# 清理 Maven 缓存
mvn clean

# 清理前端构建产物
rm -rf framework/sdk-parent/frontend/dist
rm -rf test-track/frontend/dist
rm -rf .node  # Node.js 安装目录
```

### 4. 验证前端构建
构建完成后，检查：
```bash
# 检查 SDK 前端构建产物
ls -la framework/sdk-parent/frontend/dist/

# 检查 test-track 前端构建产物
ls -la test-track/frontend/dist/
```
