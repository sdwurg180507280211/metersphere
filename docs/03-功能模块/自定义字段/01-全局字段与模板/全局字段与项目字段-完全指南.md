# MeterSphere 全局字段与项目字段完全指南

## 核心概念

### 1. 字段（Custom Field）

字段是表单中的输入项，有两个关键属性：

| 属性 | 说明 | 取值 |
|------|------|------|
| `global` | 是否全局字段 | `1`=全局字段，`0`=项目字段 |
| `system` | 是否系统字段 | `1`=系统字段，`0`=自定义字段 |
| `project_id` | 所属项目 | 全局字段为 `global` 或 `NULL`，项目字段为具体项目ID |

**字段类型组合**：

| global | system | project_id | 类型 | 说明 | 示例 |
|--------|--------|------------|------|------|------|
| 1 | 1 | `global` 或 `NULL` | 全局系统字段 | 系统预置，所有项目共享 | 状态、处理人、严重程度 |
| 1 | 0 | `NULL` | 全局自定义字段 | 管理员创建，所有项目可用 | 优先级、关闭时间 |
| 0 | 1 | 项目ID | 项目系统字段 | **不应该存在**（设计缺陷） | 测试项目的"状态"字段 |
| 0 | 0 | 项目ID | 项目自定义字段 | 项目内创建，仅本项目可用 | 项目特有字段 |

### 2. 模板（Issue Template）

模板定义了缺陷表单的字段组合和布局，也有两个关键属性：

| 属性 | 说明 | 取值 |
|------|------|------|
| `global` | 是否全局模板 | `1`=全局模板，`0`=项目模板 |
| `system` | 是否系统模板 | `1`=系统模板，`0`=自定义模板 |
| `project_id` | 所属项目 | 全局模板为 `global`，项目模板为具体项目ID |

**模板类型组合**：

| global | system | project_id | 类型 | 说明 | 示例 |
|--------|--------|------------|------|------|------|
| 1 | 1 | `global` | 全局系统模板 | 系统预置，所有项目可用 | default、JIRA-默认模版 |
| 1 | 0 | `global` | 全局自定义模板 | 管理员创建，所有项目可用 | - |
| 0 | 1 | 项目ID | 项目系统模板 | 项目创建时自动生成 | 测试项目的 default |
| 0 | 0 | 项目ID | 项目自定义模板 | 项目内创建，仅本项目可用 | - |

### 3. 模板字段关联（Custom Field Template）

通过 `custom_field_template` 表关联模板和字段：

```sql
CREATE TABLE custom_field_template (
  field_id VARCHAR(50),      -- 字段ID
  template_id VARCHAR(50),   -- 模板ID
  scene VARCHAR(30),         -- 场景（ISSUE/TEST_CASE）
  `order` INT                -- 显示顺序
);
```

**关键规则**：
- 全局模板可以绑定全局字段
- 项目模板可以绑定全局字段和项目字段
- **不应该**让全局模板绑定项目字段（会导致其他项目无法使用）

## 项目创建流程

### 1. 项目创建时发生了什么？

根据代码分析（`SystemProjectService.addProject()`），项目创建时：

```java
public Project addProject(AddProjectRequest project) {
    // 1. 保存项目基本信息
    saveProject(project);
    
    // 2. 为当前用户添加项目管理员角色
    // 3. 设置默认通知
    // 4. 创建默认版本
    addProjectVersion(project);
    
    // 5. 初始化项目应用配置
    initProjectApplication(project.getId());
    
    // 6. 发送Kafka消息，异步初始化项目节点
    kafkaTemplate.send(KafkaTopicConstants.PROJECT_CREATED_TOPIC, project.getId());
    
    // 7. 初始化项目资源池
    systemParameterService.saveProjectApplication(project.getId());
    
    return project;
}
```

**重要发现**：
- ❌ **没有创建项目模板**
- ❌ **没有创建项目字段**
- ❌ **没有设置项目的 `issue_template_id`**

### 2. 项目如何获得模板？

通过数据库查询发现，项目的 `issue_template_id` 有三种情况：

| 情况 | issue_template_id | 模板类型 | 说明 |
|------|-------------------|----------|------|
| 情况1 | 全局模板ID | 全局系统模板 | 直接使用全局模板（推荐） |
| 情况2 | 项目模板ID | 项目系统模板 | 使用项目专属模板 |
| 情况3 | NULL | 无模板 | 模板已被删除（异常） |

**实际数据示例**：

```sql
-- 个险系统：使用全局模板
| project_id | project_name | issue_template_id | template_name | template_global |
|------------|--------------|-------------------|---------------|-----------------|
| 1ad3f2bb   | 个险系统     | 5d7c87d2...       | default       | 1               |

-- 测试项目：使用项目模板
| project_id | project_name | issue_template_id | template_name | template_global |
|------------|--------------|-------------------|---------------|-----------------|
| 5d75b4c6   | 测试项目     | 319a76ca...       | default       | 0               |

-- 银保通项目：模板已删除
| project_id | project_name | issue_template_id | template_name | template_global |
|------------|--------------|-------------------|---------------|-----------------|
| 84d9cda2   | 银保通项目   | bd71a07a...       | NULL          | NULL            |
```

### 3. 项目模板是如何创建的？

**推测**：项目模板可能通过以下方式创建：

1. **前端手动创建**：
   - 用户在"项目设置 > 模板管理"中创建
   - 创建时可以选择复制全局模板

2. **Flyway数据库迁移**：
   - 系统升级时通过SQL脚本创建
   - 为已有项目批量创建模板

3. **XPack扩展功能**：
   - 企业版可能有自动创建项目模板的功能

**验证方法**：
```sql
-- 查询项目模板的创建时间
SELECT 
  it.id,
  it.name,
  it.project_id,
  p.name AS project_name,
  FROM_UNIXTIME(it.create_time/1000) AS template_create_time,
  FROM_UNIXTIME(p.create_time/1000) AS project_create_time,
  TIMESTAMPDIFF(SECOND, 
    FROM_UNIXTIME(p.create_time/1000), 
    FROM_UNIXTIME(it.create_time/1000)
  ) AS seconds_diff
FROM issue_template it
JOIN project p ON it.project_id = p.id
WHERE it.global = 0
ORDER BY it.create_time DESC;
```

## 字段和模板的使用规则

### 1. 推荐做法

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| 新建项目 | 使用全局模板 + 全局字段 | 简单、统一、易维护 |
| 项目特殊需求 | 创建项目自定义字段 | 不影响其他项目 |
| 全局字段不够用 | 创建全局自定义字段 | 所有项目都能用 |
| 需要不同表单布局 | 创建项目模板，绑定全局字段 | 灵活但统一 |

### 2. 不推荐做法

| 场景 | 问题 | 后果 |
|------|------|------|
| 创建项目系统字段 | 与全局系统字段重名 | 数据冲突、状态不一致 |
| 全局模板绑定项目字段 | 其他项目无法使用该模板 | 模板不可复用 |
| 删除正在使用的模板 | 项目的 `issue_template_id` 变成无效ID | 缺陷无法创建 |
| 删除正在使用的字段 | 历史数据丢失 | 数据完整性问题 |

### 3. 字段查找优先级

当缺陷表单需要显示字段时，系统的查找顺序：

1. 查询项目的 `issue_template_id`
2. 查询模板绑定的字段列表（`custom_field_template`）
3. 对于每个字段ID，查询字段定义（`custom_field`）
4. 如果字段是全局字段，所有项目都能访问
5. 如果字段是项目字段，只有该项目能访问

**示例**：

```sql
-- 查询"测试项目"缺陷表单的字段
SELECT 
  cf.name AS field_name,
  cf.global AS field_global,
  cf.system AS field_system,
  cf.project_id AS field_project_id,
  CASE 
    WHEN cf.global = 1 THEN '全局字段（所有项目可用）'
    WHEN cf.project_id = '5d75b4c6-a431-4743-b0b3-1876e36f2905' THEN '项目字段（仅本项目可用）'
    ELSE '其他项目字段（不可用）'
  END AS accessibility
FROM project p
JOIN issue_template it ON p.issue_template_id = it.id
JOIN custom_field_template cft ON it.id = cft.template_id
JOIN custom_field cf ON cft.field_id = cf.id
WHERE p.name = '测试项目' AND cf.scene = 'ISSUE'
ORDER BY cf.global DESC, cf.name;
```

## 常见问题

### Q1: 为什么有些项目有项目模板，有些没有？

**A**: 项目创建时**不会自动创建项目模板**，项目模板是通过以下方式产生的：
- 用户手动创建
- 系统升级时的数据迁移
- XPack扩展功能

大部分项目应该直接使用全局模板。

### Q2: 项目字段和全局字段有什么区别？

**A**: 

| 维度 | 全局字段 | 项目字段 |
|------|----------|----------|
| 可见范围 | 所有项目 | 仅所属项目 |
| 创建位置 | 系统设置 > 自定义字段 | 项目设置 > 自定义字段 |
| 适用场景 | 通用字段（状态、处理人） | 项目特有字段 |
| 数据隔离 | 共享定义，数据隔离 | 完全隔离 |

### Q3: 为什么会出现项目系统字段？

**A**: 这是**设计缺陷**或**数据异常**导致的：
- 系统字段应该是全局的（`global=1, system=1`）
- 项目系统字段（`global=0, system=1`）不应该存在
- 可能是模板复制时错误地创建了项目字段

**修复方法**：参考"银保通项目状态字段重复问题修复"文档。

### Q4: 如何判断项目使用的是全局模板还是项目模板？

**A**: 查询 `issue_template` 表的 `global` 字段：

```sql
SELECT 
  p.name AS project_name,
  it.name AS template_name,
  it.global AS template_global,
  CASE 
    WHEN it.global = 1 THEN '全局模板'
    WHEN it.global = 0 THEN '项目模板'
    ELSE '无模板（异常）'
  END AS template_type
FROM project p
LEFT JOIN issue_template it ON p.issue_template_id = it.id
WHERE p.name = '你的项目名';
```

### Q5: 创建项目时应该如何配置模板和字段？

**A**: **推荐配置**：

1. **使用全局模板**：
   - 项目的 `issue_template_id` 设置为全局模板ID（如 `5d7c87d2-f405-4ec1-9a3d-71b514cdfda3`）
   - 不需要创建项目模板

2. **使用全局字段**：
   - 不需要创建项目字段
   - 全局字段已经包含了所有必需字段（状态、处理人、严重程度等）

3. **如果需要项目特有字段**：
   - 创建项目自定义字段（`global=0, system=0`）
   - 将字段添加到项目模板中

**不推荐**：
- ❌ 创建项目系统字段（`global=0, system=1`）
- ❌ 复制全局系统字段到项目

## 数据库表结构

### custom_field（自定义字段表）

```sql
CREATE TABLE custom_field (
  id VARCHAR(50) PRIMARY KEY,
  name VARCHAR(64),              -- 字段名称
  scene VARCHAR(30),             -- 场景（ISSUE/TEST_CASE）
  type VARCHAR(30),              -- 字段类型（input/select/date等）
  remark VARCHAR(255),           -- 备注
  options TEXT,                  -- 选项（JSON格式）
  system TINYINT(1),             -- 是否系统字段
  global TINYINT(1),             -- 是否全局字段
  create_time BIGINT,            -- 创建时间
  update_time BIGINT,            -- 更新时间
  create_user VARCHAR(50),       -- 创建人
  project_id VARCHAR(50),        -- 所属项目（全局字段为global或NULL）
  third_part TINYINT(1),         -- 是否第三方字段
  default_value TEXT             -- 默认值
);
```

### issue_template（缺陷模板表）

```sql
CREATE TABLE issue_template (
  id VARCHAR(50) PRIMARY KEY,
  name VARCHAR(64),              -- 模板名称
  platform VARCHAR(30),          -- 平台（Local/JIRA/TAPD等）
  description VARCHAR(255),      -- 描述
  title VARCHAR(64),             -- 标题
  global TINYINT(1),             -- 是否全局模板
  system TINYINT(1),             -- 是否系统模板
  create_time BIGINT,            -- 创建时间
  update_time BIGINT,            -- 更新时间
  create_user VARCHAR(50),       -- 创建人
  project_id VARCHAR(50)         -- 所属项目（全局模板为global）
);
```

### custom_field_template（模板字段关联表）

```sql
CREATE TABLE custom_field_template (
  id VARCHAR(50) PRIMARY KEY,
  field_id VARCHAR(50),          -- 字段ID
  template_id VARCHAR(50),       -- 模板ID
  scene VARCHAR(30),             -- 场景（ISSUE/TEST_CASE）
  required TINYINT(1),           -- 是否必填
  `order` INT,                   -- 显示顺序
  default_value TEXT,            -- 默认值
  custom_data TEXT               -- 自定义数据
);
```

### project（项目表）

```sql
CREATE TABLE project (
  id VARCHAR(50) PRIMARY KEY,
  workspace_id VARCHAR(50),      -- 工作空间ID
  name VARCHAR(255),             -- 项目名称
  description VARCHAR(500),      -- 描述
  create_time BIGINT,            -- 创建时间
  update_time BIGINT,            -- 更新时间
  issue_template_id VARCHAR(50), -- 缺陷模板ID
  case_template_id VARCHAR(50),  -- 用例模板ID
  ...
);
```

## 总结

**核心要点**：

1. **字段分类**：全局系统字段、全局自定义字段、项目自定义字段
2. **模板分类**：全局系统模板、全局自定义模板、项目模板
3. **项目创建时不会自动创建项目模板和项目字段**
4. **推荐使用全局模板 + 全局字段**，简单且易维护
5. **避免创建项目系统字段**，会导致与全局系统字段冲突
6. **项目模板可以绑定全局字段和项目字段**
7. **全局模板不应该绑定项目字段**，会导致其他项目无法使用

**最佳实践**：
- ✅ 新项目直接使用全局模板
- ✅ 需要特殊字段时创建项目自定义字段
- ✅ 需要不同布局时创建项目模板，但绑定全局字段
- ❌ 不要创建项目系统字段
- ❌ 不要让全局模板绑定项目字段
- ❌ 不要删除正在使用的模板或字段
