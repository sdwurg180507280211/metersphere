# 项目创建时的模版与字段选择机制

## 文档说明

本文档从**页面交互角度**，详细说明在创建项目时，系统如何选择和使用全局模版/项目模版、全局字段/项目字段。

---

## 📱 页面交互流程

### 1. 创建项目入口

**路径**：系统设置 > 工作空间 > 项目管理 > 创建项目

**前端组件**：`system-setting/frontend/src/business/workspace/project/EditProject.vue`

**API接口**：`POST /project/add`

### 2. 创建项目表单

用户在创建项目时，需要填写以下信息：

| 字段 | 说明 | 是否必填 | 默认值 |
|------|------|---------|--------|
| 项目名称 | 项目的名称 | ✅ 必填 | 无 |
| 第三方平台 | 缺陷管理平台（Local/Jira/Tapd/Zentao） | ⚠️ 可选 | Local |
| 用例模版 | 测试用例模版 | ⚠️ 可选 | 系统默认模版 |
| 缺陷模版 | 缺陷管理模版 | ⚠️ 可选 | 系统默认模版 |
| 接口模版 | 接口测试模版 | ⚠️ 可选 | 系统默认模版 |
| 项目描述 | 项目的描述信息 | ❌ 非必填 | 无 |

## 🔄 模版选择机制

### 1. 前端获取模版选项

**API接口**：
```javascript
// 获取缺陷模版选项
GET /project/field/template/issue/option/{projectId}
GET /project/field/template/issue/option  // 创建项目时，projectId为空

// 获取用例模版选项
GET /project/field/template/case/option/{projectId}

// 获取接口模版选项
GET /project/field/template/api/option/{projectId}
```

**前端组件**：`system-setting/frontend/src/business/workspace/project/TemplateSelect.vue`

### 2. 后端返回模版列表

**核心逻辑**：`IssueTemplateService.getOption(String projectId)`

```java
public List<IssueTemplate> getOption(String projectId) {
    List<IssueTemplate> issueTemplates;
    IssueTemplateExample example = new IssueTemplateExample();
    
    // 情况1：创建新项目时，projectId为空
    if (StringUtils.isBlank(projectId)) {
        // 只返回全局系统模版
        example.createCriteria().andGlobalEqualTo(true).andSystemEqualTo(true);
        return issueTemplateMapper.selectByExample(example);
    }
    
    // 情况2：编辑已有项目时，projectId不为空
    // 返回：项目自定义模版 + 全局模版 + 项目系统模版
    example.createCriteria()
            .andProjectIdEqualTo(projectId)
            .andSystemEqualTo(false);
    issueTemplates = issueTemplateMapper.selectByExample(example);
    issueTemplates.addAll(getSystemTemplates(projectId));
    return issueTemplates;
}
```

### 3. 模版选项对比

| 场景 | 返回的模版类型 | 数量 | 说明 |

| **编辑已有项目** | 全局模版 + 项目模版 | 4+ | 全局模版 + 该项目的自定义模版 |

### 4. 前端自动选择默认模版

**前端逻辑**：`TemplateSelect.vue`

```javascript
getTemplateOptions() {
  promise.then((response) => {
    this.templateOptions = response.data;
    this.templateFilterOptions = this.templateOptions;
    
    // 如果用户没有手动选择模版，自动选择默认模版
    if (!this.data[this.prop]) {
      for (let item of this.templateOptions) {
        if (this.scene === 'ISSUE') {
          // 缺陷模版：选择系统模版 + Local平台 + 名称为default的模版
          if (item.system && item.platform === 'Local' && item.name === 'default') {
            this.$set(this.data, this.prop, item.id);
            break;
          }
        } else {
          // 用例模版/接口模版：选择第一个系统模版
          if (item.system) {
            this.$set(this.data, this.prop, item.id);
            break;
          }
        }
      }
    }
    this.filter();
  });
}
```

**默认选择规则**：

| 模版类型 | 默认选择规则 | 结果 |
|---------|------------|------|
| 缺陷模版 | `system=1 AND platform='Local' AND name='default'` | 全局系统模版（Local-default） |
| 用例模版 | `system=1`（第一个） | 全局系统模版 |
| 接口模版 | `system=1`（第一个） | 全局系统模版 |

## 🎯 关键结论

### 1. 创建新项目时

**模版选择**：
- ✅ **只能选择全局系统模版**（4个：Local、Jira、Tapd、Zentao）
- ❌ **不能选择项目模版**（因为项目还不存在）
- ✅ **默认选择 Local 平台的 default 模版**

**字段来源**：
- ✅ **使用全局模版绑定的字段**
- ✅ **全局模版只绑定全局字段**（系统字段 + 全局自定义字段）
- ❌ **不会使用项目字段**（因为项目还不存在）

### 2. 编辑已有项目时

**模版选择**：
- ✅ **可以选择全局模版**（4个系统模版）
- ✅ **可以选择项目模版**（该项目创建的自定义模版）
- ✅ **可以切换模版**

**字段来源**：
- ✅ **如果选择全局模版** → 使用全局字段
- ✅ **如果选择项目模版** → 使用全局字段 + 项目字段

## 🔍 字段查找流程

当用户在新创建的项目中打开缺陷表单时：

```
1. 查询项目的 issue_template_id
   ↓
   project.issue_template_id = "5d7c87d2-f405-4ec1-9a3d-71b514cdfda3"
   （全局系统模版：Local-default）

2. 查询模版绑定的字段列表
   ↓
   SELECT * FROM custom_field_template 
   WHERE template_id = "5d7c87d2-f405-4ec1-9a3d-71b514cdfda3" 
   AND scene = 'ISSUE'
   ↓
   返回：11个字段ID

3. 查询每个字段的定义
   ↓
   SELECT * FROM custom_field 
   WHERE id IN (字段ID列表)
   ↓
   返回：11个全局字段（global=1）

4. 判断字段可见性
   ↓
   所有字段都是全局字段（global=1）
   ↓
   所有项目都可见 ✅

5. 渲染缺陷表单
   ↓
   显示11个全局字段
```

## 📋 实际数据验证

### 查询新创建项目的模版和字段

```sql
-- 查询项目使用的模版
SELECT 
  p.name AS 项目名,
  it.name AS 模版名,
  CASE 
    WHEN it.global = 1 THEN '全局模版'
    WHEN it.global = 0 THEN '项目模版'
  END AS 模版类型,
  it.platform AS 平台
FROM project p
LEFT JOIN issue_template it ON p.issue_template_id = it.id
WHERE p.name = '新创建的项目';

-- 结果示例：
-- 项目名          | 模版名  | 模版类型   | 平台
-- 新创建的项目    | default | 全局模版   | Local
```

```sql
-- 查询模版绑定的字段
SELECT 
  cf.name AS 字段名,
  CASE 
    WHEN cf.global = 1 THEN '全局字段'
    WHEN cf.global = 0 THEN '项目字段'
  END AS 字段类型,
  CASE 
    WHEN cf.system = 1 THEN '系统字段'
    WHEN cf.system = 0 THEN '自定义字段'
  END AS 字段来源
FROM project p
JOIN issue_template it ON p.issue_template_id = it.id
JOIN custom_field_template cft ON it.id = cft.template_id
JOIN custom_field cf ON cft.field_id = cf.id
WHERE p.name = '新创建的项目' AND cft.scene = 'ISSUE'
ORDER BY cft.order;
```

## ⚠️ 常见误区

### 误区1：创建项目时可以选择项目模版

❌ **错误**：创建项目时可以选择项目模版

✅ **正确**：创建项目时只能选择全局系统模版，因为项目还不存在，不可能有项目模版

### 误区2：创建项目时会自动创建项目模版

❌ **错误**：创建项目时会自动创建项目模版

✅ **正确**：创建项目时不会自动创建项目模版，项目直接使用全局系统模版

### 误区3：创建项目时会使用项目字段

❌ **错误**：创建项目时会使用项目字段

✅ **正确**：创建项目时只使用全局字段，因为全局模版只绑定全局字段

### 误区4：创建项目后必须创建项目模版

❌ **错误**：创建项目后必须创建项目模版

✅ **正确**：大部分项目直接使用全局模版即可，只有特殊需求时才需要创建项目模版

### 误区5：在项目下修改全局模版会影响其他项目 ⭐

❌ **错误**：在项目设置中修改全局模版的字段，会影响所有使用该模版的项目

✅ **正确**：在项目设置中修改全局模版时，系统会自动创建项目专属模版，不会影响其他项目

**详细说明**：

当你在"项目设置 > 模版管理"中编辑全局模版（如添加/删除字段）时，系统的处理逻辑：

```java
// IssueTemplateService.update()
public void update(UpdateIssueTemplateRequest request) {
    if (request.getGlobal() != null && request.getGlobal()) {
        // 如果编辑的是全局模版
        String originId = request.getId();  // 原全局模版ID
        
        // 1. 创建一个新的项目模版（复制全局模版）
        String id = add(request);
        
        // 2. 更新项目的 issue_template_id，指向新的项目模版
        projectService.updateIssueTemplate(originId, id, request.getProjectId());
    } else {
        // 如果编辑的是项目模版，直接更新
        // 不会影响其他项目
    }
}
```

**实际效果**：

| 操作 | 系统行为 | 影响范围 |
|------|---------|---------|
| 在项目A中编辑全局模版 | 创建项目A的专属模版 | 只影响项目A |
| 在项目B中编辑全局模版 | 创建项目B的专属模版 | 只影响项目B |
| 在系统设置中编辑全局模版 | 直接修改全局模版 | 影响所有使用该全局模版的项目 |

**验证方法**：

```sql
-- 操作前：项目A使用全局模版
SELECT p.name, p.issue_template_id, it.name, it.global
FROM project p
JOIN issue_template it ON p.issue_template_id = it.id
WHERE p.name = '项目A';
-- 结果：issue_template_id = 全局模版ID, global = 1

-- 在项目A中编辑全局模版（添加字段）

-- 操作后：项目A使用项目模版
SELECT p.name, p.issue_template_id, it.name, it.global
FROM project p
JOIN issue_template it ON p.issue_template_id = it.id
WHERE p.name = '项目A';
-- 结果：issue_template_id = 新的项目模版ID, global = 0
```

**关键结论**：
- ✅ 在项目设置中修改全局模版是**安全的**，不会影响其他项目
- ✅ 系统会自动创建项目专属模版，实现**隔离修改**
- ✅ 其他项目仍然使用原来的全局模版
- ⚠️ 只有在**系统设置**中修改全局模版，才会影响所有项目

## 📝 总结

### 核心要点

1. **创建新项目时**：
   - 只能选择全局系统模版（4个：Local、Jira、Tapd、Zentao）
   - 默认选择 Local 平台的 default 模版
   - 使用全局模版绑定的全局字段
   - 不会创建项目模版和项目字段

2. **编辑已有项目时**：
   - 可以选择全局模版或项目模版
   - 可以切换模版
   - 可以创建项目模版和项目字段

3. **字段来源**：
   - 全局模版 → 全局字段
   - 项目模版 → 全局字段 + 项目字段

4. **最佳实践**：
   - 90%的项目直接使用全局模版 + 全局字段
   - 只在特殊需求时创建项目模版和项目字段
   - 保持系统一致性，降低维护成本

### 页面交互总结

| 操作 | 模版选项 | 字段来源 | 说明 |
|------|---------|---------|------|
| 创建新项目 | 全局系统模版（4个） | 全局字段 | 默认选择 Local-default |
| 编辑已有项目 | 全局模版 + 项目模版 | 全局字段 + 项目字段 | 可以切换模版 |
| 创建缺陷 | 使用项目的模版 | 模版绑定的字段 | 自动加载 |
