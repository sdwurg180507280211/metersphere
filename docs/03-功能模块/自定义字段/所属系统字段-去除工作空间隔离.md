# 所属系统字段 - 去除工作空间隔离

## 问题描述

**现象**：
- 在工作空间A创建缺陷，选择"所属系统"字段
- 切换到工作空间B后，该缺陷的"所属系统"字段值消失

**根本原因**：
- 所属系统（AssociatedSystem）数据按工作空间隔离存储
- 前端获取所属系统列表时，只查询当前工作空间的数据
- 缺陷的自定义字段值（custom_field_issues表）只存储所属系统ID
- 切换工作空间后，新工作空间的所属系统列表中找不到该ID，导致显示为空

## 解决方案

**核心思路**：去除所属系统的工作空间隔离，改为全局共享

### 数据流程

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 所属系统数据（associated_system表）                        │
│    - 保留workspace_id字段（用于记录创建来源）                  │
│    - 查询时不再按workspace_id过滤                             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. 前端获取所属系统列表                                        │
│    - /associatedSystem/list/all → 返回所有所属系统            │
│    - /associatedSystem/list/{page}/{size} → 分页查询所有      │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. 缺陷自定义字段值（custom_field_issues表）                   │
│    - 存储所属系统ID                                           │
│    - 显示时从全局列表中查找对应名称                             │
└─────────────────────────────────────────────────────────────┘
```

## 注意事项

### 1. 数据库字段保留

- `associated_system.workspace_id` 字段保留
- 用途：记录所属系统的创建来源（哪个工作空间创建的）
- 不影响查询逻辑

### 2. 权限控制

- 所属系统的增删改权限仍然按工作空间控制
- 只是查询时不再过滤，所有工作空间共享数据

### 3. 数据一致性

- 所属系统名称应全局唯一（建议）
- 避免不同工作空间创建同名所属系统导致混淆

### 4. 升级兼容性

- 无需数据迁移
- 现有数据完全兼容
- 修改后立即生效
