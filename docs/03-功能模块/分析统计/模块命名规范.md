# 分析统计模块命名规范

## 模块信息

### 基本信息
- **中文名称**: 分析统计
- **英文服务名**: `analytics-stat`
- **模块定位**: 提供工作流分析、SQL查询、数据字典、报表分析等分析统计类功能

### 命名说明
- **analytics**: 分析（Analytics）
- **stat**: 统计（Statistics）
- 组合含义：分析统计服务

## 命名规则总结

### 使用中文的场景
1. 用户界面（菜单、标题、按钮）
2. 用户文档（使用手册、帮助文档）
3. 注释和说明

### 使用英文的场景
1. 代码（类名、方法名、变量名）
2. 配置文件（application.properties、pom.xml）
3. URL 路径
4. 技术文档（架构设计、API 文档）
5. Docker 镜像名
6. 服务注册名

### 命名格式规范

| 场景 | 格式 | 示例 |
|------|------|------|
| Java 类名 | PascalCase | `AnalyticsStatApplication` |
| Java 包名 | lowercase | `io.metersphere.analyticsstat` |
| Java 方法名 | camelCase | `getAnalyticsData()` |
| Maven artifactId | kebab-case | `analytics-stat` |
| Spring 应用名 | kebab-case | `analytics-stat` |
| URL 路径 | kebab-case | `/analytics-stat/workflow` |
| JavaScript 变量 | camelCase | `analyticsStat` |
| Vue 组件名 | PascalCase | `AnalyticsStatHome` |
| 文件名 | kebab-case | `analytics-stat-config.js` |
| 数据库表名 | snake_case | `analytics_stat_config` |

## 注意事项

### 1. 一致性原则
- 同一场景下使用相同的命名格式
- 中英文命名保持对应关系
- 避免混用不同的命名风格

### 2. 可读性原则
- 使用有意义的名称
- 避免使用缩写（除非是通用缩写）
- 保持名称简洁但清晰

### 3. 兼容性原则
- 遵循 Java、JavaScript、Maven 等各自的命名规范
- 考虑 URL 编码问题（避免使用特殊字符）
- 考虑跨平台兼容性（Windows、Linux、macOS）

### 4. 扩展性原则
- 预留扩展空间
- 避免使用过于具体的名称
- 考虑未来可能的功能扩展
