# 菜单不显示问题排查

## 问题现象

主应用 `http://127.0.0.1:8000/` 中没有显示"分析统计"菜单入口。

### 1. 数据库配置 ✅
```sql
-- 已插入模块配置
INSERT INTO system_parameter (param_key, param_value, type, sort) 
VALUES ('metersphere.module.analytics-stat', 'ENABLE', 'text', 1);
```

验证：
```bash
curl -s http://127.0.0.1:8000/module/list | jq '.data[] | select(.key == "analytics-stat")'
# 返回：{"key": "analytics-stat", "status": "ENABLE"}
```

### 2. Eureka 注册 ✅
```bash
curl -s http://127.0.0.1:8000/services | jq '.data[] | select(.serviceId == "analytics-stat")'
# 返回：{"serviceId": "analytics-stat", "port": 8009}
```

### 3. 前端代码修改 ✅
- ✅ 创建了图标文件：`framework/sdk-parent/frontend/src/assets/module/analytics-stat.svg`
- ✅ 修改了菜单文件：`framework/sdk-parent/frontend/src/components/layout/AsideMenus.vue`
- ✅ SDK 前端已编译：`npm run build`

### 4. Gateway JAR 验证 ✅
```bash
unzip -p framework/gateway/target/gateway-2.10.jar BOOT-INF/classes/static/js/app.*.js | grep -o "analytics-stat"
# 返回：analytics-stat
```

## 问题原因

**你在 IDEA 中启动的 Gateway 使用的是旧的 JAR 文件！**

IDEA 的 Spring Boot 运行配置通常会：
1. 使用 `target/` 目录下的 JAR 文件
2. 或者使用 `target/classes/` 目录下的编译产物

如果你在 IDEA 中启动 Gateway 之后才重新打包，那么 IDEA 中运行的 Gateway 还是旧版本。

## 解决方案

### 方案一：在 IDEA 中重启 Gateway（推荐）

1. **停止 IDEA 中的 Gateway 进程**
   - 点击 IDEA 右上角的红色停止按钮
   - 或者在 Run 窗口中点击停止

2. **重新启动 Gateway**
   - 点击 IDEA 右上角的绿色运行按钮
   - 或者使用快捷键 `Ctrl+R` (Mac: `Cmd+R`)

3. **等待 Gateway 启动完成**
   - 查看控制台日志，确认启动成功
   - 看到 "Started GatewayApplication" 表示启动完成

4. **清除浏览器缓存**
   - Chrome/Edge: `Cmd+Shift+R` (Mac) 或 `Ctrl+Shift+R` (Windows)
   - Safari: `Cmd+Option+E` 清空缓存，然后 `Cmd+R` 刷新
   - Firefox: `Cmd+Shift+R` (Mac) 或 `Ctrl+Shift+R` (Windows)

5. **访问主应用**
   - 打开 `http://127.0.0.1:8000/`
   - 检查左侧菜单栏是否显示"分析统计"

### 方案二：确保 IDEA 使用最新的编译产物

如果方案一不行，可能是 IDEA 的缓存问题：

1. **清理 IDEA 缓存**
   ```bash
   # 在项目根目录执行
   ./mvnw clean
   ```

2. **在 IDEA 中重新构建项目**
   - 菜单：Build → Rebuild Project
   - 或者：Build → Build Project

3. **重启 Gateway**
   - 停止 Gateway
   - 重新启动 Gateway

4. **清除浏览器缓存并刷新**

### 方案三：检查 IDEA 的运行配置

1. **打开运行配置**
   - 点击 IDEA 右上角的运行配置下拉菜单
   - 选择 "Edit Configurations..."

2. **检查 Gateway 配置**
   - 找到 Gateway 的运行配置
   - 查看 "Working directory" 是否正确
   - 查看 "Use classpath of module" 是否选择了 gateway

3. **确保使用最新的 JAR**
   - 如果配置中指定了 JAR 路径，确保指向 `framework/gateway/target/gateway-2.10.jar`

## 验证步骤

### 1. 验证 Gateway 是否使用了新代码

在浏览器中访问：
```
http://127.0.0.1:8000/
```

打开浏览器开发者工具（F12），切换到 Network 标签，刷新页面，查找 `app.*.js` 文件：

1. **找到 app.js 文件**
   - 在 Network 标签中搜索 "app"
   - 找到类似 `app.c52a5110.js` 的文件

2. **查看文件内容**
   - 点击该文件
   - 切换到 "Response" 标签
   - 搜索 "analytics-stat"
   - 如果找到，说明 Gateway 已经使用了新代码

3. **检查文件时间戳**
   - 查看 Response Headers 中的 `Last-Modified` 时间
   - 确认是最近的时间（应该是今天）

### 2. 验证浏览器缓存

如果 app.js 中包含 analytics-stat，但菜单还是不显示，说明是浏览器缓存问题：

1. **强制刷新**
   - Mac: `Cmd+Shift+R`
   - Windows: `Ctrl+Shift+R`

2. **清空缓存**
   - Chrome: 设置 → 隐私和安全 → 清除浏览数据 → 缓存的图片和文件
   - Safari: 开发 → 清空缓存
   - Firefox: 设置 → 隐私与安全 → Cookie 和网站数据 → 清除数据

3. **使用无痕模式**
   - 打开无痕窗口
   - 访问 `http://127.0.0.1:8000/`
   - 如果无痕模式下能看到菜单，说明是缓存问题

### 3. 验证菜单显示逻辑

打开浏览器控制台（F12），切换到 Console 标签，执行：

```javascript
// 检查 modules 配置
console.log(localStorage.getItem('modules'))

// 检查 microApps 配置
console.log(sessionStorage.getItem('microApps'))

// 检查 analytics-stat 是否启用
const modules = JSON.parse(localStorage.getItem('modules') || '{}')
console.log('analytics-stat enabled:', modules['analytics-stat'] === 'ENABLE')

// 检查 analytics-stat 服务是否注册
const microApps = JSON.parse(sessionStorage.getItem('microApps') || '{}')
console.log('analytics-stat service:', microApps['analytics-stat'])
```

如果输出显示：
- `analytics-stat enabled: true`
- `analytics-stat service: { port: 8009, ... }`

但菜单还是不显示，说明是前端代码问题。

## 常见问题

### Q1: 为什么 IDEA 中的 Gateway 没有使用最新代码？

**A**: IDEA 的 Spring Boot 运行配置通常会缓存编译产物。即使你重新打包了 JAR，IDEA 可能还在使用旧的 `target/classes/` 目录。

**解决方案**：
1. 停止 Gateway
2. 执行 `./mvnw clean` 清理所有编译产物
3. 在 IDEA 中重新构建项目（Build → Rebuild Project）
4. 重新启动 Gateway

### Q2: 浏览器强制刷新后还是看不到菜单？

**A**: 可能是以下原因：

1. **Gateway 还是旧版本**
   - 验证方法：查看 Network 标签中 app.js 的内容
   - 解决方案：重启 Gateway

2. **localStorage 缓存了旧数据**
   - 验证方法：在控制台执行 `localStorage.clear()`
   - 解决方案：清空 localStorage 后刷新页面

3. **sessionStorage 缓存了旧数据**
   - 验证方法：在控制台执行 `sessionStorage.clear()`
   - 解决方案：清空 sessionStorage 后刷新页面

### Q3: 无痕模式下能看到菜单，但正常模式下看不到？

**A**: 这是典型的浏览器缓存问题。

**解决方案**：
1. 清空浏览器缓存
2. 清空 localStorage 和 sessionStorage
3. 刷新页面
