# analytics-stat 访问问题诊断与解决方案

## 问题现象

访问分析统计模块时，URL 变成：
```
http://localhost:8000/analytics-stat/analytics-stat/dashboard#/analytics-stat
```

而其他模块正常：
```
http://localhost:8000/#/track/home  ✅
http://localhost:8000/#/project/home  ✅
```

## 根本原因

### 1. 技术栈差异

| 模块 | 构建工具 | Vue 版本 | 路由模式 | qiankun 插件 |
|------|---------|---------|---------|-------------|
| test-track | Webpack | Vue 2 | hash | webpack-plugin-qiankun |
| api-test | Webpack | Vue 2 | hash | webpack-plugin-qiankun |
| **analytics-stat** | **Vite** | **Vue 3** | **history** | **vite-plugin-qiankun** |

### 2. qiankun 加载机制

主应用的 micro-app.js 配置：

```javascript
// 开发环境
entry: '//127.0.0.1:' + (svc.port - 4000)
// analytics-stat: //127.0.0.1:4009

// 生产环境
entry: window.location.host + "/" + app.name
// analytics-stat: localhost:8000/analytics-stat
```

**关键点**：
- 开发环境：qiankun 从前端开发服务器加载（4009 端口）
- 生产环境：qiankun 从 Gateway 代理加载（通过后端 8009）

### 3. 当前问题

你现在**没有启动前端开发服务器**，所以 qiankun 尝试从生产环境加载，但是：

1. **路由模式不匹配**：
   - 主应用：hash 模式（`#/track`）
   - analytics-stat：history 模式（`/dashboard`）

2. **路径处理不一致**：
   - Webpack 模块：qiankun 自动处理路径
   - Vite 模块：需要手动配置 base 路径

## 解决方案

### 方案 1：启动前端开发服务器（推荐用于开发）

```bash
cd analytics-stat/frontend
npm run dev
```

**优点**：
- 热更新，开发体验好
- 不需要每次都重新构建

**缺点**：
- 需要额外启动一个服务

### 方案 2：使用生产模式（推荐用于测试生产环境）

**步骤**：

1. **构建前端**：
```bash
cd analytics-stat/frontend
npm run build
```

2. **打包后端**：
```bash
cd analytics-stat/backend
mvn clean package -DskipTests
```

3. **重启后端服务**（在 IDEA 中）

4. **清除浏览器缓存并刷新**

**但是**，这个方案需要修改路由配置以匹配其他模块。

### 方案 3：修改为与其他模块一致（推荐用于生产）

#### 3.1 修改路由模式为 hash

**原因**：主应用使用 hash 模式，子应用也应该使用 hash 模式以保持一致。

**修改 `analytics-stat/frontend/src/router/index.ts`**：

```typescript
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),  // 改为 hash 模式
  routes
})
```

#### 3.2 简化路由路径

**原因**：qiankun 会自动处理路径前缀，子应用不需要包含 `/analytics-stat` 前缀。

**修改路由配置**：

```typescript
const routes: RouteRecordRaw[] = [
  {
    path: '/',
    redirect: '/dashboard'  // 去掉 /analytics-stat 前缀
  },
  {
    path: '/',
    component: () => import('@/business/AnalyticsStat.vue'),
    children: [
      {
        path: 'dashboard',  // 不是 /analytics-stat/dashboard
        name: 'Dashboard',
        component: () => import('@/views/Dashboard.vue')
      },
      // ...
    ]
  }
]
```

#### 3.3 修改二级导航菜单路径

**修改 `AnalyticsStatHeaderMenus.vue`**：

```typescript
const menus = ref([
  { path: '/dashboard', name: '数据概览' },  // 去掉 /analytics-stat 前缀
  { path: '/sql-console', name: 'SQL查询台' },
  { path: '/data-dictionary', name: '数据字典' }
])
```

#### 3.4 重新构建

```bash
# 1. 构建前端
cd analytics-stat/frontend
npm run build

# 2. 打包后端
cd ../backend
mvn clean package -DskipTests

# 3. 重启后端服务
```

## 为什么其他模块不需要这么复杂？

### Webpack vs Vite 的差异

1. **Webpack（test-track, api-test）**：
   - qiankun 插件自动处理路径
   - 默认 hash 模式
   - 与主应用天然兼容

2. **Vite（analytics-stat）**：
   - 需要手动配置 base 路径
   - 默认 history 模式
   - 需要额外配置才能与主应用兼容

### 历史原因

MeterSphere 的其他模块都是用 Vue 2 + Webpack 开发的，qiankun 的配置也是基于这个技术栈优化的。

analytics-stat 使用 Vue 3 + Vite，是新技术栈，需要额外的配置才能与现有系统兼容。

## 快速诊断清单

遇到问题时，按以下顺序检查：

### 1. 检查前端开发服务器是否启动

```bash
curl http://localhost:4009
```

- ✅ 返回 HTML：前端开发服务器正在运行
- ❌ 连接失败：需要启动前端开发服务器

### 2. 检查后端服务是否启动

```bash
curl http://localhost:8009/health
```

- ✅ 返回 `{"status":"UP"}`：后端服务正常
- ❌ 连接失败：需要启动后端服务

### 3. 检查 Eureka 注册

访问 `http://localhost:8761`，确认 ANALYTICS-STAT 服务已注册。

### 4. 检查浏览器控制台

打开浏览器开发者工具，查看：
- Network 标签：检查资源加载情况
- Console 标签：查看 qiankun 的日志

**正常情况应该看到**：
```
[analytics-stat] app bootstraped
[analytics-stat] props from main framework
[analytics-stat] mount
```

### 5. 检查 URL

- ✅ 正常：`http://localhost:8000/#/analytics-stat`
- ❌ 异常：`http://localhost:8000/analytics-stat/analytics-stat/...`

## 总结

analytics-stat 难以配置的根本原因是：

1. **技术栈不同**：Vue 3 + Vite vs Vue 2 + Webpack
2. **路由模式不同**：history vs hash
3. **qiankun 配置需要适配**：Vite 需要额外配置

**最简单的解决方案**：
- 开发时：启动前端开发服务器（`npm run dev`）
- 生产时：修改路由模式为 hash，简化路由路径

这样就能像其他模块一样正常工作了。
