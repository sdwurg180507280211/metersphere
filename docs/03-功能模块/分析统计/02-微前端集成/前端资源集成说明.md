# 前端资源集成说明

## 问题描述

在analytics-stat微服务中，前端资源虽然已经构建到`dist`目录，但是访问`http://127.0.0.1:8009/`时返回404错误。

## 根本原因

前端构建产物（`dist`目录）没有被复制到backend的`src/main/resources/static`和`src/main/resources/public`目录，导致Spring Boot无法提供静态资源服务。

## 解决方案

### 1. Maven配置

在`analytics-stat/backend/pom.xml`中使用`maven-antrun-plugin`插件，在`generate-resources`阶段自动复制前端资源：

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-antrun-plugin</artifactId>
    <executions>
        <execution>
            <id>main-class-placement</id>
            <phase>generate-resources</phase>
            <configuration>
                <skip>${skipAntRunForJenkins}</skip>
                <target>
                    <!-- 复制静态资源（JS、CSS、图片等）到static目录 -->
                    <copy todir="src/main/resources/static">
                        <fileset dir="../frontend/dist">
                            <exclude name="*.html"/>
                        </fileset>
                    </copy>
                    <!-- 复制HTML文件到public目录 -->
                    <copy todir="src/main/resources/public">
                        <fileset dir="../frontend/dist">
                            <include name="*.html"/>
                        </fileset>
                    </copy>
                </target>
            </configuration>
            <goals>
                <goal>run</goal>
            </goals>
        </execution>
    </executions>
</plugin>
```

### 2. 构建流程

完整的构建流程如下：

```
1. 前端构建
   npm run build
   ↓
   生成 frontend/dist/ 目录
   ├── index.html
   └── assets/
       ├── *.js
       ├── *.css
       └── *.png

2. Maven复制（maven-antrun-plugin）
   ↓
   backend/src/main/resources/
   ├── public/
   │   └── index.html
   └── static/
       └── assets/
           ├── *.js
           ├── *.css
           └── *.png

3. Maven打包（spring-boot-maven-plugin）
   ↓
   backend/target/analytics-stat-2.10.jar
   └── BOOT-INF/classes/
       ├── public/
       │   └── index.html
       └── static/
           └── assets/
```

### 3. Spring Boot静态资源机制

Spring Boot会自动提供以下位置的静态资源：

| 位置 | 用途 | 访问路径 |
|------|------|---------|
| `classpath:/public/` | 公共资源（HTML等） | `/` 或 `/index.html` |
| `classpath:/static/` | 静态资源（JS、CSS、图片） | `/assets/*` |
| `classpath:/resources/` | 其他资源 | `/resources/*` |
| `classpath:/META-INF/resources/` | META-INF资源 | `/META-INF/*` |

**优先级**：`META-INF/resources` > `resources` > `static` > `public`

### 4. 访问路径映射

| 请求路径 | 实际文件位置 |
|---------|------------|
| `http://127.0.0.1:8009/` | `classpath:/public/index.html` |
| `http://127.0.0.1:8009/index.html` | `classpath:/public/index.html` |
| `http://127.0.0.1:8009/assets/index-BOFi7fDo.js` | `classpath:/static/assets/index-BOFi7fDo.js` |
| `http://127.0.0.1:8009/analytics-stat/health` | Controller接口 |

## 常见问题

### Q1: 为什么要分别复制到static和public？

**A**: Spring Boot对这两个目录有不同的处理：
- `public/` - 用于存放欢迎页（index.html），Spring Boot会自动将其映射到根路径`/`
- `static/` - 用于存放其他静态资源（JS、CSS、图片等）

### Q2: 为什么不直接在frontend目录提供静态资源？

**A**: 因为：
1. 微服务架构中，每个服务应该是独立的、可部署的单元
2. 前端资源打包到JAR中，可以简化部署（只需要一个JAR文件）
3. 便于Docker镜像构建和容器化部署

### Q3: 如果前端资源很大，会影响JAR大小吗？

**A**: 会的。如果前端资源很大（比如包含大量图片、视频），可以考虑：
1. 使用CDN存储静态资源
2. 使用对象存储（如MinIO、OSS）
3. 使用Nginx单独提供静态资源服务

### Q4: 开发时如何避免每次都重新构建？

**A**: 开发时可以：
1. 前端独立运行：`npm run dev` （端口4009）
2. 后端独立运行：直接运行`AnalyticsStatApplication`
3. 前端通过代理访问后端API（vite.config.ts中已配置）
