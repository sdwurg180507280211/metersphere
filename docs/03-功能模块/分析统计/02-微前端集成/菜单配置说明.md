# MeterSphere 模块菜单配置说明

## 概述

MeterSphere 通过 `system_parameter` 表控制左侧菜单的显示与隐藏。每个微服务模块需要在该表中插入一条配置记录，才能在主应用的左侧菜单中显示。

---

## 一、system_parameter 表结构

```sql
CREATE TABLE IF NOT EXISTS `system_parameter` (
    `param_key`   varchar(64) NOT NULL COMMENT '参数名',
    `param_value` text DEFAULT NULL COMMENT '参数值',
    `type`        varchar(100) NOT NULL DEFAULT 'text' COMMENT '参数类型',
    `sort`        int(5) DEFAULT NULL COMMENT '排序',
    PRIMARY KEY (`param_key`)
) ENGINE = InnoDB DEFAULT CHARSET = utf8mb4 COLLATE utf8mb4_general_ci;
```

**字段说明**：
- `param_key`：参数键，格式为 `metersphere.module.{moduleCamelCase}`
- `param_value`：参数值，`ENABLE`（启用）或 `DISABLE`（禁用）
- `type`：参数类型，固定为 `text`
- `sort`：排序，固定为 `1`

---

## 二、模块配置规则

### 2.1 命名规则

**param_key 格式**：`metersphere.module.{moduleCamelCase}`

**驼峰命名转换规则**：
- kebab-case（短横线命名）→ camelCase（驼峰命名）
- 去掉短横线，将短横线后的首字母大写

**示例**：

| 模块名（kebab-case） | param_key（camelCase） |
| -------------------- | ---------------------- |
| `api-test` | `metersphere.module.api` |
| `performance-test` | `metersphere.module.performance` |
| `test-track` | `metersphere.module.testTrack` |
| `report-stat` | `metersphere.module.reportStat` |
| `workstation` | `metersphere.module.workstation` |
| `analytics-stat` | `metersphere.module.analyticsStat` |
| `ui-test` | `metersphere.module.ui` |

### 2.2 配置值

| 值 | 说明 | 效果 |
| -- | ---- | ---- |
| `ENABLE` | 启用 | 左侧菜单显示该模块 |
| `DISABLE` | 禁用 | 左侧菜单隐藏该模块 |

## 四、新增模块配置

### 4.1 使用 Flyway 迁移脚本（推荐）

**创建文件**：`{module}/backend/src/main/resources/db/migration/V2__add_{module}_module.sql`

```sql
-- {模块中文名}模块菜单配置
-- 创建时间: YYYY-MM-DD
-- 说明: 在 system_parameter 表中添加模块的启用配置，用于控制左侧菜单显示

SET SESSION innodb_lock_wait_timeout = 7200;

-- 插入模块配置
INSERT INTO system_parameter (param_key, param_value, type, sort)
VALUES ('metersphere.module.{moduleCamelCase}', 'ENABLE', 'text', 1)
ON DUPLICATE KEY UPDATE param_value = 'ENABLE';

SET SESSION innodb_lock_wait_timeout = DEFAULT;
```

**示例**（analytics-stat 模块）：

```sql
-- 分析统计模块菜单配置
-- 创建时间: 2026-02-09

SET SESSION innodb_lock_wait_timeout = 7200;

INSERT INTO system_parameter (param_key, param_value, type, sort)
VALUES ('metersphere.module.analyticsStat', 'ENABLE', 'text', 1)
ON DUPLICATE KEY UPDATE param_value = 'ENABLE';

SET SESSION innodb_lock_wait_timeout = DEFAULT;
```

**执行时机**：
- 模块首次启动时，Flyway 会自动执行迁移脚本
- 脚本只执行一次，不会重复执行

### 4.2 手动插入（不推荐）

如果 Flyway 未执行或需要临时测试，可以手动插入：

```sql
INSERT INTO metersphere.system_parameter (param_key, param_value, type, sort)
VALUES ('metersphere.module.{moduleCamelCase}', 'ENABLE', 'text', 1)
ON DUPLICATE KEY UPDATE param_value = 'ENABLE';
```

## 五、菜单显示机制

### 5.1 读取流程

1. **Gateway 启动时**读取 `system_parameter` 表中所有 `metersphere.module.*` 配置
2. **主应用**通过 Eureka 获取所有已注册的微服务
3. **过滤**：只显示 `param_value = 'ENABLE'` 的模块
4. **渲染**：在左侧菜单中显示启用的模块

### 5.2 配置生效时机

- **新增配置**：需要重启 Gateway 和主应用
- **修改配置**：需要重启 Gateway 和主应用
- **删除配置**：需要重启 Gateway 和主应用

### 5.3 缓存机制

- Gateway 会缓存模块配置，避免频繁查询数据库
- 修改配置后，必须重启 Gateway 才能生效

## 六、常见问题

### 6.1 菜单不显示

**症状**：新模块启动成功，但左侧菜单不显示

**原因**：
1. `system_parameter` 表中没有该模块的配置
2. `param_value` 为 `DISABLE`
3. `param_key` 命名错误（驼峰命名不正确）
4. Gateway 未重启，配置未生效

**解决方案**：

1. **检查配置是否存在**：
```sql
SELECT * FROM metersphere.system_parameter 
WHERE param_key = 'metersphere.module.{moduleCamelCase}';
```

2. **检查 Flyway 执行状态**：
```sql
SELECT * FROM metersphere.flyway_schema_history 
WHERE script LIKE '%{module}%';
```

3. **手动插入配置**（如果不存在）：
```sql
INSERT INTO metersphere.system_parameter (param_key, param_value, type, sort)
VALUES ('metersphere.module.{moduleCamelCase}', 'ENABLE', 'text', 1);
```

4. **重启 Gateway**：
```bash
cd framework/gateway && mvn spring-boot:run
```

5. **清除浏览器缓存**并重新登录

### 6.2 驼峰命名错误

**错误示例**：

| 错误 | 正确 |
| ---- | ---- |
| `metersphere.module.analytics-stat` | `metersphere.module.analyticsStat` |
| `metersphere.module.AnalyticsStat` | `metersphere.module.analyticsStat` |
| `metersphere.module.analyticsstat` | `metersphere.module.analyticsStat` |

**规则**：
- 第一个单词全小写
- 后续单词首字母大写
- 不包含短横线

### 6.3 配置未生效

**症状**：已插入配置，但菜单仍不显示

**原因**：Gateway 缓存未刷新

**解决方案**：
1. 重启 Gateway
2. 清除浏览器缓存
3. 重新登录

## 七、最佳实践

### 7.1 使用 Flyway 管理配置

**优点**：
- 版本化管理，可追溯
- 自动执行，无需手动操作
- 支持回滚
- 团队协作友好

**推荐文件命名**：
- `V2__add_{module}_module.sql`
- 版本号从 V2 开始（V1 通常是表结构初始化）

### 7.2 配置注释

在 SQL 脚本中添加详细注释：
- 创建时间
- 模块说明
- 配置作用

### 7.3 测试验证

新增模块后，务必验证：
1. 数据库中配置已插入
2. Flyway 迁移历史中有记录
3. 左侧菜单显示新模块
4. 点击菜单可以正常跳转

## 九、总结

MeterSphere 的模块菜单配置机制：

1. **配置存储**：`system_parameter` 表
2. **配置格式**：`metersphere.module.{moduleCamelCase} = ENABLE/DISABLE`
3. **配置管理**：使用 Flyway 迁移脚本
4. **生效时机**：重启 Gateway 和主应用
5. **命名规则**：驼峰命名，首字母小写

新增模块时，务必创建 Flyway 迁移脚本，插入模块配置，否则左侧菜单不会显示该模块。
