# 主应用菜单集成实施记录

## 一、问题背景

analytics-stat 微服务已成功启动并注册到 Eureka，但主应用 `http://127.0.0.1:8000/` 中没有显示新模块的菜单入口。

## 二、问题分析

### 2.1 菜单显示机制

MeterSphere 主应用的菜单显示需要满足两个条件：

1. **模块配置**：在 `system_parameter` 表中配置模块启用状态
2. **菜单定义**：在 `AsideMenus.vue` 中定义菜单项

### 2.2 检查结果

- ✅ Eureka 注册：analytics-stat 已成功注册（serviceId: analytics-stat, port: 8009）
- ✅ qiankun 配置：子应用已正确配置 qiankun 支持
- ❌ 模块配置：数据库中缺少 `metersphere.module.analytics-stat` 配置
- ❌ 菜单定义：`AsideMenus.vue` 中未定义菜单项

## 三、实施步骤

### 3.1 数据库配置

**目的**：在系统参数表中添加 analytics-stat 模块配置，使其能被 `/module/list` 接口返回。

```sql
-- 插入模块配置
INSERT INTO system_parameter (param_key, param_value, type, sort) 
VALUES ('metersphere.module.analytics-stat', 'ENABLE', 'text', 1);
```

**验证**：
```bash
curl -s http://127.0.0.1:8000/module/list | jq '.data[] | select(.key == "analytics-stat")'
```

返回结果：
```json
{
  "key": "analytics-stat",
  "status": "ENABLE"
}
```

### 3.2 创建菜单图标

**目的**：为 analytics-stat 模块创建专属的 SVG 图标。

**文件路径**：`framework/sdk-parent/frontend/src/assets/module/analytics-stat.svg`

**图标设计**：
- 柱状图：表示数据统计
- 折线图：表示趋势分析
- 配色：白色填充，适配深色和浅色主题

### 3.3 添加菜单项

**目的**：在主应用侧边栏中添加"分析统计"菜单入口。

**文件路径**：`framework/sdk-parent/frontend/src/components/layout/AsideMenus.vue`

**修改内容**：
```vue
<!-- 在 report 和 project 之间添加 -->
<el-menu-item index="/analytics-stat" v-if="check('analytics-stat')" onselectstart="return false">
  <div>
    <svg-icon iconClass="analytics-stat" class-name="ms-menu-img"/>
    <span slot="title" class="ms-menu-item-title">分析统计</span>
  </div>
</el-menu-item>
```

**菜单位置**：
- 在"报告统计"之后
- 在"项目设置"之前

### 3.4 重新编译和部署

**步骤 1：编译 SDK 前端**
```bash
cd framework/sdk-parent/frontend
npm run build
```

**步骤 2：打包 Gateway**
```bash
./mvnw clean package -DskipTests -pl framework/gateway
```

**步骤 3：重启 Gateway 服务**
```bash
# 杀掉旧进程
lsof -ti:8000 | xargs kill -9

# 启动新进程
java -jar framework/gateway/target/gateway-2.10.jar
```

## 四、技术要点

### 4.1 模块配置机制

**配置表**：`system_parameter`

**配置格式**：
- `param_key`: `metersphere.module.{模块名}`
- `param_value`: `ENABLE` 或 `DISABLE`
- `type`: `text`
- `sort`: 排序值（通常为 1）

**读取接口**：`/module/list`

**实现类**：`SystemParameterService.listModules()`

### 4.2 菜单显示逻辑

**判断方法**：`AsideMenus.vue` 中的 `check(key)` 方法

**判断条件**：
1. `modules[key] === 'ENABLE'`：模块在数据库中启用
2. `microApps[key]`：服务已注册到 Eureka
3. 权限检查（可选）：通过 `v-permission` 指令

**数据来源**：
- `modules`：从 localStorage 读取，由 `/module/list` 接口填充
- `microApps`：从 sessionStorage 读取，由 `/services` 接口填充

### 4.3 qiankun 路由规则

**主应用路由**：`/#/{serviceId}`

**子应用入口**：`//127.0.0.1:{port - 4000}`

**analytics-stat 示例**：
- 主应用访问：`http://127.0.0.1:8000/#/analytics-stat`
- 子应用端口：4009（8009 - 4000）
- 子应用地址：`http://127.0.0.1:4009`

## 六、注意事项

### 6.1 二次开发原则

**改动文件**：
1. `system_parameter` 表（数据库）
2. `framework/sdk-parent/frontend/src/assets/module/analytics-stat.svg`（新增）
3. `framework/sdk-parent/frontend/src/components/layout/AsideMenus.vue`（修改）

**改动特点**：
- 改动面小：仅涉及 3 个位置
- 边界清晰：集中在菜单配置和模块注册
- 可回滚：升级时易于对照和搬运

### 6.2 升级兼容性

**数据库迁移**：
- 建议创建 Flyway 迁移脚本：`V{版本}__add_analytics_stat_module.sql`
- 脚本内容：插入 `metersphere.module.analytics-stat` 配置

**前端升级**：
- 图标文件：独立文件，不影响其他模块
- 菜单项：独立代码块，易于识别和迁移
