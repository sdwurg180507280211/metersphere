# 分析统计模块 - 前端技术栈选型

## 微前端架构优势

### 技术栈独立性

项目已从 qiankun 切换到 **micro-app**（`@micro-zoe/micro-app ^1.0.0-rc.4`）。micro-app 基于 WebComponent 渲染子应用，天然支持技术栈无关，**analytics-stat 前端可以完全独立选择技术栈**，不受主应用限制：

| 主应用 (framework/sdk-parent/frontend) | 新模块 (analytics-stat/frontend)    |
| -------------------------------------- | ----------------------------------- |
| Vue 2.7.3                              | ✅**可以使用 Vue 3.x**        |
| Vue Router 3.x                         | ✅**可以使用 Vue Router 4.x** |
| Pinia 2.0.14                           | ✅**可以使用最新 Pinia**      |
| Element UI 2.15.13                     | ✅**可以使用 Element Plus**   |
| Webpack (通过 Vue CLI)                 | ✅**可以使用 Vite**           |
| JavaScript                             | ✅**可以使用 TypeScript**     |

### 隔离机制

micro-app 通过以下机制实现技术栈隔离：

1. **WebComponent 容器**: 基于 CustomElement 渲染子应用，天然隔离
2. **JavaScript 沙箱**: with 沙箱 / iframe 沙箱隔离全局变量和副作用
3. **样式隔离**: 自动添加前缀，防止 CSS 污染
4. **独立构建**: 各自独立打包，互不影响
5. **按需加载**: 只有访问时才 fetch 子应用 HTML 并解析渲染

### Vite 子应用支持说明

micro-app 对 Vite 构建的子应用支持情况：

- **生产环境**：完全支持。Vite build 产出的是普通 JS 文件，micro-app 正常加载，无需额外配置
- **开发环境**：Vite dev server 使用原生 ESM（`<script type="module">`），需要注意：
  - micro-app 1.0.0-rc 版本已支持 ESM 模块加载
  - 需要在主应用的 `<micro-app>` 标签上配置 `iframe` 沙箱模式，或使用 `esmodule` 属性
  - 子应用 Vite dev server 需要配置 CORS headers
- **UMD 生命周期**：与现有子应用一致，通过 `window.mount` / `window.unmount` 暴露生命周期

```typescript
// main.ts - micro-app UMD 生命周期模式
import { createApp } from 'vue'
import { createRouter, createWebHashHistory } from 'vue-router'
import { createPinia } from 'pinia'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import App from './App.vue'
import routes from './router'
import { isMicroAppEnv } from './micro-app-env'

import type { App as VueApp } from 'vue'
import type { Router } from 'vue-router'

let app: VueApp | null = null
let router: Router | null = null

/**
 * 渲染函数
 *
 * 【关键】micro-app with 沙箱解析子应用 HTML 时，空的 <div id="app"></div>
 * 可能被丢弃。因此 mount 时需要确保挂载点存在，不存在则手动创建。
 */
function mount() {
  // 确保挂载点 #app 存在（micro-app 可能丢弃空 div）
  let appEl = document.querySelector('#app')
  if (!appEl) {
    appEl = document.createElement('div')
    appEl.id = 'app'
    document.body.appendChild(appEl)
  }

  router = createRouter({
    // 使用 hash 模式，与主应用路由保持一致
    history: createWebHashHistory(),
    routes,
  })

  app = createApp(App)
  app.use(router)
  app.use(createPinia())
  app.use(ElementPlus)
  app.mount('#app')
}

function unmount() {
  app?.unmount()
  app = null
  router = null
}

// micro-app UMD 生命周期模式
// micro-app 会在子应用渲染时自动调用 window.mount()
// 非微前端环境直接调用 mount()
;(window as any).mount = () => { mount() }
;(window as any).unmount = () => { unmount() }

// 非微前端环境直接挂载，micro-app 环境由框架调用 window.mount()
if (!isMicroAppEnv()) {
  mount()
}
```

#### Composition API 示例

```typescript
// views/sqlconsole/SqlConsole.vue
<script setup lang="ts">
import { ref, computed, onMounted } from 'vue'
import { useSqlConsoleStore } from '@/stores/sqlconsole'
import type { QueryResult } from '@/types/sqlconsole'

const store = useSqlConsoleStore()
const sqlText = ref('')
const loading = ref(false)
const results = ref<QueryResult[]>([])

const canExecute = computed(() => {
  return sqlText.value.trim().length > 0 && !loading.value
})

const executeQuery = async () => {
  if (!canExecute.value) return
  
  loading.value = true
  try {
    results.value = await store.executeQuery(sqlText.value)
  } catch (error) {
    console.error('Query failed:', error)
  } finally {
    loading.value = false
  }
}

onMounted(() => {
  store.loadHistory()
})
</script>

<template>
  <div class="sql-console">
    <el-input
      v-model="sqlText"
      type="textarea"
      :rows="10"
      placeholder="输入 SQL 语句..."
    />
    <el-button
      type="primary"
      :loading="loading"
      :disabled="!canExecute"
      @click="executeQuery"
    >
      执行查询
    </el-button>

  </div>
</template>
```

## 现代化特性应用

### 1. Composition API

```typescript
// composables/useQuery.ts
import { ref, computed } from 'vue'
import type { Ref } from 'vue'

export function useQuery<T>(fetchFn: () => Promise<T>) {
  const data: Ref<T | null> = ref(null)
  const loading = ref(false)
  const error = ref<Error | null>(null)
  
  const execute = async () => {
    loading.value = true
    error.value = null
    try {
      data.value = await fetchFn()
    } catch (e) {
      error.value = e as Error
    } finally {
      loading.value = false
    }
  }
  
  const isSuccess = computed(() => !loading.value && !error.value && data.value !== null)
  
  return {
    data,
    loading,
    error,
    isSuccess,
    execute
  }
}
```

### 2. Pinia Store

```typescript
// stores/sqlconsole.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import type { QueryHistory, QueryResult } from '@/types/sqlconsole'
import { executeQuery as apiExecuteQuery } from '@/api/sqlconsole'

export const useSqlConsoleStore = defineStore('sqlconsole', () => {
  // State
  const history = ref<QueryHistory[]>([])
  const currentResult = ref<QueryResult | null>(null)
  
  // Getters
  const recentQueries = computed(() => {
    return history.value.slice(0, 10)
  })
  
  // Actions
  const executeQuery = async (sql: string) => {
    const result = await apiExecuteQuery(sql)
    currentResult.value = result
  
    history.value.unshift({
      sql,
      timestamp: Date.now(),
      success: true
    })
  
    return result
  }
  
  const loadHistory = () => {
    const saved = localStorage.getItem('sql-history')
    if (saved) {
      history.value = JSON.parse(saved)
    }
  }
  
  return {
    history,
    currentResult,
    recentQueries,
    executeQuery,
    loadHistory
  }
})
```
