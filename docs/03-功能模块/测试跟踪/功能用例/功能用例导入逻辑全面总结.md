# 功能用例导入逻辑全面总结

## 一、两个核心开关

### 1. 自定义ID开关（项目级配置）
- 位置：项目设置 → 测试跟踪 → 启用自定义ID
- 开启后：用例显示用户自定义的ID（如 TC-001）
- 关闭时：用例显示系统自增数字ID（如 1, 2, 3...）

### 2. 导入类型（导入时选择）
- **导入新建**：创建全新用例
- **导入更新**：根据ID匹配已有用例并更新

---

## 二、四种组合场景

| 场景 | 自定义ID | 导入类型 | ID列要求 | 核心行为 |
|:----:|:--------:|:--------:|:--------:|----------|
| A | 关闭 | 新建 | 不需要 | 系统自动分配ID，直接创建 |
| B | 关闭 | 更新 | 必填数字 | 用系统ID匹配，找到就更新 |
| C | 开启 | 新建 | 必填 | 用自定义ID创建，不能重复 |
| D | 开启 | 更新 | 必填 | 用自定义ID匹配，找到就更新 |

---

## 三、场景A：关闭自定义ID + 导入新建

**最简单的场景**

### 操作流程
1. 下载模板（模板中没有ID列）
2. 填写用例名称、模块、步骤等
3. 选择"导入新建"上传

### 系统行为
- 自动为每条用例分配递增的数字ID
- 检查用例是否重复（名称+模块+步骤组合判断）

### 可能的错误
| 错误提示 | 原因 | 解决方法 |
|----------|------|----------|
| 用例已存在 | 数据库中有相同用例 | 修改用例名称或模块路径 |
| Excel中用例重复 | 同一文件中有重复行 | 删除重复的行 |
| 模块名不能为空 | 模块路径格式错误 | 检查模块路径，如 /模块1/子模块 |

---

## 四、场景B：关闭自定义ID + 导入更新

**用系统数字ID更新用例**

### 操作流程
1. 先导出现有用例（导出文件包含系统ID列）
2. 修改需要更新的内容（不要改ID列）
3. 选择"导入更新"上传

### 系统行为
- 读取Excel中的ID列（数字）
- 在数据库中查找对应num的用例
- 找到则更新，找不到则报错

### 可能的错误
| 错误提示 | 原因 | 解决方法 |
|----------|------|----------|
| ID必填 | ID列为空 | 填写系统分配的数字ID |
| ID格式不正确 | ID不是有效数字 | 确保ID是纯数字，如 1, 2, 3 |
| 自定义ID不存在 | 数据库中没有这个ID | 检查ID是否正确，或改用导入新建 |
| Excel中ID重复 | 同一文件中ID重复 | 删除重复ID的行 |

### 常见踩坑
- ❌ 手动填写ID想创建新用例 → 会报"ID不存在"
- ❌ 修改了ID列的值 → 会找不到原用例
- ✅ 正确做法：先导出获取ID，只改内容不改ID

---

## 五、场景C：开启自定义ID + 导入新建

**用自定义ID创建新用例**

### 操作流程
1. 下载模板（模板中有ID列）
2. 填写自定义ID（如 TC-001, CASE-001）
3. 填写用例内容
4. 选择"导入新建"上传

### 系统行为
- 使用Excel中填写的ID作为用例的自定义ID
- 同时系统也会分配一个内部数字ID（用户不可见）
- 检查自定义ID是否已存在

### 可能的错误
| 错误提示 | 原因 | 解决方法 |
|----------|------|----------|
| ID必填 | ID列为空 | 必须填写自定义ID |
| 自定义ID已存在 | 数据库中已有这个ID | 换一个不重复的ID |
| Excel中ID重复 | 同一文件中ID重复 | 确保每行ID唯一 |

### 常见踩坑
- ❌ 不填ID列 → 会报"ID必填"
- ❌ 填写已存在的ID → 会报"ID已存在"
- ❌ 想更新已有用例 → 应该选"导入更新"而不是"导入新建"

---

## 六、场景D：开启自定义ID + 导入更新

**用自定义ID更新用例**

### 操作流程
1. 先导出现有用例（导出文件包含自定义ID列）
2. 修改需要更新的内容（不要改ID列）
3. 选择"导入更新"上传

### 系统行为
- 读取Excel中的ID列（自定义ID）
- 在数据库中查找对应customNum的用例
- 找到则更新，找不到则报错

### 可能的错误
| 错误提示 | 原因 | 解决方法 |
|----------|------|----------|
| ID必填 | ID列为空 | 填写自定义ID |
| 自定义ID不存在 | 数据库中没有这个ID | 检查ID是否正确，或改用导入新建 |
| Excel中ID重复 | 同一文件中ID重复 | 删除重复ID的行 |

## 七、边界情况详解

### 7.1 自定义ID开启 + 导入新建 + 不填ID

**结果**：报错"ID必填"，整行数据无法导入

**原因**：开启自定义ID后，系统认为用户需要自己管理ID，所以强制要求填写

**解决**：
- 方案1：填写自定义ID
- 方案2：如果不想管理ID，关闭项目的自定义ID开关

---

### 7.2 自定义ID关闭 + 导入更新 + ID不存在

**结果**：报错"自定义ID不存在"，整行数据无法导入

**原因**：导入更新的本质是"根据ID找到用例再更新"，找不到就无法更新

**注意**：这里不会自动创建新用例！想创建新用例必须选"导入新建"

**解决**：
- 检查ID是否填错
- 确认该用例是否已被删除
- 如果是想创建新用例，改用"导入新建"

---

### 7.3 自定义ID关闭 + 导入更新 + ID填的是字母

**结果**：报错"ID格式不正确"

**原因**：关闭自定义ID时，系统使用数字num作为标识，必须是纯数字

**解决**：确保ID列只填数字，如 1, 2, 100

---

### 7.4 自定义ID开启 + 导入新建 + ID已存在

**结果**：报错"自定义ID已存在"，整行数据无法导入

**原因**：新建模式下，自定义ID必须唯一，不能和已有用例重复

**注意**：这里不会自动变成更新！想更新必须选"导入更新"

**解决**：
- 换一个不重复的ID
- 如果是想更新已有用例，改用"导入更新"

---

### 7.5 自定义ID开启 + 导入更新 + ID不存在

**结果**：报错"自定义ID不存在"，整行数据无法导入

**原因**：更新模式必须找到已有用例才能更新

**注意**：这里不会自动创建新用例！

**解决**：
- 检查ID是否填错
- 如果是想创建新用例，改用"导入新建"

---

### 7.6 导入时Excel中同一ID出现多次

**结果**：第二次出现时报错"Excel中ID重复"

**原因**：无论新建还是更新，同一文件中不允许ID重复

**解决**：删除重复的行，或者分批导入

---

### 7.7 导入新建时用例完全相同

**结果**：报错"用例已存在"

**判断标准**：名称 + 模块路径 + 步骤内容 完全相同才算重复

**注意**：只改名称或只改模块就不算重复，可以导入

---

### 7.8 导入更新时不选版本

**结果**：更新最新版本（latest=true）的用例

**如果该ID有多个版本**：只更新标记为"最新"的那个版本

---

### 7.9 导入更新时选了一个不存在的版本

**结果**：创建该版本的新用例，但保持相同的ID和num

**例如**：
- 数据库中有 TC-001 的 v1.0 版本
- 导入更新时选择 v2.0 版本
- 系统会创建 TC-001 的 v2.0 版本（新记录，但ID相同）

---

## 八、版本相关逻辑

### 导入时选择版本
- 不选版本：新建时用默认版本，更新时更新最新版本
- 选择版本：新建时创建到指定版本，更新时更新指定版本

### 更新时的版本处理
- 如果指定版本的用例存在 → 直接更新
- 如果指定版本的用例不存在 → 创建该版本的新用例（保持相同ID）

---

## 九、"继续上传"功能

当导入文件中部分数据有错误时：
1. 系统显示错误列表
2. 用户可以点击"继续上传"
3. 系统会跳过有错误的行，只导入正确的数据

---

## 十、完整决策树

```
用户点击导入
    │
    ├─ 选择"导入新建"
    │      │
    │      ├─ 自定义ID关闭
    │      │      └─ 不需要填ID → 系统自动分配 → 检查用例是否重复 → 创建
    │      │
    │      └─ 自定义ID开启
    │             │
    │             ├─ 不填ID → ❌ 报错"ID必填"
    │             ├─ 填的ID已存在 → ❌ 报错"ID已存在"
    │             └─ 填的ID不存在 → 检查用例是否重复 → 创建
    │
    └─ 选择"导入更新"
           │
           ├─ 不填ID → ❌ 报错"ID必填"
           │
           ├─ 自定义ID关闭
           │      │
           │      ├─ ID不是数字 → ❌ 报错"ID格式不正确"
           │      ├─ ID在数据库不存在 → ❌ 报错"ID不存在"
           │      └─ ID存在 → 更新该用例
           │
           └─ 自定义ID开启
                  │
                  ├─ ID在数据库不存在 → ❌ 报错"ID不存在"
                  └─ ID存在 → 更新该用例
```

## 十一、完整试错清单

### 通用错误
| 操作 | 错误 | 原因 |
|------|------|------|
| 上传非Excel文件 | 格式错误 | 只支持 .xls/.xlsx |
| 文件超过100MB | 文件过大 | 分批导入 |
| 模块路径有空节点 | 模块名不能为空 | 如 /模块1//子模块 是错的 |
| 模块名超100字 | 模块名过长 | 缩短模块名称 |

### 导入新建时
| 操作 | 错误 | 原因 |
|------|------|------|
| 开启自定义ID但不填ID | ID必填 | 开启后必须填 |
| 填写已存在的自定义ID | ID已存在 | 换一个ID |
| 同一文件中ID重复 | Excel中ID重复 | 确保唯一 |
| 用例完全相同 | 用例已存在 | 修改内容或模块 |

### 导入更新时
| 操作 | 错误 | 原因 |
|------|------|------|
| 不填ID | ID必填 | 更新必须有ID |
| 填写不存在的ID | ID不存在 | 检查ID或改用新建 |
| 关闭自定义ID时填非数字 | ID格式不正确 | 必须是数字 |
| 同一文件中ID重复 | Excel中ID重复 | 确保唯一 |

## 十二、常见误解澄清

| 误解 | 实际情况 |
|------|----------|
| 导入更新时ID不存在会自动创建 | ❌ 不会，会报错 |
| 导入新建时ID已存在会自动更新 | ❌ 不会，会报错 |
| 关闭自定义ID后可以用字母ID更新 | ❌ 不行，必须是数字 |
| 开启自定义ID后可以不填ID新建 | ❌ 不行，必须填写 |
| 同一文件中可以有重复ID | ❌ 不行，会报错 |

---

## 十三、最佳实践

### 导入新建
1. 关闭自定义ID时：直接填内容，不用管ID
2. 开启自定义ID时：规划好ID命名规则，确保不重复

### 导入更新
1. 先导出现有用例
2. 在导出文件上修改（保留ID列不变）
3. 选择导入更新上传

### 批量操作建议
- 单次不超过5000条
- 大量数据分批处理
- 导入前备份数据
