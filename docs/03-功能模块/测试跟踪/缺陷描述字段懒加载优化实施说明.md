# 缺陷描述字段懒加载优化实施说明

## 一、问题背景

### 1.1 性能问题

在缺陷管理列表页面，缺陷描述字段支持富文本预览功能。原有实现方式存在以下问题：

- **列表加载时即渲染所有描述内容**：每条缺陷的描述字段都会立即渲染MsMarkDownText组件
- **图片资源提前加载**：描述中的图片会在列表渲染时全部加载，即使用户未点击预览
- **性能影响**：当列表有大量缺陷且描述中包含多张图片时，会导致：
  - 页面加载时间显著增加
  - 网络请求数量激增
  - 浏览器内存占用增加
  - 用户体验下降

### 1.2 用户场景分析

实际使用中，用户的行为特征：
- 大多数情况下只浏览列表，不会查看每条缺陷的详细描述
- 只有在需要了解具体内容时才会点击"预览"按钮
- 描述中的图片资源往往较大，提前加载造成资源浪费

---

## 二、优化方案

### 2.1 核心思路

**懒加载策略**：将描述内容的渲染延迟到用户真正需要查看时才执行

- **触发时机**：鼠标悬停到"预览"按钮，Popover显示时
- **加载范围**：仅加载当前预览的缺陷描述
- **缓存策略**：首次加载后保持渲染状态，避免重复加载

### 2.2 技术实现

#### 方案选择：基于Popover的@show事件

利用Element UI的Popover组件的生命周期事件：
- `@show`：Popover显示时触发
- 在此事件中设置标记，控制MsMarkDownText组件的渲染

#### 组件设计

创建新组件 `MsReviewTableItemLazy.vue`，替代原有的 `MsReviewTableItem.vue`：

**关键特性**：
1. 使用 `v-if` 控制MsMarkDownText的渲染
2. 通过 `contentLoaded` 标记控制加载状态
3. 首次显示时设置标记为true，触发内容渲染
4. 加载前显示占位提示（loading图标）

## 三、实施步骤

### 3.1 创建懒加载组件

**文件位置**：`test-track/frontend/src/business/issue/MsReviewTableItemLazy.vue`

**核心逻辑**：
- 初始状态：`contentLoaded = false`，不渲染MsMarkDownText
- Popover显示：触发 `handlePopoverShow()` 方法
- 设置标记：`contentLoaded = true`，开始渲染内容
- 占位显示：未加载时显示loading提示

**优势**：
- 完全兼容原有组件的props接口
- 不影响其他使用MsReviewTableItem的地方
- 可独立维护和升级

## 七、注意事项

### 7.1 开发注意事项

1. **组件命名**：
   - 新组件命名为 `MsReviewTableItemLazy`，明确表示懒加载特性
   - 避免与原组件混淆

2. **状态管理**：
   - `contentLoaded` 标记在组件实例内部管理
   - 不需要全局状态或父组件传递

3. **事件处理**：
   - 使用 `@show` 而非 `@mouseenter`，确保Popover真正显示时才加载
   - 使用 `$nextTick` 确保DOM更新后再设置标记

4. **占位提示**：
   - loading提示样式与系统整体风格保持一致
   - 提示文案使用国际化（`$t('commons.loading')`）

### 7.2 升级注意事项

**版本升级时的处理**：
1. 检查官方 `MsReviewTableItem` 是否有更新
2. 如有更新，同步修改 `MsReviewTableItemLazy`
3. 重点关注MsMarkDownText组件的props变化
4. 测试验证懒加载功能是否正常

**回滚方案**：
- 如遇问题，可快速回滚到原组件

## 八、技术细节

### 8.1 渲染时机控制

**v-if vs v-show**：
- 选择 `v-if`：完全不渲染DOM，节省内存
- 不选择 `v-show`：虽然切换快，但初始就会渲染，无法实现懒加载

**$nextTick的作用**：
- 确保Vue完成DOM更新后再设置标记
- 避免渲染时序问题
- 保证Popover已完全显示

### 8.2 Popover事件机制

**@show事件特点**：
- 在Popover完全显示后触发
- 比@mouseenter更可靠（考虑了动画时间）
- 不会因鼠标快速移动而重复触发

**trigger="hover"**：
- 鼠标悬停触发，符合用户习惯
- 无需点击，交互更自然
- 移开鼠标自动关闭

### 8.3 组件通信

**Props传递**：
- `data`：缺陷数据对象或displayValueMap
- `prop`：要显示的字段名（description或自定义字段ID）

**无需Emit**：
- 预览功能为只读，不需要向父组件传递事件
- 简化组件通信逻辑
