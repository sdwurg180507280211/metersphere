# 用户组迁移影响分析

## 📊 数据库变更范围

### 涉及的表

| 表名 | 变更内容 | 影响范围 |
|------|---------|---------|
| `group` | 删除旧记录，插入新记录（ID 变更） | 2 条记录 |
| `user_group` | 更新 `group_id` 字段 | 所有关联到开发/测试人员组的用户 |
| `user_group_permission` | 更新 `group_id` 字段 | 所有关联到开发/测试人员组的权限 |

### 数据量统计（测试环境）

```sql
-- 查看受影响的数据量
SELECT 
    '用户组' as table_name,
    COUNT(*) as affected_rows
FROM `group` 
WHERE name IN ('开发人员组', '测试人员组')

UNION ALL

SELECT 
    '用户关联' as table_name,
    COUNT(*) as affected_rows
FROM user_group ug
INNER JOIN `group` g ON ug.group_id = g.id
WHERE g.name IN ('开发人员组', '测试人员组')

UNION ALL

SELECT 
    '权限关联' as table_name,
    COUNT(*) as affected_rows
FROM user_group_permission ugp
INNER JOIN `group` g ON ugp.group_id = g.id
WHERE g.name IN ('开发人员组', '测试人员组');
```

## 🔍 迁移前后对比

### 迁移前

```sql
-- 用户组表
id: a1b2c3d4-e5f6-7890-abcd-ef1234567890
name: 开发人员组
system: 0  -- 普通用户组
scope_id: <项目ID>

-- 用户关联表
user_id: user123
group_id: a1b2c3d4-e5f6-7890-abcd-ef1234567890
source_id: project456

-- 权限关联表
group_id: a1b2c3d4-e5f6-7890-abcd-ef1234567890
permission_id: PROJECT:READ+EDIT
```

### 迁移后

```sql
-- 用户组表
id: developer  -- 固定 ID
name: 开发人员组
system: 1  -- 系统用户组
scope_id: <保持原有值>  -- 重要：保留原有的 scope_id

-- 用户关联表（自动更新）
user_id: user123
group_id: developer  -- 已更新
source_id: project456

-- 权限关联表（自动更新）
group_id: developer  -- 已更新
permission_id: PROJECT:READ+EDIT
```

**关键变化**：
1. 用户组 ID：`随机UUID` → `developer`/`tester`
2. 用户组类型：`普通用户组(0)` → `系统用户组(1)`
3. 作用域：**保持不变**（工作空间级别仍然是工作空间级别，全局级别仍然是全局级别）
4. 所有关联关系自动更新，保持一致

**重要说明**：
- ✅ `scope_id` 字段会保留原有的值
- ✅ 如果原来是工作空间ID，迁移后仍然是工作空间ID
- ✅ 如果原来是 `global`，迁移后仍然是 `global`
- ✅ 用户组的作用域范围不会改变

## 🛡️ 安全性保障

### 1. 事务保护
```sql
START TRANSACTION;
-- 所有操作
COMMIT;  -- 或 ROLLBACK
```
- 所有操作在一个事务中执行
- 如果任何步骤失败，自动回滚
- 保证数据一致性

### 2. 数据备份
```sql
-- 备份用户组表
CREATE TABLE `group_backup_20260126` AS ...

-- 备份用户关联表
CREATE TABLE `user_group_backup_20260126` AS ...

-- 备份权限关联表
CREATE TABLE `user_group_permission_backup_20260126` AS ...
```
- 迁移前自动备份所有相关数据
- 如果出现问题，可以快速恢复

### 3. 验证步骤
```sql
-- 验证用户组
SELECT id, name, system FROM `group` WHERE id IN ('developer', 'tester');

-- 验证用户关联
SELECT COUNT(*) FROM user_group WHERE group_id IN ('developer', 'tester');

-- 验证权限关联
SELECT COUNT(*) FROM user_group_permission WHERE group_id IN ('developer', 'tester');
```
- 迁移后自动验证数据完整性
- 确保所有关联关系正确更新

## 🚨 风险评估

### 风险等级：低

| 风险项 | 风险等级 | 缓解措施 |
|--------|---------|---------|
| 数据丢失 | 极低 | 事务保护 + 数据备份 |
| 数据不一致 | 极低 | 事务保护 + 验证步骤 |
| 用户无法登录 | 无 | 不涉及用户认证 |
| 权限丢失 | 极低 | 自动更新关联关系 |
| 业务中断 | 极低 | 执行时间短（< 30 秒） |
| 回滚困难 | 低 | 有完整备份 |

### 最坏情况处理

**如果迁移失败**：
1. 事务自动回滚，数据恢复到迁移前状态
2. 或者手动执行 `ROLLBACK`

**如果迁移后发现问题**：
1. 从备份表恢复数据
2. 或者从数据库备份文件恢复
