# 银保通项目状态字段重复问题分析与修复

## 问题描述

用户反馈：银保通项目下，缺陷状态无法从"新建"流转到"已接收"。

## 问题分析

### 1. 数据库调查

通过查询数据库发现，银保通项目的每个缺陷都存在**两条状态字段记录**：

```sql
-- bug2003 的状态字段
| resource_id                          | field_id                             | value      |
|--------------------------------------|--------------------------------------|------------|
| f8300bf4-e5c1-45b4-82fa-040c0788de29 | beb57501-19c8-4ca3-8dfb-2cef7c0ea087 | "new"      |  -- 全局字段
| f8300bf4-e5c1-45b4-82fa-040c0788de29 | d755edd6-7fc6-4a4c-9de9-977befa4738c | "accepted" |  -- 项目字段
```

### 2. 字段来源

系统中存在两个"状态"字段：

| field_id | 类型 | project_id | global | 创建时间 |
|----------|------|------------|--------|----------|
| `beb57501-19c8-4ca3-8dfb-2cef7c0ea087` | 全局系统字段 | global | 1 | 2025-01-04 |
| `d755edd6-7fc6-4a4c-9de9-977befa4738c` | 项目专属字段 | 84d9cda2-0cd0-4e86-a34f-45f062f580cc | 0 | 2025-01-11 |

### 3. 问题根源

- **模板绑定**：全局默认模板绑定的是全局字段 `beb57501...`
- **前端更新**：前端更新状态时使用的是项目字段 `d755edd6...`
- **前端显示**：前端显示状态时读取的是全局字段 `beb57501...`（值始终是 `"new"`）
- **数据不一致**：
  - 全局字段值：`"new"`（未更新）
  - 项目字段值：`"accepted"`、`"resolved"`（实际状态）
  - issues 表 status 字段：`"new"`（未同步）

### 4. 影响范围

银保通项目的所有缺陷（3个）：
- bug2001：显示"新建"，实际应该是"已解决"
- bug2002：显示"新建"，实际应该是"已解决"
- bug2003：显示"新建"，实际应该是"已接收"

## 解决方案

### 方案选择

**方案A：删除项目专属字段，统一使用全局字段**（已采用）

优点：
- 符合系统设计（模板绑定的是全局字段）
- 避免字段冲突
- 数据一致性好

缺点：
- 需要迁移数据

**方案B：修改模板绑定，使用项目专属字段**（未采用）

优点：
- 不需要迁移数据

缺点：
- 违背系统设计（全局模板不应该绑定项目字段）
- 其他项目无法使用该模板

### 修复步骤

#### 1. 备份数据

```sql
CREATE TABLE IF NOT EXISTS custom_field_issues_backup_20250126 AS
SELECT * FROM custom_field_issues
WHERE field_id = 'd755edd6-7fc6-4a4c-9de9-977befa4738c';
```

备份记录数：3条

#### 2. 迁移数据到全局字段

```sql
UPDATE custom_field_issues cfi
JOIN (
  SELECT resource_id, value
  FROM custom_field_issues
  WHERE field_id = 'd755edd6-7fc6-4a4c-9de9-977befa4738c'
) AS project_field ON cfi.resource_id = project_field.resource_id
SET cfi.value = project_field.value
WHERE cfi.field_id = 'beb57501-19c8-4ca3-8dfb-2cef7c0ea087';
```

更新记录数：3条

#### 3. 同步 issues 表的 status 字段

```sql
UPDATE issues i
JOIN custom_field_issues cfi ON i.id = cfi.resource_id
SET i.status = TRIM(BOTH '"' FROM cfi.value)
WHERE cfi.field_id = 'beb57501-19c8-4ca3-8dfb-2cef7c0ea087'
  AND i.project_id = '84d9cda2-0cd0-4e86-a34f-45f062f580cc';
```

更新记录数：3条

#### 4. 删除项目专属字段数据

```sql
DELETE FROM custom_field_issues
WHERE field_id = 'd755edd6-7fc6-4a4c-9de9-977befa4738c';
```

删除记录数：3条

#### 5. 删除项目专属字段的模板关联

```sql
DELETE FROM custom_field_template
WHERE field_id = 'd755edd6-7fc6-4a4c-9de9-977befa4738c';
```

删除记录数：1条

#### 6. 删除项目专属字段定义

```sql
DELETE FROM custom_field
WHERE id = 'd755edd6-7fc6-4a4c-9de9-977befa4738c';
```

删除记录数：1条

## 问题根本原因

**系统允许创建与全局字段同名的项目专属字段，导致字段冲突。**

### 可能的触发场景

1. 用户在项目设置中手动创建了"状态"字段
2. 系统升级或数据迁移时自动创建了重复字段
3. 模板复制时错误地创建了项目专属字段

### 预防措施

1. **前端校验**：创建项目字段时，检查是否与全局系统字段同名
2. **后端校验**：在 `CustomFieldService` 中添加字段名唯一性校验
3. **数据库约束**：添加唯一索引（需要考虑兼容性）
4. **定期检查**：添加数据一致性检查脚本
