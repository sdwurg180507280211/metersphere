# MeterSphere 密码加密逻辑说明文档

## 1. 概述

MeterSphere 使用 **RSA 非对称加密算法**对用户登录时的用户名和密码进行加密传输，确保敏感信息在传输过程中的安全性。

## 2. 加密流程

### 2.1 整体流程图

```
前端登录页面
    ↓
1. 获取RSA公钥 (/is-login 接口)
    ↓
2. 使用公钥加密用户名和密码
    ↓
3. 发送加密后的数据到后端 (/signin 接口)
    ↓
后端接收
    ↓
4. 使用私钥解密用户名和密码
    ↓
5. 验证用户身份
```

### 2.2 详细步骤

#### 步骤 1：前端获取 RSA 公钥

**接口**：`GET /is-login`

**后端实现**：
- 文件：`framework/sdk-parent/sdk/src/main/java/io/metersphere/controller/LoginController.java`
- 文件：`framework/gateway/src/main/java/io/metersphere/gateway/controller/LoginController.java`

**逻辑**：
1. 后端检查是否存在已保存的 RSA 密钥对
2. 如果不存在，生成新的 RSA 密钥对（1024位）
3. 将密钥对序列化后存储到文件系统（`file_metadata` 表，ID 为 `ms.login.rsa.key`）
4. 返回公钥给前端

**前端处理**：
- 文件：`framework/sdk-parent/frontend/src/business/login/index.vue`
- 将公钥保存到 `localStorage` 中，key 为 `publicKey`

```javascript
// 前端代码示例
localStorage.setItem('publicKey', data.message);
```

#### 步骤 2：前端加密用户名和密码

**实现位置**：
- 文件：`framework/sdk-parent/frontend/src/utils/token.js`
- 使用库：`JSEncrypt`（基于 RSA 的 JavaScript 加密库）

**加密函数**：
```javascript
export function publicKeyEncrypt(input, publicKey) {
  let jsencrypt = new JSEncrypt({default_key_size: 1024});
  jsencrypt.setPublicKey(publicKey);
  return jsencrypt.encrypt(input);
}
```

**登录时使用**：
```javascript
// framework/sdk-parent/frontend/src/business/login/index.vue
let publicKey = localStorage.getItem('publicKey');
let form = {
  username: publicKeyEncrypt(this.form.username, publicKey),
  password: publicKeyEncrypt(this.form.password, publicKey),
  authenticate: this.form.authenticate,
};
```

#### 步骤 3：后端解密用户名和密码

**实现位置**：
- 文件：`framework/sdk-parent/sdk/src/main/java/io/metersphere/request/LoginRequest.java`

**解密逻辑**：
```java
public String getUsername() {
    try {
        RsaKey rsaKey = RsaUtil.getRsaKey();
        return RsaUtil.privateDecrypt(username, rsaKey.getPrivateKey());
    } catch (Exception e) {
        return username;
    }
}

public String getPassword() {
    try {
        RsaKey rsaKey = RsaUtil.getRsaKey();
        return RsaUtil.privateDecrypt(password, rsaKey.getPrivateKey());
    } catch (Exception e) {
        return password;
    }
}
```

**说明**：
- 使用私钥解密前端传来的加密数据
- 如果解密失败（异常），则返回原始值（兼容未加密的情况）

## 3. 核心组件

### 3.1 RSA 工具类

**文件**：`framework/sdk-parent/sdk/src/main/java/io/metersphere/commons/utils/RsaUtil.java`

**主要方法**：

| 方法 | 说明 | 参数 |
|------|------|------|
| `getRsaKey()` | 获取 RSA 密钥对（从缓存或生成新的） | - |
| `createKeys(int keySize)` | 创建指定长度的 RSA 密钥对 | keySize: 密钥长度（默认1024） |
| `publicEncrypt(String originalText, String publicKey)` | 公钥加密 | originalText: 原文, publicKey: 公钥 |
| `privateDecrypt(String cipherText, String privateKey)` | 私钥解密 | cipherText: 密文, privateKey: 私钥 |
| `rsaSplitCodec()` | RSA 分块加解密处理 | 处理超过密钥长度的数据 |

**密钥生成**：
- 算法：RSA
- 密钥长度：1024 位（默认）
- 编码格式：
  - 公钥：X509 编码（Base64）
  - 私钥：PKCS#8 编码（Base64）

### 3.2 RSA 密钥对象

**文件**：`framework/sdk-parent/sdk/src/main/java/io/metersphere/commons/utils/RsaKey.java`

```java
@Setter
@Getter
public class RsaKey implements Serializable {
    private String publicKey;   // 公钥（Base64编码）
    private String privateKey; // 私钥（Base64编码）
}
```

### 3.3 RSA 配置类

**文件**：`framework/sdk-parent/sdk/src/main/java/io/metersphere/autoconfigure/RsaConfig.java`

**功能**：
- 应用启动时自动加载 RSA 密钥对
- 从文件系统读取或生成新的密钥对
- 将密钥对缓存到内存中

### 3.4 文件服务

**文件**：`framework/sdk-parent/sdk/src/main/java/io/metersphere/service/FileService.java`

**方法**：`checkRsaKey()`

**逻辑**：
1. 检查文件系统中是否存在 RSA 密钥（ID: `ms.login.rsa.key`）
2. 如果存在，反序列化并返回
3. 如果不存在，生成新的密钥对并保存到文件系统
4. 返回密钥对对象

**存储位置**：
- 数据库表：`file_metadata`
- 文件 ID：`ms.login.rsa.key`
- 文件类型：`RSA_KEY`
- 存储格式：序列化的 `RsaKey` 对象（字节数组）

## 4. 技术细节

### 4.1 加密算法参数

| 参数 | 值 |
|------|-----|
| 算法 | RSA |
| 密钥长度 | 1024 位 |
| 字符编码 | UTF-8 |
| 编码格式 | Base64 |
| 填充方式 | PKCS1Padding（JSEncrypt 默认） |

### 4.2 数据分块处理

由于 RSA 加密有长度限制，对于超过密钥长度的数据需要进行分块处理：

**加密时**：
- 最大块大小 = (密钥长度 / 8) - 11
- 对于 1024 位密钥：最大块 = 117 字节

**解密时**：
- 最大块大小 = 密钥长度 / 8
- 对于 1024 位密钥：最大块 = 128 字节

**实现**：`rsaSplitCodec()` 方法自动处理数据分块

### 4.3 密钥管理

**密钥生成时机**：
- 首次访问 `/is-login` 接口时
- 如果文件系统中不存在密钥，自动生成

**密钥存储**：
- 持久化：文件系统（通过 `file_metadata` 表）
- 内存缓存：`RsaUtil.rsaKey` 静态变量

**密钥生命周期**：
- 应用启动时加载到内存
- 应用运行期间一直使用同一对密钥
- 除非手动删除文件系统中的密钥，否则不会重新生成

## 5. 安全考虑

### 5.1 优点

1. **传输安全**：密码在传输过程中是加密的，即使被截获也无法直接获取明文
2. **非对称加密**：公钥可以公开，私钥仅服务器持有
3. **密钥持久化**：密钥对保存在服务器端，不会丢失

### 5.2 注意事项

1. **密钥长度**：当前使用 1024 位密钥，建议升级到 2048 位以提高安全性
2. **密钥轮换**：当前实现中密钥不会自动轮换，建议定期更换密钥对
3. **HTTPS**：虽然密码已加密，但仍建议使用 HTTPS 协议传输
4. **私钥保护**：确保私钥文件的安全，防止泄露

## 7. 使用示例

### 7.1 前端加密示例

```javascript
import { publicKeyEncrypt } from '@/utils/token';

// 获取公钥
const publicKey = localStorage.getItem('publicKey');

// 加密用户名
const encryptedUsername = publicKeyEncrypt('admin', publicKey);

// 加密密码
const encryptedPassword = publicKeyEncrypt('password123', publicKey);

// 发送到后端
const loginData = {
  username: encryptedUsername,
  password: encryptedPassword,
  authenticate: 'LOCAL'
};
```

### 7.2 后端解密示例

```java
// LoginRequest 对象会自动解密
LoginRequest request = new LoginRequest();
request.setUsername(encryptedUsername);  // 加密后的用户名
request.setPassword(encryptedPassword);   // 加密后的密码

// 获取时会自动解密
String username = request.getUsername();  // 解密后的用户名
String password = request.getPassword();  // 解密后的密码
```

## 8. 故障排查

### 8.1 常见问题

**问题 1：登录时提示解密失败**

**可能原因**：
- 前端使用的公钥与后端私钥不匹配
- 密钥文件损坏

**解决方案**：
1. 清除浏览器 `localStorage` 中的 `publicKey`
2. 删除数据库中的密钥文件（`file_metadata` 表中 ID 为 `ms.login.rsa.key` 的记录）
3. 重启应用，系统会自动生成新的密钥对

**问题 2：加密后的数据过长**

**原因**：RSA 加密有长度限制

**解决方案**：
- 系统已自动实现分块处理，正常情况下不会出现此问题
- 如果用户名或密码过长（超过 117 字符），可能需要调整密钥长度

### 8.2 调试方法

**查看公钥**：
```javascript
// 浏览器控制台
console.log(localStorage.getItem('publicKey'));
```

**查看加密结果**：
```javascript
// 浏览器控制台
import { publicKeyEncrypt } from '@/utils/token';
const publicKey = localStorage.getItem('publicKey');
console.log(publicKeyEncrypt('test', publicKey));
```

**后端日志**：
- 查看应用启动日志，确认 RSA 密钥是否成功加载
- 查看登录接口日志，确认解密是否成功

## 9. 升级建议

### 9.1 密钥长度升级

当前使用 1024 位密钥，建议升级到 2048 位：

```java
// RsaUtil.java
public static RsaKey createKeys() throws NoSuchAlgorithmException {
    return createKeys(2048);  // 从 1024 改为 2048
}
```

**注意**：升级后需要删除旧的密钥文件，让系统重新生成。

### 9.2 密钥轮换机制

建议实现密钥轮换机制：
- 定期（如每 90 天）自动生成新密钥
- 保留旧密钥一段时间以兼容正在进行的会话
- 通知前端更新公钥

### 9.3 性能优化

对于高并发场景，可以考虑：
- 使用连接池缓存 RSA 密钥对象
- 使用更高效的加密库（如 BouncyCastle）
- 考虑使用 HTTPS + 对称加密的组合方案
