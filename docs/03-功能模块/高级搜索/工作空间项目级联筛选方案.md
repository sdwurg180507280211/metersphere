# 高级搜索-工作空间项目级联筛选方案

## 一、需求分析

### 1.1 功能目标

在高级搜索组件中添加"所属工作空间"和"所属项目"筛选条件，实现：

1. **多选支持**：工作空间和项目都支持多选
2. **级联响应**：选择工作空间后，项目下拉框自动过滤只显示该工作空间下的项目
3. **跨空间筛选**：支持同时选择多个工作空间，查看跨工作空间的数据
4. **跨项目筛选**：支持同时选择多个项目，查看跨项目的数据
5. **独立筛选**：可以只选工作空间（查看该空间下所有项目的数据），或只选项目（不限工作空间）

### 1.2 适用场景

- **缺陷管理页面**：查看跨项目的缺陷
- **功能用例页面**：查看跨项目的用例
- **测试计划页面**：查看跨项目的测试计划
- **其他需要跨项目查询的场景**

## 二、技术方案

### 2.1 方案选型

#### 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **方案一：新增级联选择组件** | 交互体验好，符合用户习惯 | 需要开发新组件，工作量大 | ⭐⭐⭐⭐ |
| **方案二：复用现有Select组件** | 开发量小，快速实现 | 级联逻辑需要手动处理 | ⭐⭐⭐⭐⭐ |
| **方案三：使用Element UI Cascader** | 原生支持级联 | 不支持多选多个父节点 | ⭐⭐ |

**最终选择**：**方案二（复用现有Select组件）**

**理由**：
1. 项目已有 `MsTableSearchSelect` 组件，功能完善
2. 通过监听工作空间变化，动态更新项目选项，实现级联效果
3. 开发量小，改动面小，符合二次开发原则
4. 灵活性高，支持独立筛选和级联筛选

// 所属项目（多选，支持级联）
export const PROJECT = {
  key: "projectIds",  // 注意：使用复数形式，表示支持多选
  name: 'MsTableSearchSelect',
  label: "所属项目",
  operator: {
    value: OPERATORS.IN.value,
    options: [OPERATORS.IN, OPERATORS.NOT_IN]
  },
  options: {
    url: "/project/list/related",  // 获取项目列表（支持按工作空间过滤）
    labelKey: "name",
    valueKey: "id",
    showLabel: option => {
      return option.label;
    }
  },
  props: {
    multiple: true,
    'collapse-tags': true
  },
  // 级联逻辑：监听工作空间变化
  cascadeKey: "workspaceIds",  // 依赖的字段key
  cascadeUpdate: function(component, workspaceIds) {
    // 当工作空间变化时，重新加载项目列表
    if (workspaceIds && workspaceIds.length > 0) {
      // 更新请求参数，只获取选中工作空间下的项目
      component.options.params = { workspaceIds: workspaceIds };
    } else {
      // 未选择工作空间时，显示所有项目
      component.options.params = {};
    }
    // 触发重新加载
    if (component.init && typeof component.init === 'function') {
      component.init();
    }
  }
};

// 将新字段添加到内置字段白名单
export const BUILTIN_ADV_SEARCH_KEYS = [
  // ... 现有字段
  'workspaceIds',  // 新增
  'projectIds',    // 新增
];

#### 2.3.2 在高级搜索组件中添加级联逻辑

**位置**：`framework/sdk-parent/frontend/src/components/new-ui/MsTableAdvSearch.vue`

在 `search()` 方法中添加级联处理逻辑：

```javascript
search() {
  this.conditionNum = 0;
  let condition = {};
  
  // 第一步：处理所有搜索条件
  this.optional.components.forEach((component) => {
    let value = component.value;
    if (Array.isArray(value)) {
      if (value.length > 0) {
        this.setCondition(condition, component);
        this.conditionNum += 1;
      }
    } else {
      if (value !== undefined && value !== null && value !== "") {
        this.setCondition(condition, component);
        this.conditionNum += 1;
      }
    }
  });

  // 第二步：处理级联逻辑（新增）
  this.handleCascadeLogic();

  // ... 其余代码保持不变
}

// 新增方法：处理级联逻辑
handleCascadeLogic() {
  this.optional.components.forEach((component) => {
    // 检查是否有级联配置
    if (component.cascadeKey && component.cascadeUpdate) {
      // 查找依赖的组件
      const cascadeComponent = this.optional.components.find(
        c => c.key === component.cascadeKey
      );
      if (cascadeComponent && cascadeComponent.value) {
        // 触发级联更新
        component.cascadeUpdate(component, cascadeComponent.value);
      }
    }
  });
}
```

**Service实现**：

```java
public List<Project> listRelatedProjects(ProjectRequest request) {
    ProjectExample example = new ProjectExample();
    ProjectExample.Criteria criteria = example.createCriteria();
    
    // 如果指定了工作空间ID，则只查询这些工作空间下的项目
    if (request.getWorkspaceIds() != null && !request.getWorkspaceIds().isEmpty()) {
        criteria.andWorkspaceIdIn(request.getWorkspaceIds());
    }
    
    example.setOrderByClause("create_time DESC");
    return projectMapper.selectByExample(example);
}
```

#### 2.4.3 修改查询接口：支持工作空间和项目筛选

**位置**：`test-track/backend/src/main/java/io/metersphere/service/IssuesService.java`

在 `list()` 方法中添加工作空间和项目筛选逻辑：

```java
public Pager<List<IssuesDao>> list(IssuesRequest request, int goPage, int pageSize) {
    // ... 现有代码
    
    // 处理工作空间筛选（新增）
    if (request.getCombine() != null) {
        Map<String, Object> combine = request.getCombine();
        
        // 工作空间筛选
        if (combine.containsKey("workspaceIds")) {
            Map<String, Object> workspaceFilter = (Map<String, Object>) combine.get("workspaceIds");
            List<String> workspaceIds = (List<String>) workspaceFilter.get("value");
            String operator = (String) workspaceFilter.get("operator");
            
            if (workspaceIds != null && !workspaceIds.isEmpty()) {
                // 根据工作空间ID查询项目ID列表
                List<String> projectIds = projectService.getProjectIdsByWorkspaceIds(workspaceIds);
                
                if ("in".equals(operator)) {
                    request.setProjectIds(projectIds);
                } else if ("not in".equals(operator)) {
                    // 排除这些工作空间下的项目
                    request.setExcludeProjectIds(projectIds);
                }
            }
        }
        
        // 项目筛选（新增）
        if (combine.containsKey("projectIds")) {
            Map<String, Object> projectFilter = (Map<String, Object>) combine.get("projectIds");
            List<String> projectIds = (List<String>) projectFilter.get("value");
            String operator = (String) projectFilter.get("operator");
            
            if (projectIds != null && !projectIds.isEmpty()) {
                if ("in".equals(operator)) {
                    request.setProjectIds(projectIds);
                } else if ("not in".equals(operator)) {
                    request.setExcludeProjectIds(projectIds);
                }
            }
        }
    }
    
    // ... 其余代码保持不变
}
```

## 四、级联逻辑详解

### 4.1 级联场景

| 场景 | 工作空间选择 | 项目选择 | 查询结果 |
|------|-------------|---------|---------|
| 场景1 | 未选择 | 未选择 | 查询所有数据 |
| 场景2 | 选择 [WS1] | 未选择 | 查询 WS1 下所有项目的数据 |
| 场景3 | 选择 [WS1, WS2] | 未选择 | 查询 WS1 和 WS2 下所有项目的数据 |
| 场景4 | 未选择 | 选择 [P1, P2] | 查询 P1 和 P2 的数据（不限工作空间） |
| 场景5 | 选择 [WS1] | 选择 [P1, P2] | 查询 P1 和 P2 的数据（P1、P2 必须属于 WS1） |
| 场景6 | 选择 [WS1, WS2] | 选择 [P1, P2, P3] | 查询 P1、P2、P3 的数据（这些项目必须属于 WS1 或 WS2） |

### 4.2 级联响应流程

```
用户选择工作空间 [WS1, WS2]
    ↓
触发 watch 监听 (workspaceIds 变化)
    ↓
调用 PROJECT.cascadeUpdate(component, ['WS1', 'WS2'])
    ↓
更新 component.options.params = { workspaceIds: ['WS1', 'WS2'] }
    ↓
调用 component.init() 重新加载项目列表
    ↓
发送请求: POST /project/list/related { workspaceIds: ['WS1', 'WS2'] }
    ↓
后端返回: [P1(WS1), P2(WS1), P3(WS2), P4(WS2)]
    ↓
更新项目下拉选项（只显示 WS1 和 WS2 下的项目）
    ↓
用户从过滤后的列表中选择项目
```

### 4.3 优先级规则

当同时选择工作空间和项目时：

1. **项目优先**：如果同时指定了工作空间和项目，以项目为准
2. **交集过滤**：只查询"既属于选中工作空间，又在选中项目列表中"的数据
3. **前端提示**：如果选中的项目不属于选中的工作空间，前端给出提示

## 五、性能优化

### 5.1 前端优化

1. **缓存工作空间列表**
   - 工作空间列表不常变化，可以缓存到 localStorage
   - 设置缓存过期时间（如1小时）

2. **防抖处理**
   - 工作空间变化时，延迟300ms再触发项目列表刷新
   - 避免用户快速切换时频繁请求

3. **懒加载**
   - 项目列表较多时，使用虚拟滚动
   - 或者分页加载项目列表

### 5.2 后端优化

1. **索引优化**
   - 确保 `project.workspace_id` 有索引
   - 确保 `issues.project_id` 有索引

2. **查询优化**
   - 使用 IN 查询时，限制 ID 数量（如最多50个）
   - 超过限制时，提示用户缩小筛选范围

3. **缓存优化**
   - 工作空间列表使用 Redis 缓存
   - 项目列表使用 Redis 缓存（按工作空间ID分组）

## 六、兼容性考虑

### 6.1 向后兼容

1. **不影响现有功能**
   - 新增字段不影响现有的高级搜索配置
   - 现有页面不使用这两个字段时，功能不受影响

2. **渐进式升级**
   - 先在缺陷管理页面试点
   - 验证无问题后，再推广到其他页面

### 6.2 数据兼容

1. **历史数据**
   - 历史数据的 `project_id` 和 `workspace_id` 保持不变
   - 新增筛选条件不影响历史数据的查询

2. **权限兼容**
   - 用户只能查看有权限的工作空间和项目
   - 后端需要根据用户权限过滤工作空间和项目列表

## 十、总结

1. **改动面小**：复用现有组件，只需添加配置和少量逻辑
2. **边界清晰**：改动集中在 search-components.js 和高级搜索组件
3. **可回滚**：新增功能不影响现有功能，可随时回滚
4. **扩展性强**：级联逻辑可复用到其他字段（如部门-用户）
