关键差异：
访问 http://127.0.0.1:8002/（project-management）→ 跳转到登录页面 ✅
访问 http://127.0.0.1:8009/（analytics-stat）→ 直接显示 HTML，但资源路径错误 ❌

问题根源： HTML 中的资源路径是 /analytics-stat/assets/...，但是直接访问后端服务（端口 8009）时，应该是 /assets/...（不带 /analytics-stat/ 前缀）。这是因为我们在 vite.config.ts 中设置了 base: '/analytics-stat/'，这个配置是为了通过 Gateway 访问时使用的，但直接访问后端服务时就会出错








根本原因
当点击"分析统计"菜单时，URL变成了 http://localhost:8000/analytics-stat/dashboard#/analytics-stat，这导致：

浏览器直接访问了analytics-stat后端服务，而不是通过主应用的路由
analytics-stat作为独立应用加载，不是作为qiankun微前端
主应用的布局（侧边栏、顶部栏）消失，因为完全是另一个应用
问题根源
主应用的路由配置中缺少 /analytics-stat 路由，导致：

Vue Router无法识别这个路径
浏览器直接访问了后端服务的URL
qiankun的 activeRule: '#/analytics-stat' 永远不会被触发
解决方案
我创建了主应用的路由配置文件：
analytics-stat.js

这个路由配置让主应用能够识别 /#/analytics-stat 路径，然后qiankun会在 #micro-app 容器中加载analytics-stat子应用。、






window.__POWERED_BY_QIANKUN__

undefined

window.location.hash

'#/analytics-stat'

window.location.href

'http://localhost:8000/analytics-stat/dashboard#/analytics-stat'

先变成 http://localhost:8000/#/analytics-stat，后变成 http://localhost:8000/analytics-stat/dashboard#/analytics-stat
