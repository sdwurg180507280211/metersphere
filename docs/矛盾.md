关键差异：
访问 http://127.0.0.1:8002/（project-management）→ 跳转到登录页面 ✅
访问 http://127.0.0.1:8009/（analytics-stat）→ 直接显示 HTML，但资源路径错误 ❌

问题根源： HTML 中的资源路径是 /analytics-stat/assets/...，但是直接访问后端服务（端口 8009）时，应该是 /assets/...（不带 /analytics-stat/ 前缀）。这是因为我们在 vite.config.ts 中设置了 base: '/analytics-stat/'，这个配置是为了通过 Gateway 访问时使用的，但直接访问后端服务时就会出错
