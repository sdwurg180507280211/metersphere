 # MeterSphere 项目打包文档

## 环境要求

- JDK 17+
- Maven 3.6+
- Docker
- Node.js 和 npm（前端打包需要）

## 打包步骤

### 1. 初始化项目依赖

```bash
cd /Users/edy/ideaProjects/metersphere

# 仅安装父 POM，不构建子模块（-N 表示非递归）
mvn install -N
```

### 2. 编译核心框架模块

```bash
mvn clean install -pl framework,framework/sdk-parent,framework/sdk-parent/domain,framework/sdk-parent/sdk,framework/sdk-parent/xpack-interface,framework/sdk-parent/jmeter
```

### 3. 配置镜像版本和打包路径

```bash
# 设置镜像版本
imageVersion=v2.10.23.02-lts
```

### 4. Test-Track 模块打包

```bash
# 进入 test-track 目录
cd /Users/edy/ideaProjects/metersphere/test-track

# 编译打包
mvn clean install -DskipTests

# 解压依赖
mkdir -p backend/target/dependency && (cd backend/target/dependency; jar -xf ../*.jar)

# 构建 Docker 镜像（注意 Dockerfile 在 test-track 目录下）
cd /Users/edy/ideaProjects/metersphere
docker build --build-arg MS_VERSION=${imageVersion} -t registry.fit2cloud.com/north/test-track:${imageVersion} -f test-track/Dockerfile .

# 添加到保存列表
saveImages="docker save registry.fit2cloud.com/north/test-track:${imageVersion} "
```

### 5. Project-Management 模块打包

```bash
# 进入 project-management 目录
cd /Users/edy/ideaProjects/metersphere/project-management

# 编译打包
mvn clean install -DskipTests

# 解压依赖
mkdir -p backend/target/dependency && (cd backend/target/dependency; jar -xf ../*.jar)

# 构建 Docker 镜像
cd /Users/edy/ideaProjects/metersphere
docker build --build-arg MS_VERSION=${imageVersion} -t registry.fit2cloud.com/north/project-management:${imageVersion} -f project-management/Dockerfile .

# 添加到保存列表
saveImages="docker save registry.fit2cloud.com/north/test-track:${imageVersion}  registry.fit2cloud.com/north/project-management:${imageVersion} "
```

### 6. 导出所有镜像到 tar 文件

```bash
# 完成保存命令
saveImages="docker save registry.fit2cloud.com/north/test-track:${imageVersion}  registry.fit2cloud.com/north/project-management:${imageVersion}  > /Users/edy/Desktop/metersphere-$(date +%y%m%d).tar"

# 显示命令（用于检查）
echo "${saveImages}"

# 执行保存命令
eval ${saveImages}
```
