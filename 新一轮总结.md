  2.5 Shared Commons Properties (/Users/edy/ideaProjects/metersphere/framework/sdk-parent/sd
  k/src/main/resources/commons.properties)

  This is loaded by every service via @PropertySource. Key configurations:

  - Database: HikariCP (max 200 connections, 10 min idle)
  - Quartz: Dedicated datasource (defaults to main DB), quartz.enabled=false by default
  - Kafka: Bootstrap servers from ${kafka.bootstrap-servers}, consumer group
  metersphere_group_id, producer max request 500MB
  - MyBatis: Cache enabled, camelCase mapping, 25s statement timeout
  - Flyway: Disabled by default (each service enables individually)
  - Eureka: http://localhost:8761/eureka/, prefer IP address, 5s heartbeat
  - Redis/Redisson: Config from file:/opt/metersphere/conf/redisson.yml
  - Session: 43200s timeout, default store type none
  - File Upload: 1024MB max
  - JMeter Home: /opt/jmeter
  - Compression: Gzip enabled for JSON/XML/HTML/JS/CSS

  2.6 External Configuration

  Each Spring Boot app loads external config via:
  @PropertySource(value = {
      "classpath:commons.properties",
      "file:/opt/metersphere/conf/metersphere.properties",
  }, encoding = "UTF-8", ignoreResourceNotFound = true)

  The metersphere.properties file (deployed at /opt/metersphere/conf/) provides runtime
  overrides for database URLs, Kafka brokers, Redis, MinIO, and other environment-specific
  settings.

  3.3 Backend Build Flow (per service module)

  Each business module follows this pattern (example: api-test):

  1. Frontend build (api-test/frontend/pom.xml): Uses frontend-maven-plugin to install Node
  v20.8.1 + npm 8.4.0 into ../../.node/, runs npm install then npm run build, outputting to
  dist/
  2. Backend build (api-test/backend/pom.xml): Uses maven-antrun-plugin to copy
  ../frontend/dist/ into src/main/resources/static/ (assets) and src/main/resources/public/
  (HTML). Then spring-boot-maven-plugin packages the fat JAR with the embedded frontend

  3.4 Gateway Build Flow

  The Gateway builds differently -- it copies the shared SDK frontend
  (framework/sdk-parent/frontend/dist/) into its src/main/resources/static/ instead of a
  per-service frontend. This is the main entry point for the UI.

  3.5 How to Start Locally

  # Build everything
  ./mvnw clean package -DskipTests

  # Or skip frontend builds
  mvn clean package -DskipTests -DskipAntRunForJenkins -pl "!framework/sdk-parent/frontend,!
  api-test/frontend,!performance-test/frontend,!project-management/frontend,!report-stat/fro
  ntend,!system-setting/frontend,!test-track/frontend,!workstation/frontend"

  # Start individual services (requires Eureka + MySQL + Redis + Kafka running)
  ./mvnw spring-boot:run -pl api-test/backend
  ./mvnw spring-boot:run -pl system-setting/backend
  # etc.

  4. Frontend Service Startup

  4.1 Micro-Frontend Architecture

  The project uses a micro-frontend architecture based on @micro-zoe/micro-app:

  - Gateway serves as the host/shell application, built from framework/sdk-parent/frontend
  - Each business module (api-test, test-track, etc.) builds its own child micro-app
  - The shared metersphere-frontend package provides common components, routing, i18n, Pinia
   stores, and layout

  4.2 Frontend Build via Maven

  Each frontend module's pom.xml uses the frontend-maven-plugin:
  <installDirectory>../../.node</installDirectory>  <!-- shared Node install -->
  <nodeVersion>v20.8.1</nodeVersion>
  <npmVersion>8.4.0</npmVersion>

  The Maven build runs three phases:
  1. install-node-and-npm -- installs Node into .node/
  2. npm install -- installs dependencies
  3. npm run build -- runs vue-cli-service build

  4.3 Frontend Dev Server (vue.config.js)

  Example for API Test
  (/Users/edy/ideaProjects/metersphere/api-test/frontend/vue.config.js):
  - Dev server port: 4004
  - Proxies all requests (except /login and /document) to http://localhost:8004 (the
  api-test backend)
  - WebSocket proxy at /websocket to http://localhost:8004
  - Multi-page setup: index.html, share-api-report.html, share-document.html, api-doc.html

  For the shared SDK frontend
  (/Users/edy/ideaProjects/metersphere/framework/sdk-parent/frontend/vue.config.js):
  - Dev server port: 3000
  - Proxies to http://localhost:8000 (the gateway)
  - Built as library MsFrontend

  4.4 Local Frontend Development

  5.2 Database (MySQL)

  - Driver: com.mysql:mysql-connector-j (in SDK)
  - Connection pool: HikariCP (max 200 connections, 10 min idle, 15 min max lifetime)
  - Migration: Flyway, enabled per service with separate version tables:
    - api-test -> api_version
    - performance-test -> performance_version
    - project-management -> project_management_version
    - system-setting -> metersphere_version
    - test-track -> track_version
    - report-stat -> report_version
    - analytics-stat -> analytics_version
    - workflow-service -> workflow_version
  - Migration scripts: Located at {module}/backend/src/main/resources/db/migration/
  - The actual DB URL, username, and password are supplied at runtime via
  /opt/metersphere/conf/metersphere.properties
  - Workflow service uses a separate datasource:
  spring.datasource.url=${workflow.datasource.url}
  - Quartz scheduler uses a separate datasource config (defaults to main DB)

  5.3 Redis / Redisson

  - Redisson client: redisson-spring-boot-starter:3.25.0
  - Config loaded from: file:/opt/metersphere/conf/redisson.yml
  - Spring Session: spring-session-data-redis with repository-type=indexed
  - Session timeout: 43200s (12 hours)
  - Default store type: none (Redis session storage activated via external config)

  5.4 Kafka

  - Spring Kafka with bootstrap servers from ${kafka.bootstrap-servers}
  - Consumer group: metersphere_group_id
  - Producer: acks=1 (api-test), max request size 524MB, buffer 524MB
  - Consumer: max partition fetch 52MB
  - Analytics service has additional Kafka config for knowledge-base processing (topic:
  knowledge-file-processing)
  - JMeter Kafka Backend Listener configuration is included in commons.properties

  5.5 Service Discovery (Eureka)

  - Eureka server at port 8761 (standalone, self-preservation disabled)
  - All backend services register as Eureka clients
  - Gateway discovers services via Eureka
  (spring.cloud.gateway.discovery.locator.enabled=true)
  - Heartbeat interval: 5s, expiration: 15s, registry fetch: 10s

  5.6 API Gateway (Spring Cloud Gateway)

  - Port 8000, WebFlux-based (not servlet)
  - Routes analytics service explicitly (others auto-discovered via Eureka)
  - Connect timeout: 30s, response timeout: 5 min
  - WebSocket max frame: 200MB
  - Swagger/OpenAPI aggregation for all services
  - Serves the main frontend shell (from framework/sdk-parent/frontend/dist/)

  5.7 Other Infrastructure

  - MinIO: Object storage for file uploads (config via MinioProperties, version 8.5.3)
  - JMeter: Embedded JMeter 5.5 engine for test execution (installed at /opt/jmeter)
  - Flowable: Workflow engine in workflow-service (flowable.database-schema-update=true)
  - Shiro: Authentication/authorization framework (version 2.0.1, Jakarta classifier)
  - LDAP: spring-boot-starter-data-ldap (health check disabled by default)
  - Quartz: Job scheduling (enabled in most services except workstation)
  - K8s Client: fabric8:kubernetes-client for container orchestration
  - Selenium: selenium-java:4.10.0 for UI automation testing
  - Elasticsearch: Referenced in analytics service for knowledge-base search (config via
  external metersphere.properties)

  ---
  配置文件体系

  1. commons.properties (framework/sdk-parent/sdk/src/main/resources/) —
  所有服务共享的基础配置，通过 @PropertySource 加载
  2. 各服务 application.properties ({module}/backend/src/main/resources/) —
  服务端口、Eureka注册、Flyway等
  3. 外部运行时配置 /opt/metersphere/conf/metersphere.properties —
  数据库URL、Kafka、Redis等环境相关配置
  4. Redisson /opt/metersphere/conf/redisson.yml — Redis客户端配置

  关键共享配置内容：
  - HikariCP 连接池最大200连接
  - Kafka producer 最大请求 524MB，consumer group metersphere_group_id
  - Redis/Session 12小时超时，Spring Session indexed 模式
  - 文件上传 最大 1024MB
  - Eureka 心跳5s，过期15s

  ---
  前端架构（微前端）

  - 主壳应用：framework/sdk-parent/frontend (dev port 3000)，使用 @micro-zoe/micro-app
  加载子应用
  - 子应用：每个业务模块有独立前端，如 api-test (port 4004) 等
  - 共享库：metersphere-frontend 封装通用组件、路由、i18n、Pinia Store

  ---
  后端启动方式

  # 全量构建（含前端）
  ./mvnw clean package -DskipTests

  # 仅构建后端（跳过前端）
  ./mvnw clean package -DskipTests -DskipAntRunForJenkins \
    -pl "!framework/sdk-parent/frontend,!api-test/frontend,!performance-test/frontend,\
  !project-management/frontend,!report-stat/frontend,!system-setting/frontend,\
  !test-track/frontend,!workstation/frontend"

  # 启动单个服务（需要先启动 Eureka + MySQL + Redis + Kafka）
  ./mvnw spring-boot:run -pl api-test/backend
  ./mvnw spring-boot:run -pl system-setting/backend

  ---
  前端开发启动

  # 启动主壳（Gateway前端）
  cd framework/sdk-parent/frontend && npm install && npm run gateway   # port 3000 → proxy 
  :8000

  # 启动子应用（以 API Test 为例）
  cd api-test/frontend && npm install && npm run api   # port 4004 → proxy :8004

  # 启动工作站前端
  cd workstation/frontend && npm install && npm run workstation
