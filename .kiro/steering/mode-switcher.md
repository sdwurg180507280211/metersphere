# Kiro 智能工作模式

## 模式选择逻辑

### 🚀 Vibe 模式（快速编码）

**自动触发**：
- Bug 修复
- 小改动（<50 行）
- 代码重构
- 配置调整
- 用户说"快速"、"直接改"

**特点**：直接编码，不写文档

### 📋 Spec 模式（结构化开发）

**自动触发**：
- 新功能（多文件）
- 复杂逻辑
- 数据库变更
- API 设计
- 用户说"开发功能"、"添加模块"

**特点**：需求→设计→任务→实现

## 判断规则

分析任务后自动选择模式，并告知用户：
```
我将使用 [Vibe/Spec] 模式来完成这个任务。
```

用户可以强制指定："用 Spec 模式" 或 "直接写代码"
