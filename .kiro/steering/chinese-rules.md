## 二、主动行为（核心·必须遵守）

### 2.1 直接执行原则（最高优先级）
```
❌ 禁止：你需要我执行xxx吗？/ 是否需要我查询？/ 要我帮你xxx吗？
✅ 正确：直接执行，直接给结果
```

**行为准则**：
- 能做的事情立即做，不询问确认
- 需要查库就直接查，不问"要不要查"
- 需要看文件就直接看，不问"要不要看"
- 需要执行命令就直接执行，不问"要不要执行"
- 只在涉及**破坏性操作**（删除、覆盖、修改生产数据）时才询问确认

### 2.2 数据库自动查询
当问题涉及以下场景时，**立即主动查询数据库**：

| 触发场景 | 查询目标 |
|---------|---------|
| 涉及表结构、字段 | 先查 information_schema 获取表结构 |
| 涉及业务数据 | 直接查询相关业务表 |
| 涉及数据关系 | 查询外键、关联表 |
| 涉及配置项 | 查询 system_parameter 等配置表 |
| 涉及工作流 | 查询 metersphere_workflow 库 |
| 不确定表名/字段名 | 先 SHOW TABLES / DESC 表名 |
| 代码中出现实体类 | 查询对应表结构验证字段 |

**执行原则**：
- 先查后答，用数据说话
- 不猜测表结构，以实际查询结果为准
- 复杂问题可多次查询，逐步深入

### 2.3 数据库连接信息
```bash
mysql -h localhost -u root -p'Password123@mysql' -e "USE metersphere_dev;"
```

### 2.5 常用查询模板
```sql
-- 查看所有表
SHOW TABLES;

-- 查看表结构
DESC 表名;

-- 查看建表语句（含索引、外键）
SHOW CREATE TABLE 表名;

-- 模糊搜索表名
SHOW TABLES LIKE '%关键词%';

-- 搜索包含某字段的表
SELECT TABLE_NAME, COLUMN_NAME 
FROM information_schema.COLUMNS 
WHERE TABLE_SCHEMA = 'metersphere_test' 
AND COLUMN_NAME LIKE '%关键词%';
```

## 三、代码规范

### 3.1 质量标准
- 遵循最新最佳实践，确保代码健壮、兼容性好、可直接运行
- 高内聚、低耦合
- 不重复造轮子，优先使用现有工具类

### 3.2 注释要求
- 添加详细中文技术注释
- 解释所有语法糖和简写语法
- 说明复杂表达式的逻辑
- 标注关键函数/方法的作用

### 3.3 二次开发原则（核心）
| 原则 | 说明 |
|------|------|
| 改动面小 | 尽量少改官方核心链路 |
| 边界清晰 | 集中在少数模块/少数文件 |
| 可回滚 | 升级时能快速对照、快速搬运 |

## 五、回答规范

### 5.1 操作说明句式（必须）
每一步按以下格式说明：
```
- 我在做...（具体操作）
- 目的是...（为什么要这样做）
- 如果不这样做，就无法实现...（后果/替代方案）
```

### 5.2 工具依赖检查（必须）
使用任何命令或工具前，递归检查依赖链：
1. 从最底层开始（Mac自带: curl、which等）
2. 逐层向上检查每个依赖工具是否存在
3. 考虑工具版本差异
4. 对缺失工具给出完整安装路径
5. 不假设任何非系统自带工具已存在

### 5.3 文档规范
- 格式: 一律 Markdown
- 风格: 重思路，轻代码
- 触发词: 问题包含【文档】时，回答后自动整理成文档

```
1. 直接执行 > 询问确认（除非破坏性操作）
```
